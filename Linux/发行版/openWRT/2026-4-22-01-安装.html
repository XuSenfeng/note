下载固件

树莓派 4B 官方固件下载：https://downloads.openwrt.org/releases/23.05.5/targets/bcm27xx/bcm2711/

下载文件：openwrt-23.05.5-bcm27xx-bcm2711-rpi-4-squashfs-factory.img.gz

  • openwrt-23.05.5：OpenWRT 版本号（23.05 系列第 5 个稳定版）
  • bcm27xx-bcm2711：目标芯片平台（树莓派 4B 使用的 Broadcom BCM2711 芯片）
  • rpi-4：具体设备型号（Raspberry Pi 4）
  • squashfs：根文件系统格式
  • factory：刷写场景（第一次烧录）
  • .img：原始磁盘镜像格式
  • .gz：gzip 压缩格式（烧录工具会自动解压）

squashfs-factory是 OpenWRT 固件文件名中两个独立关键标识的组合，分别定义了固件的根文件系统格式和刷写场景

SquashFS 是一种高压缩率的只读 Linux 文件系统，是 OpenWRT 官方推荐的标准格式，专为嵌入式设备有限的存储空间设计。

  • 工作原理

OpenWRT 采用 "只读 SquashFS + 可写 OverlayFS" 的双分区架构：

SquashFS 分区：存放系统核心文件（内核、驱动、基础软件），永远只读，任何修改都不会改变这个分区OverlayFS 分区：存放用户的所有配置和安装的软件，可写，位于 SD 卡的剩余空间

  • 固件模式

factory表示这是工厂固件，专门用于设备还在运行原厂系统时的第一次刷写。

对比：factory vs sysupgrade（另一种常见固件类型）

对比维度 factory 固件 sysupgrade 固件
刷写场景 第一次刷写（原厂系统→OpenWRT） 系统升级（OpenWRT→新版 OpenWRT）
包含内容 完整的系统镜像（包含分区表、bootloader、内核、根文件系统） 仅包含内核和根文件系统
刷写方式 用 SD 卡烧录工具（如 Raspberry Pi Imager）直接写入 SD 卡 在 OpenWRT 的 LuCI 界面或命令行中升级
数据保留 会清除所有数据 默认保留用户配置

连接

连接控制opwnWRT的时候, 首先需要使用一个网线

默认的ip地址是192.168.1.1, 在Windows配置静态地址以后就可以连接, 可以通过网页的形式进行控制

第一个是物理网卡, 第二个是无线网卡, 目前使用第一个连接电脑进行通信, 第二个连接热点进行外网的连接

在这个网页里面进行无线网络的连接, 这里连接了jiao这个wifi, radio0 是你树莓派自带的物理无线网卡硬件，是底层的、唯一的设备

下面的客户端是基于 radio0 配置的一个软件层面的无线接口，是给硬件加的 “工作规则”, 模式是 Client（客户端模式），也就是让 radio0 主动去连你家主路由的 jiao WiFi, 这个配置绑定了你的 wwan 接口，是你树莓派现在的外网出口

原理

OpenWrt 为了防崩溃、防闪存损坏、好升级，设计成了双层分离架构：

  1. 上层：只读系统分区（固件本体）

对应你的：/dev/mmcblk0p2

  • 格式：压缩只读文件系统 SquashFS

  • 特点：

    • 内核、驱动、基础命令、网络核心配置 全在这里
    • 完全不能写入、不能修改
    • 体积很小（官方固件就几十 MB）

  • 作用：保证系统本体永远纯净、不坏、不乱改

  1. 下层：可读写数据分区（overlay）

挂载目录：/overlay

  • 唯一能写入、能保存数据的地方
  • 所有改动全部存在这里：
    • 安装的插件 / 软件
    • 你改的网络配置、密码、后台设置
    • 缓存、日志、docker、自定义脚本

系统原理：OverlayFS 联合挂载

把「只读系统 p2」+「可写 overlay」拼在一起，拼成你看到的完整 / 根目录。

初始的SD卡

/dev/mmcblk0   整张SD卡
 ├─ p1  boot    启动分区（FAT32，引导树莓派开机，不动）
 ├─ p2  root    只读系统固件（不可写、不能扩容）
 └─ p3  我们新建的空分区（整块剩余大容量空间，ext4）

SD 卡除了 p1/p2，剩下几十 G 都是空白未分配，

我们新建一个独立 Linux 分区，专门拿来当大容量 overlay。OpenWrt 的 overlay 必须用 Linux 标准文件系统 ext4，FAT32/NTFS 权限不达标，不能当做系统读写目录。

  • OpenWrt 开机流程：
  1. 先加载只读系统 → 读取fstab挂载表 → 自动挂载 /overlay

原本：系统自动用「p2 里一小块极小空间」挂载为 /overlay

改完之后：强制让系统开机自动把大容量分区 /dev/mmcblk0p3 挂载到 /overlay
