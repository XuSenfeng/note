文件系统02

2026-03-06
编辑本页

根文件挂载

fs\super.c

在sys_setup函数里面调用mount_root

  • 初始化文件表数组
  • 初始化超级块数组
  • 读取根文件设备的超级块
  • 从这个磁盘读取第一个inode, 设置引用数量为4, p->s_isup = p->s_imount = mi;自己挂载一下自己, 当前进程的pwd和root也是他
  • 统计设备的空闲块和inode的数量

文件

fs\namei.c

文件的权限, 文件系统的操作chmod chown mknod, 内存区域的管理检索

权限

  • permission: 获取当前的请求的用户对于一个inode的权限

查询

  • match: 文件名和一个文件夹里面的项的匹配
  • find_entry: 在指定的目录中查找一个与名字匹配的目录项, 返回对应的dir_entry结构, 和目录文件缓冲区
  • get_dir根据一个路径的名称获取一个目录, 获取对应的inode节点, 是一个处理字符串, 依次查询inode的函数, 不是目录返回NULL, 返回最后一个名字之后是/的目录
  • dir_namei: 在 get_dir() 基础上，额外返回“最后一段名字”和长度，用于创建/查找该名字, 获取的是顶层目录的最后一个/之前的目录的inode
  • namei: 获取指定的路径的i节点
  • open_namei: 打开指定的文件的inode, 使用文件的打开标志以及属性, 可以打开普通文件和目录文件, 可以使用标志创建文件

文件夹

struct dir_entry {
    unsigned short inode;          // i 节点
    char name[NAME_LEN];           // 文件名
};

目录文件的数据块就是一串 struct dir_entry 记录，用来把“名字 → i 节点号”映射起来

i 节点本身不直接包含文件夹内容，而是通过数据块指针 i_zone[] 指向存储目录项的数据块。

  • add_entry在一个目录文件里面添加新的目录项
  • sys_mknod: 创建一个特殊文件或普通文件节点。由mod和dev指定文件系统节点, 可以创建块设备文件, 字符型文件, 命名管道和 普通文件, 权限检查：必须是超级用户才能调用, 如果是设备文件，将设备号保存在 i_zone[0]
  • sys_mkdir创建目录, 添加两个子项, .和..
  • empty_dir: 目录是不是为空
  • sys_rmdir: 删除目录, 没有递归处理子目录
  • sys_unlink: 从文件系统删除一个名字, 不删除目录。如果是一个文件的最后一个连接，并且没有进程正打开该文件, 则该文件也将被删除，并释放所占用的设备空间
  • sys_link: 给文件创建一个link, 实际是创建一个新的目录项指向同一个inode, 不能是目录文件

管道

节点获取

获取管道节点, 使用的函数是get_free_page, 在这个函数里面申请一个inode结构, 为这个节点分配一个页面, 记录在i_size里面, 之后设置一下标志位, 以及设置引用次数为2(一读一写), 头记录在i_zone[0], 尾记录在i_zone[1]里面

虚拟文件系统

不同的文件在操作的时候, 使用相同的文件接口, 在上层文件操作的时候, 去查找的是接口层VFS

操控设备的时候依据主从设备号把不同的设备封装成一个文件

fd=>file_table=>file=>inode=>i_zone

struct file { 
    unsigned short f_mode;      // 文件操作模式（RW 位）    
    unsigned short f_flags;     // 文件打开和控制的标志, 只读只写等
    unsigned short f_count;     // 这个文件被引用的次数
    struct m_inode * f_inode;   // 指向对应 i 节点
    off_t f_pos;                // 文件位置（读写偏移值）
};

读写函数

更具文件的类型不同使用不同文件里面的处理函数

  • file_dev.c: 读取文本文件
    • file_read: 普通文件的读取, 读取的文件小于读取的长度的时候, 在buf最后补0
  • block_dev.c: 块设备文件
    • block_write:
    • block_read:
  • pipe.c
    • read_pipe: 从循环缓冲区读取数据
    • write_pipe: 循环缓冲区写入数据
  • char_dev.c
    • rw_char: 使用设备号, 从crw_table加载不同的设备的操作函数
    • rw_port: 端口的读写函数
    • rw_tty: 串口的读写函数, 使用outb
    • rw_memory: 读写内存, 操作几种不同的内存, 实际实现的只有读写端口

Open函数

从file的全局数组里面获取一个空闲的位置, fd是当前任务的file指针数组里面的位置

在处理的时候需要处理tty终端, 只有先打开设备号为4的tty, 这个时候当前进程的tty标号设置为4, 之后才可以打开tty==5

utime设置时间

配置读写当前文件的时间

可执行文件加载

程序在运行之前存放在磁盘上面, 运行的时候加载程序的头到高速缓冲区, 运行的时候使用fork函数, 程序运行的时候需要记录参数和环境变量, 设置栈

参数拷贝

copy_string: 参数的拷贝, 传入参数的个数, 参数指针数组, 当前分配的页记录数组, 记录的数据偏移的位置, 拷贝的参数空间配置

需要把参数记录在当前的环境里面, 一个最大连续的32页的位置, 这个函数需要处理的用户和内核等不同权限之间数据段的参数拷贝

数据放到新进程用户栈区域, 目前只是分配了段, 随后这些页被映射到新数据段顶部（也就是栈顶附近）, copy string只是分配了页, 还没有映射, 由于这给函数只会在内核态使用, 内核做了恒等映射, 所以可以直接操作

cs: 代码段

ds: 数据段

es: 额外段

ss 栈

fs 用户数据段(用户数据空间)

在用户态的时候, ds和fs是等价的, 但是在内核态的时候, ds是内核的数据段, fs还是用户的数据段, 这个函数在处理的时候徐切换fs为实际的数据所在的位置, 同时保证退出的时候复原

#define MAX_ARG_PAGES 32
/*
 * 需要处理的是参数表和实际的数据之间的映射
 * from_kmem     argv *        argv **
 *    0          user space    user space
 *    1          kernel space  user space
 *    2          kernel space  kernel space
*/

change_ldt

修改局部描述符表中的描述符基址和段限长，并将参数和环境空间页面放置在数据段末端

根据代码段的大小, 重新配置一下代码段数据段的LDT的数值, 重新加载fs, 之后根据之间记录的参数的page, 从后向前配置页表真实地址和虚拟地址的映射

create_tables

在新用户堆栈中创建环境和参数变量指针表, 依次解析之前的参数表, 创建

  • argv[] 参数指针表
  • envp[] 环境变量指针表

现在的栈里面的数据

  • argc
  • argv 指针
  • envp 指针
  • argv[] ... NULL
  • envp[] ... NULL
  • 后面是前面 copy_strings() 已拷好的字符串数据区（参数串+环境串）

exec

使用函数do_execve进行执行

  • 检查一下当前是不是用户态调用进入内核态的, 只可以在用户态的时候时候调用
  • 打开指定文件的inode, 判断是不是普通文件
  • 依据文件的性质, 获取一下执行的用户和组ID
  • 读取文件的第一个块
    • 脚本文件
        1. 读取执行器的名字
        2. 获取执行器的参数(如果存在)
        3. 如果是第一次获取到#!, 复制环境参数和函数参数列表到栈里面
        4. 把文件名和执行器参数, 解释器的名字复制到栈里面
        5. 加载解释器的inode
        6. 返回之前的步骤, 加载执行器的可执行文件, 这个脚本的路径作为参数被bash执行
      • 可执行文件
        1. 判断一下这个文件执行的模式, 代码段只读, 可以按需求加载, 并且大小合适
        2. 不是shell脚本的话, 加载一下环境变量和基础变量
        3. 记录当前进程的执行的inode变量
        4. 重置一下当前task的信号处理函数
        5. 关闭一下设置了退出市场关闭标志位的文件
        6. 释放之前的ldt表里面的表项
        7. 释放协处理器
        8. 重新配置一下ldt表里面的偏移以及可以控制的大小
        9. 配置一下页表里面的映射, 把之前分配的物理地址进行映射
        10. 创建参数的表
        11. 配置一下当前任务里面的各种参配置eip是任务入口, esp是栈的位置

Linux 0.11 采用需求分页（demand-paging），bss 段的页面在首次访问时触发缺页中断，do_no_page() 会分配一个已清零的空页（get_free_page() 返回的页面在分配时已清零）
