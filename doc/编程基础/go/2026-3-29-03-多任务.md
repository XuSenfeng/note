# 多任务

通常的多进程在开发的时候越来越困难, 从任务到线程到协程

## goroutine

Golang因其高效的并发编程能力而闻名，而**Goroutine**正是这一能力的核心。它像一个“**轻量级线程**”，但比传统线程更灵活、更节省资源。

![img](https://pic3.zhimg.com/v2-8e9f94308e908d319f7e0b2e33027416_1440w.jpg)

- **M（OS线程）**：操作系统线程，是Goroutine的载体。
- **N（Goroutine）**：协程，由调度器分配到M上执行。
- **核心思想**：用少量OS线程（M）高效复用大量Goroutine（N），避免线程切换的高昂开销。

全局对了用于分配不同的Goroutine(协程)，等待分配到OS线程, 每个处理器（Processor）有自己的本地队列，优先从本地队列调度任务, M(thread线程)内核里面的物理线程

> 一个Goroutine占用的资源只有几KB

### 设计策略

+ 复用线程

work stealing机制: 当一个P的本地队列空闲时，会“偷”其他P的本地队列任务

hand off机制: 在运行的一个协程出现阻塞的时候, 唤醒一个新的线程, 把这个线程里面的其他G切换过去

+ 利用并行: 可以通过GOMAXPROCS的参数限制最大的P的个数
+ 抢占: 

**Go 1.14前**：Goroutine需主动让出CPU（协作式调度）。

**Go 1.14后**：支持异步抢占，可在安全点强制切换，防止“霸占CPU”。

+ 全局G队列

当一个P没有可执行的G, 同时其他的调度器里面也没有的时候, 会从全局队列里面获取芯的G(减少锁的使用)

### 实际使用

```go
package main

import (
	"fmt"
	"time"
)

func NewTask() {
	i := 0
	for i < 10 {
		fmt.Println("Task is running")
		i++
		time.Sleep(1 * time.Second)
	}
}

func main() {
	go NewTask() // 创建一个新的 goroutine 来执行 NewTask 函数

	// 主 goroutine 继续执行其他任务
	for i := 0; i < 5; i++ {
		fmt.Println("Main function is running")
		time.Sleep(2 * time.Second)
	}
}
```

> 主goroutine退出的时候, 其他的goroutine也退出

在一个goroutine里面退出当前的协程可以使用`runtime.Goexit()`

```c
package main

import (
	"fmt"
	"sync"
	"time"
)

func main() {
	var wg sync.WaitGroup // 创建一个WaitGroup对象
	wg.Add(1) // 添加一个等待的goroutine
	go func() {
		defer wg.Done()
		for i := 0; i < 5; i++ {
			fmt.Println("Hello, World!")
			time.Sleep(1 * time.Second)
		}
	}()
	wg.Wait()
}
```

## channel通信

两个goroutine之间的通信使用的是这个

+ 创建

```go
make(chan Type)
make(chan Type, capacity) //带缓冲
```

+ 写入读取

```go
channel <- value // 写入
<-channel // 读取
x := <-channel
x, ok := <-channel
```

+ 关闭

```go
close(c)
```



### 使用

```go
package main

import (
	"fmt"
	"sync"
	"time"
)

func main() {
	c := make(chan int)
	var wg sync.WaitGroup
	wg.Add(1)

	go func() {
		defer wg.Done() // 确保在 goroutine 结束时调用 Done() 来通知 WaitGroup
		for i := 0; i < 5; i++ {
			c <- i
			time.Sleep(500 * time.Millisecond)
		}
		close(c)
	}()

	go func() {
		for num := range c {
			fmt.Println(num)
		}
	}()

	wg.Wait()
}
```

- wg.Add(1) 表示“我要等待1个 goroutine 完成”。
- 第一个 goroutine（生产者）负责往 channel 里写数据，写完后 close(c)，并调用 wg.Done()。
- 第二个 goroutine（消费者）只是读取 channel 数据，并不需要主 goroutine等待它结束，因为它会在 channel 关闭后自动退出。
- 主 goroutine 只需 wg.Wait() 等待生产者 goroutine 完成即可，消费者 goroutine 会随着 channel 关闭自然退出。

默认会进行阻塞

### 缓冲

在没有缓冲的时候, 写入以及读取的时候, 一定要等待另一个协程一起到达的时候才会进行数据的传递

有缓冲的时候可以暂存一定数量的数据, 可以使用`len`以及`cap`查看当前通道的里面数据的个数以及缓冲区的大小
