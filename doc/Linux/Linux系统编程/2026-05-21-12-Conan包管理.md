# 包管理

Conan是一款免费开源的 C/C++语言的依赖项和包管理器，适用于所有平台（Windows，Linux，OSX，FreeBSD，Solaris等）。可用于开发所有目标，包括嵌入式，移动（iOS，Android）和裸机。conan的角色类似于anaconda

## 配置文件

使用的配置文件名字是 conanfile.txt , 里面记录使用的库以及版本, 所有的库可以在 [这个仓库](https://conan.io/center) 里面查看

![Pasted image 20260521111957|246](https://picture-01-1316374204.cos.ap-beijing.myqcloud.com/lenovo-xiangong/20260521112041961.png?imageSlim)

Conan负责：
- 下载、构建和部署所有库到我们的系统；
- 输出一个文件，告诉我们的操作系统 .bin 到包含的路径。


```bash
[requires]
cjson/1.7.17

[generators]
cmake_find_package
```

`[generators]` 里 `cmake_find_package` 会生成供 CMake 使用的查找文件（类似 `Findcjson`），让你在 CMake 里 `find_package(cjson)`

在当前 conanfile 所在路径打开

```bash
conan install .
```

这个会报错，缺少所需的依赖项。所以我们需要告诉 conan 根据下载的配方构建任何遗漏的东西

```bash
conan install . --build missing
```

如果安装错了，可以根据库名称移除已安装的库

```bash
conan remove <lib_name>
```

安装之后可以获取到以下的文件

```bash
.
├── conanbuildinfo.txt
├── conanfile.txt
├── conaninfo.txt
├── conan.lock
├── FindcJSON.cmake
└── graph_info.json
```

- `conanbuildinfo.txt`：纯文本依赖信息（include/lib/path 等），通常给手动脚本或非 CMake 使用。
- `conanbuildinfo.cmake`：给 CMake 用的依赖配置文件。
- `conaninfo.txt`：本次安装解析出的依赖版本、选项、配置摘要。
- `graph_info.json`：依赖图的机器可读信息（调试/可视化用）。
- `FindcJSON.cmake`: 让 `find_package(cJSON)` 能成功
- + cJSON 找到了，版本是 1.7.17
- + 头文件在哪里：Conan 缓存目录里的 include
- + 库文件在哪里：Conan 缓存目录里的 lib
- + 可以链接的目标/组件是什么：比如 cJSON::cjson
这些生成的文件在项目 git 上都不需要提交，所以编写一个 `.gitignore` 来排除这些更改

```text
# Exclude conan-generated files
conan.lock
conanbuildinfo**
conaninfo.txt
graph_info.json
```

- Conan 在构建目录写入 FindcJSON.cmake
- `list(APPEND CMAKE_MODULE_PATH ...)` 让 CMake 能找到它
- `find_package(cJSON REQUIRED)` 拿到 `cJSON::cjson`
## 测试程序

```c
#include <stdio.h>
#include <cjson/cJSON.h>

int main(void) {
    cJSON *root = cJSON_CreateObject();
    cJSON_AddStringToObject(root, "hello", "world");
    char *out = cJSON_Print(root);
    puts(out);

    cJSON_free(out);
    cJSON_Delete(root);
    return 0;
}
```
可以使用Cmake进行编译
```cmake
cmake_minimum_required(VERSION 3.10)
project(cjson_demo VERSION 1.0 LANGUAGES C CXX)
  
set(CMAKE_C_STANDARD 99)
set(CMAKE_C_STANDARD_REQUIRED ON)
set(CMAKE_C_EXTENSIONS OFF)
  
set(CMAKE_CXX_STANDARD 11)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
set(CMAKE_CXX_EXTENSIONS OFF)
  
add_executable(app src/main.cpp)
  
# Conan's cmake_find_package generator writes FindcJSON.cmake into the build dir.
list(APPEND CMAKE_MODULE_PATH "${CMAKE_CURRENT_BINARY_DIR}")
  
# 由 Conan 生成的 FindcJSON.cmake 提供 cJSON::cjson
find_package(cJSON REQUIRED)
target_link_libraries(app PRIVATE cJSON::cjson)
```
