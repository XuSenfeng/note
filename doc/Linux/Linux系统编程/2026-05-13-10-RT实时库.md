Linux系统中的librt库是一个支持多种POSIX可移植性技术的运行时库，用于提供Linux系统的基本实现。
在CMake编译的时候添加一行
```cmake
target_link_libraries(lowlevel_canfd PRIVATE rt)
```
添加这个库可以用于精确延时,获取准确时间等

## 获取精确时间

使用头文件

```c
 #include <time.h>
 int clock_gettime(clockid_t clockid, struct timespec *tp);
```
获取的时间数据如下
```c
struct timespec {
   time_t   tv_sec;        /* seconds */
   long     tv_nsec;       /* nanoseconds */
};
```

>CLOCK_MONOTONIC **时钟单调**
一个不可设置的系统级时钟，代表单调时间。由于——正如 POSIX 所描述的——“在某个未指定的点过去”。在 Linux 系统中，这一点对应于秒数。表明系统自启动以来一直在运行。  
**CLOCK_MONOTONIC** 时钟不受不连续性的影响系统时间跳跃（例如，如果系统管理员
手动更改时钟），但受增量影响-[adjtime(3)](https://www.man7.org/linux/man-pages/man3/adjtime.3.html) 和 NTP 执行 tal 调整。此时钟不计入系统暂停时间。所有**CLOCK_MONOTONIC** 变体保证返回的时间连续的呼叫不会倒退，但连续的呼叫可能——取决于架构——返回相同的（非）增加）时间值。
## 示例

+ 延时
```c
static void nsleep(long ns)
{
    struct timespec ts;
    ts.tv_sec  = 0;
    ts.tv_nsec = ns;
    nanosleep(&ts, NULL);
}
```

+ 定时轮训

```c
struct timespec t_start, t_end;
// 获取开始的时间
clock_gettime(CLOCK_MONOTONIC, &t_start);


// 处理各种事务

/* ── Rate-limit to 1ms ── */

clock_gettime(CLOCK_MONOTONIC, &t_end);
// 计算实际处理的时间
long elapsed_ns = (t_end.tv_sec - t_start.tv_sec) * 1000000000L
				+ (t_end.tv_nsec - t_start.tv_nsec);

// 当前周期如果还没到1ms，睡眠剩余时间；如果已经过了1ms，
// 直接进入下一周期（可能会有积累的延迟）
if (elapsed_ns < period_ns)
	nsleep(period_ns - elapsed_ns);
```
