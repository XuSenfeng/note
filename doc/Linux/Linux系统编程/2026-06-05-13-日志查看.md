# 日志查看

## journalctl

`journalctl`是Linux系统中用于查看和管理日志的核心工具，尤其在基于 systemd 的系统（如Ubuntu、CentOS、Fedora等）中
`journalctl`是`systemd`日志系统（称为`[journald](https://zhida.zhihu.com/search?content_id=259910258&content_type=Article&match_order=1&q=journald&zhida_source=entity)`）的日志查看工具。它统一管理系统的所有日志，包括：

- **内核日志**：硬件驱动、设备连接等。
- **服务日志**：Apache、MySQL、Docker等服务的运行状态。
- **用户程序日志**：通过`systemd`管理的用户级程序。

### 实际使用

```bash
# 查看所有日志
journalctl

# 当前启动以后
journalctl

# 制定时间
journalctl --since "1 hour ago"
journalctl --since "2025-04-29 09:00:00" --until "2025-04-29 10:00:00"

# 指定单元
journalctl -u nginx.service

# 实时监控
journalctl -p err -f
```

| 参数           | 作用                                                | 示例                              |
| ------------ | ------------------------------------------------- | ------------------------------- |
| -f           | 实时跟踪日志（类似tail -f）                                 | journalctl -f -u docker.service |
| -k           | 仅显示内核日志                                           | journalctl -k                   |
| -p <优先级>     | 按日志优先级过滤（如emerg, alert, crit, err, warning, info） | journalctl -p err               |
| --no-pager   | 直接输出日志（不分页）                                       | journalctl --no-pager           |
| -o json      | 以JSON格式输出日志（适合编程处理）                               | journalctl -o json              |
| --disk-usage | 查看日志占用的磁盘空间                                       | journalctl --disk-usage         |
