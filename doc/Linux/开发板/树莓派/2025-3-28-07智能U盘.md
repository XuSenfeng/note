## 模拟键盘

在Linux下面使用USB设备的模拟实际是可以通过创建一系列文件就可以实现

[使用树莓派 Zero 实现带回显的新型 Bad USB | 树莓派实验室](https://shumeipai.nxez.com/2018/06/26/using-raspberry-pi-zero-to-implement-new-bad-usb-with-echo.html)

添加配置文件`sudo vim /boot/firmware/config.txt`最下面

```bash
dtoverlay=dwc2
dr_mode=peripheral
```

保存为 `sudo /usr/local/bin/enable-hid-gadget.sh`

```bash
#!/bin/bash
# USB HID Gadget 开机自启脚本
set -e

# 1. 加载核心依赖模块，确保configfs目录存在
modprobe libcomposite
sleep 0.5 # 等待模块加载完成，目录创建

# 2. 进入configfs usb_gadget根目录（绝对路径，避免开机路径错误）
cd /sys/kernel/config/usb_gadget/

# 3. 清理已存在的gadget实例（防止开机重复执行报错）
if [ -d g1 ]; then
    echo "" > g1/UDC 2>/dev/null || true
    rm -f g1/configs/c.1/hid.usb0 2>/dev/null || true
    rmdir g1/configs/c.1/strings/0x409 2>/dev/null || true
    rmdir g1/configs/c.1 2>/dev/null || true
    rmdir g1/functions/hid.usb0 2>/dev/null || true
    rmdir g1/strings/0x409 2>/dev/null || true
    rmdir g1 2>/dev/null || true
fi

# 4. 创建gadget实例
mkdir -p g1
cd g1

# 5. 配置基础USB设备信息
echo 0x1d6b > idVendor
echo 0x0104 > idProduct
echo 0x0100 > bcdDevice
echo 0x0200 > bcdUSB

# 6. 配置设备字符串描述符
mkdir -p strings/0x409
echo "fedcba9876543210" > strings/0x409/serialnumber
echo "Pi Dev" > strings/0x409/manufacturer
echo "Pi HID Keyboard" > strings/0x409/product

# 7. 创建配置
mkdir -p configs/c.1/strings/0x409
echo "HID Config" > configs/c.1/strings/0x409/configuration
echo 500 > configs/c.1/MaxPower

# 8. 创建HID键盘功能实例
mkdir -p functions/hid.usb0
echo 1 > functions/hid.usb0/protocol
echo 1 > functions/hid.usb0/subclass
echo 8 > functions/hid.usb0/report_length
echo -ne '\x05\x01\x09\x06\xa1\x01\x05\x07\x19\xe0\x29\xe7\x15\x00\x25\x01\x75\x01\x95\x08\x81\x02\x95\x01\x75\x08\x81\x03\x95\x05\x75\x01\x05\x08\x19\x01\x29\x05\x91\x02\x95\x01\x75\x03\x91\x03\x95\x06\x75\x08\x15\x00\x25\x65\x05\x07\x19\x00\x29\x65\x81\x00\xc0' > functions/hid.usb0/report_desc

# 9. 关联HID功能与配置
ln -s functions/hid.usb0 configs/c.1/

# 10. 绑定UDC，启用设备（自动识别本机UDC）
sleep 0.2
UDC_NAME=$(ls /sys/class/udc | head -1)
echo ${UDC_NAME} > UDC

# 写入日志，方便排查开机执行情况
echo "[$(date)] HID Gadget 启动成功，UDC: ${UDC_NAME}" >> /var/log/hid-gadget.log
```

当你向 `/dev/hidg0` 写入数据时，必须遵循这个格式（对应 `report_length=8`）：

| 字节 0 | 字节 1    | 字节 2 | 字节 3 | 字节 4 | 字节 5 | 字节 6 | 字节 7 |
| ------ | --------- | ------ | ------ | ------ | ------ | ------ | ------ |
| 修饰键 | 保留（0） | 按键 1 | 按键 2 | 按键 3 | 按键 4 | 按键 5 | 按键 6 |

- **修饰键（字节 0）**：Ctrl (0x01)、Shift (0x02)、Alt (0x04)、Win (0x08) 等的组合，比如同时按 Ctrl+Shift 就是 `0x03`。
- **按键 1-6**：最多同时按下 6 个普通键，每个键对应一个 **KeyCode**（比如 `t` 是 `0x17`）。
- **全 0 报文**：`\x00\x00\x00\x00\x00\x00\x00\x00`，代表 “所有按键松开”，必须在按下按键后发送，否则会变成 “一直按住这个键”。

### 自动启动

树莓派最新的 Bookworm 系统默认弱化了 rc.local，推荐使用 systemd 服务管理开机任务，稳定性、可排查性更强。

1. 创建 systemd 服务文件：

```bash
sudo vim /etc/systemd/system/hid-gadget.service
```

```bash
[Unit]
Description=USB HID Gadget Keyboard Service
After=sys-kernel-config.mount multi-user.target # 等待configfs挂载完成再执行
Requires=sys-kernel-config.mount

[Service]
Type=oneshot # 一次性执行脚本，执行完成就退出
ExecStart=/usr/local/bin/enable-hid-gadget.sh
RemainAfterExit=yes # 执行完成后仍标记服务为活跃状态
StandardOutput=journal+console

[Install]
WantedBy=multi-user.target
```

2, 开启服务

```bash
# 重载服务配置
sudo systemctl daemon-reload
# 设置开机自动启动
sudo systemctl enable hid-gadget.service
# 手动启动测试，确认无报错
sudo systemctl start hid-gadget.service
# 查看服务状态，确认启动成功
sudo systemctl status hid-gadget.service
```

> 关闭服务
>
> ```c
> # 关闭开机自启
> sudo systemctl disable hid-gadget.service
> # 停止当前运行的服务（解绑设备）
> sudo echo "" > /sys/kernel/config/usb_gadget/g1/UDC
> # 删除服务文件和脚本
> sudo rm /etc/systemd/system/hid-gadget.service
> sudo rm /usr/local/bin/enable-hid-gadget.sh
> # 重载systemd配置
> sudo systemctl daemon-reload
> ```

## 自动开启网页

`vim ./open-pi-18800.sh`

```bash
#!/bin/bash
set -e

# ==================== 自定义配置区（无特殊情况无需修改） ====================
# 目标端口，固定为18800
TARGET_PORT=18800
# HID设备路径，默认/dev/hidg0
HID_DEV="/dev/hidg0"
# 按键输入延迟（单位：秒），老电脑反应慢可调大
KEY_DELAY=0.08
# 窗口/浏览器启动等待延迟
WAIT_DELAY=1.2

# ==================== 前置检查 ====================
# 检查是否为root权限
if [ "$(id -u)" -ne 0 ]; then
    echo "错误：必须使用sudo/root权限执行脚本！"
    exit 1
fi

# 检查HID设备是否存在
if [ ! -e "$HID_DEV" ]; then
    echo "错误：找不到HID设备 $HID_DEV，请先执行Gadget配置脚本！"
    exit 1
fi

# 自动获取树莓派的局域网IP（取第一个有效非回环IP）
BOARD_IP=$(hostname -I | awk '{print $1}')

# 检查IP获取是否成功
if [ -z "$BOARD_IP" ]; then
    echo "错误：无法获取树莓派IP地址，请检查网络连接！"
    exit 1
fi

# 拼接完整目标网址（必须带http://前缀，否则Windows无法识别）
TARGET_URL="http://$BOARD_IP:$TARGET_PORT"
echo "成功获取树莓派IP：$BOARD_IP"
echo "目标网址：$TARGET_URL"

# ==================== 核心按键函数 ====================
# 发送单个按键：参数1=修饰键，参数2=按键键码
send_key() {
    local modifier=$1
    local keycode=$2
    # 按下按键
    echo -ne "$(printf '\\x%02x' $modifier)\x00$(printf '\\x%02x' $keycode)\x00\x00\x00\x00\x00" > $HID_DEV
    sleep $KEY_DELAY
    # 释放所有按键（必须执行，否则会一直按住按键）
    echo -ne "\x00\x00\x00\x00\x00\x00\x00\x00" > $HID_DEV
    sleep $KEY_DELAY
}

# 输入单个字符（自动处理大小写、网址常用符号）
send_char() {
    local char=$1
    local modifier=0x00
    local keycode=0x00

    case $char in
        # 小写字母
        a) keycode=0x04 ;; b) keycode=0x05 ;; c) keycode=0x06 ;; d) keycode=0x07 ;;
        e) keycode=0x08 ;; f) keycode=0x09 ;; g) keycode=0x0a ;; h) keycode=0x0b ;;
        i) keycode=0x0c ;; j) keycode=0x0d ;; k) keycode=0x0e ;; l) keycode=0x0f ;;
        m) keycode=0x10 ;; n) keycode=0x11 ;; o) keycode=0x12 ;; p) keycode=0x13 ;;
        q) keycode=0x14 ;; r) keycode=0x15 ;; s) keycode=0x16 ;; t) keycode=0x17 ;;
        u) keycode=0x18 ;; v) keycode=0x19 ;; w) keycode=0x1a ;; x) keycode=0x1b ;;
        y) keycode=0x1c ;; z) keycode=0x1d ;;
        # 大写字母（按住Shift）
        A) modifier=0x02; keycode=0x04 ;; B) modifier=0x02; keycode=0x05 ;;
        C) modifier=0x02; keycode=0x06 ;; D) modifier=0x02; keycode=0x07 ;;
        E) modifier=0x02; keycode=0x08 ;; F) modifier=0x02; keycode=0x09 ;;
        G) modifier=0x02; keycode=0x0a ;; H) modifier=0x02; keycode=0x0b ;;
        I) modifier=0x02; keycode=0x0c ;; J) modifier=0x02; keycode=0x0d ;;
        K) modifier=0x02; keycode=0x0e ;; L) modifier=0x02; keycode=0x0f ;;
        M) modifier=0x02; keycode=0x10 ;; N) modifier=0x02; keycode=0x11 ;;
        O) modifier=0x02; keycode=0x12 ;; P) modifier=0x02; keycode=0x13 ;;
        Q) modifier=0x02; keycode=0x14 ;; R) modifier=0x02; keycode=0x15 ;;
        S) modifier=0x02; keycode=0x16 ;; T) modifier=0x02; keycode=0x17 ;;
        U) modifier=0x02; keycode=0x18 ;; V) modifier=0x02; keycode=0x19 ;;
        W) modifier=0x02; keycode=0x1a ;; X) modifier=0x02; keycode=0x1b ;;
        Y) modifier=0x02; keycode=0x1c ;; Z) modifier=0x02; keycode=0x1d ;;
        # 数字
        0) keycode=0x27 ;; 1) keycode=0x1e ;; 2) keycode=0x1f ;; 3) keycode=0x20 ;;
        4) keycode=0x21 ;; 5) keycode=0x22 ;; 6) keycode=0x23 ;; 7) keycode=0x24 ;;
        8) keycode=0x25 ;; 9) keycode=0x26 ;;
        # 网址常用符号
        '.') keycode=0x37 ;; '/') keycode=0x38 ;; ':') modifier=0x02; keycode=0x33 ;;
        '-') keycode=0x2d ;; '_') modifier=0x02; keycode=0x2d ;; '?') modifier=0x02; keycode=0x38 ;;
        '=') keycode=0x2e ;; '&') modifier=0x02; keycode=0x24 ;; '%') modifier=0x02; keycode=0x22 ;;
        # 空格
        ' ') keycode=0x2c ;;
        # 回车
        $'\n') keycode=0x28 ;;
        *) echo "跳过不支持的字符: $char"; return ;;
    esac

    send_key $modifier $keycode
}

# 输入完整字符串
send_string() {
    local text=$1
    for (( i=0; i<${#text}; i++ )); do
        send_char "${text:$i:1}"
    done
}

# ==================== 主执行流程 ====================
echo "3秒后开始执行，请确保Windows主机处于桌面未锁屏状态..."
sleep 3

echo "1. 按下Win+R打开系统运行框"
send_key 0x08 0x15  # Win键(0x08) + R键(0x15)
sleep $WAIT_DELAY

echo "2. 输入目标网址"
send_string "$TARGET_URL"
sleep $KEY_DELAY

echo "3. 按下回车，调用默认浏览器打开网站"
send_key 0x00 0x28  # 回车
send_key 0x00 0x28  # 回车
echo "执行完成！已成功打开树莓派18800端口网站"
```

### 自动启动

/usr/local/bin/auto-start-after-network.sh

```bash
#!/bin/bash
set -e

# ==================== 请修改这里的路径为你的实际路径 ====================
# picoclaw-launcher 的完整绝对路径
PICOCLAW_PATH="/home/jiao/picoclaw-launcher"
# picoclaw-launcher 的工作目录（如果需要）
PICOCLAW_WORK_DIR="/home/jiao/.picoclaw/workspace"
# HID打开网站脚本的完整绝对路径
HID_SCRIPT_PATH="/home/jiao/open-pi-18800.sh"
# 日志文件路径
LOG_FILE="/var/log/auto-start-after-network.log"
# 重试间隔（单位：秒）
RETRY_INTERVAL=5

# ==================== 前置检查 ====================
# 创建日志目录（如果不存在）
mkdir -p "$(dirname "$LOG_FILE")"
# 重定向所有输出到日志文件
exec > >(tee -a "$LOG_FILE") 2>&1

echo "=========================================="
echo "[$(date)] 开始执行自动启动任务，正在等待网络连接..."

# ==================== 1. 循环等待网络稳定并获取IP ====================
BOARD_IP=""
RETRY_COUNT=0

while [ -z "$BOARD_IP" ]; do
    BOARD_IP=$(hostname -I | awk '{print $1}')
    
    if [ -z "$BOARD_IP" ]; then
        RETRY_COUNT=$((RETRY_COUNT + 1))
        echo "[$(date)] 网络未就绪，未获取到IP，${RETRY_INTERVAL}秒后重试... (重试次数: ${RETRY_COUNT})"
        sleep $RETRY_INTERVAL
    else
        echo "[$(date)] 网络连接成功！树莓派IP: $BOARD_IP"
        break
    fi
done

# ==================== 2. 启动 picoclaw-launcher ====================
echo "[$(date)] 正在清理 18800 端口..."
# 使用 ss 命令强制杀掉所有占用18800端口的进程（兼容无 lsof 的系统）
ss -tulpn | grep ':18800' | awk '{print $6}' | cut -d',' -f2 | cut -d'=' -f2 | xargs -r sudo kill -9 2>/dev/null || true
sleep 2

echo "[$(date)] 正在启动 picoclaw-launcher..."
if [ ! -f "$PICOCLAW_PATH" ]; then
    echo "[$(date)] 错误：找不到 picoclaw-launcher 文件：$PICOCLAW_PATH"
    exit 1
fi

cd "$PICOCLAW_WORK_DIR" || exit 1
nohup "$PICOCLAW_PATH" -public >> "$LOG_FILE" 2>&1 &
PICOCLAW_PID=$!
echo "[$(date)] picoclaw-launcher 已启动，PID: $PICOCLAW_PID"

# 给picoclaw一点启动时间
sleep 5

# 验证picoclaw是否真的启动成功了
if ! ss -tulpn | grep ':18800' > /dev/null; then
    echo "[$(date)] 警告：picoclaw-launcher 可能启动失败，18800端口未监听！"
else
    echo "[$(date)] 确认：picoclaw-launcher 已成功监听 18800 端口"
fi

# ==================== 3. 【新增】循环等待 /dev/hidg0 设备初始化 ====================
echo "[$(date)] 正在等待 HID 设备 /dev/hidg0 初始化..."
HID_RETRY_COUNT=0
HID_MAX_RETRY=30 # 最多等待30秒

while [ ! -e "/dev/hidg0" ]; do
    HID_RETRY_COUNT=$((HID_RETRY_COUNT + 1))
    
    if [ $HID_RETRY_COUNT -ge $HID_MAX_RETRY ]; then
        echo "[$(date)] 错误：等待超过${HID_MAX_RETRY}秒，/dev/hidg0 仍未出现，跳过HID脚本执行"
        # 注意：这里可以选择 exit 1 直接报错，或者选择 continue 跳过HID只启动服务
        # 这里我们选择跳过HID，保证picoclaw能启动
        break
    fi
    
    echo "[$(date)] /dev/hidg0 未就绪，0.5秒后重试... (${HID_RETRY_COUNT}/${HID_MAX_RETRY})"
    sleep 0.5
done

if [ -e "/dev/hidg0" ]; then
    echo "[$(date)] 确认：/dev/hidg0 设备已就绪"
    
    # ==================== 4. 执行HID脚本打开网站 ====================
    echo "[$(date)] 正在执行HID脚本打开网站..."
    if [ ! -f "$HID_SCRIPT_PATH" ]; then
        echo "[$(date)] 错误：找不到HID脚本：$HID_SCRIPT_PATH"
    else
        "$HID_SCRIPT_PATH"
    fi
else
    echo "[$(date)] 跳过HID脚本执行"
fi

echo "[$(date)] 所有自动启动任务执行完成"
echo "=========================================="
```



创建文件/etc/systemd/system/auto-start-after-network.service

```txt
[Unit]
Description=Auto Start Picoclaw and HID Script
After=multi-user.target hid-gadget.service
Wants=hid-gadget.service

[Service]
Type=oneshot
ExecStart=/usr/local/bin/auto-start-after-network.sh
RemainAfterExit=yes
TimeoutStartSec=0

[Install]
WantedBy=multi-user.target
```



```bash
sudo systemctl daemon-reload
sudo systemctl enable auto-start-after-network.service
# 测试
sudo systemctl start auto-start-after-network.service
# 查看日志
sudo systemctl status auto-start-after-network.service
```

### 停止服务

```bash
# 停止当前运行的服务
sudo systemctl stop auto-start-after-network.service

# 禁止开机自启
sudo systemctl disable auto-start-after-network.service

# 如果想彻底删除服务
sudo rm /etc/systemd/system/auto-start-after-network.service
sudo rm /usr/local/bin/auto-start-after-network.sh
sudo systemctl daemon-reload
```
