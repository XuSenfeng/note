# 电源管理

一个真实生活中的设备驱动除了要处理设备的基本功能以外，还需要处理电源管理，主要提供挂起和恢复用的suspend()、resume()两个函数

对于platform_driver而言，该结构体已经包含

```c
struct platform_driver {
	int (*probe)(struct platform_device *);
	int (*remove)(struct platform_device *);
	void (*shutdown)(struct platform_device *);
	int (*suspend)(struct platform_device *, pm_message_t state);
	int (*resume)(struct platform_device *);
	struct device_driver driver;
	const struct platform_device_id *id_table;
	bool prevent_deferred_probe;

	ANDROID_KABI_RESERVE(1);
};
```

在实际使用的时候可以处理一下内核是否启用电源管理宏定义

```c
#ifdef CONFIG_PM
static int xxx_suspend(struct platform_device *pdev, pm_message_t state)
{
    ...
    return 0;
}

static int xxx_resume(struct platform_device *pdev)
{
    ...
    return 0;
}
#else
#define xxx_suspend NULL
#define xxx_resume  NULL
#endif
```

通常而言，在suspend()函数里面会停止设备，并关闭给它提供的时钟`clk_disable(xxx->clk); `
