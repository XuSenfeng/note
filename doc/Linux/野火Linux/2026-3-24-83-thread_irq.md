# thread_irq

**thread_irq = 把中断处理程序，放进一个独立内核线程里运行。**

它解决了 **tasklet / 软中断** 最大的痛点：

**不能休眠、不能阻塞、不能做耗时操作**。

```c
// 申请一个线程化中断
request_threaded_irq(irq, handler, thread_fn, flags, name, dev_id);
```

+ **handler**：上半部（硬中断），快处理，不能休眠, 可以设置为NULL, 表示不使用(需要设置标志位), 返回`IRQ_WAKE_THREAD`触发线程处理, 返回`IRQ_HANDLED`不触发
+ **thread_fn**：下半部（内核线程），真正干活，可以休眠
+ 标志位, IRQF_ONESHOT, 使用默认的顶部中断

```c
static irqreturn_t irq_default_primary_handler(int irq, void *dev_id)
{
	return IRQ_WAKE_THREAD;
}
```
