# DMA

DMA 是一种无需CPU 的参与就可以让外设与系统内存之间进行双向数据传输的硬件机制。使用DMA 可以使系统CPU 从实际的I/O 数据传输过程中摆脱出来，从而大大提高系统的吞吐率

DMA 方式的数据传输由DMA 控制器（DMAC）控制，在传输期间，CPU 可以并发地执行其他任务。当DMA 结束后，DMAC 通过中断通知CPU 数据传输已经结束，然后由CPU 执行相应的中断服务程序进行后处理

DMA 映射机制的基本原理，是把硬件设备的物理内存地址巧妙地映射到 CPU 能够访问的虚拟地址空间里 。这就如同给设备与 CPU 之间搭建了一条高效的 “沟通桥梁”，让设备能直接对系统内存进行数据读写，无需 CPU 在中间 “传话”，从而极大地提升了数据传输的效率

## 出现问题

DMA的访问的地址可能会和Cache重叠, 导致DMA更新的地址但是CPU不知道

解决由于DMA 导致的Cache 一致性问题的最简单方法是直接禁止DMA 目标地址范围内内存的Cache 功能

## DMA编程

DMA 本身不属于一种等同于字符设备、块设备和网络设备的外设，它只是外设与内存交互数据的一种方式

在使用kmalloc()和`__get_free_pages()`及其类似函数申请DMA 缓冲区时应使用GFP_DMA 标志，这样能保证获得的内存位于DMA_ZONE，是具备DMA 能力的

### 三种地址

| 地址类型             | 用途         | 类型定义      | 说明                                                         |
| -------------------- | ------------ | ------------- | ------------------------------------------------------------ |
| 虚拟地址             | CPU 使用     | `void *`      | 由`kmalloc()`、`vmalloc()`等返回，通过页表转换为物理地址     |
| CPU 物理地址         | 内核管理内存 | `phys_addr_t` | 系统内存的物理地址，位于`/proc/iomem`中                      |
| 总线地址（DMA 地址） | 外设使用     | `dma_addr_t`  | 从设备角度看到的地址，可能与 CPU 物理地址不同（通过 IOMMU 转换） |

```c
enum dma_data_direction {
    DMA_BIDIRECTIONAL = 0,  // 双向传输（性能损耗较大）
    DMA_TO_DEVICE = 1,      // 内存 -> 外设（发送数据）
    DMA_FROM_DEVICE = 2,    // 外设 -> 内存（接收数据）
    DMA_NONE = 3            // 未使用（调试用）
};
```

## 实际使用

Linux 提供两种主要的 DMA 映射方式：一致性映射和流式映射

### 一致性 DMA 映射

一致性映射保证 CPU 和设备对内存的访问始终保持一致，无需手动进行缓存同步。

#### 适用场景

- 长寿命的缓冲区（驱动生命周期内持续存在）
- 设备控制结构（如网卡的描述符环形缓冲区）
- CPU 与设备需要频繁双向更新数据的场景

```c
// 分配一致性DMA缓冲区
void *dma_alloc_coherent(struct device *dev, size_t size,
                         dma_addr_t *dma_handle, gfp_t gfp);

// 释放一致性DMA缓冲区
void dma_free_coherent(struct device *dev, size_t size,
                       void *cpu_addr, dma_addr_t dma_handle);
```

### 流式传输

流式映射用于一次性或临时性的数据传输，在传输前映射，传输后立即解除映射。

#### 适用场景

- 短寿命的数据传输（如网络数据包、磁盘块）
- 使用已有的内核缓冲区（如`kmalloc`分配的内存）
- 高频率、大数据量的传输

```c
// 映射单个内存区域
dma_addr_t dma_map_single(struct device *dev, void *cpu_addr,
                          size_t size, enum dma_data_direction dir);

// 检查映射是否成功
int dma_mapping_error(struct device *dev, dma_addr_t dma_addr);

// 取消单个内存区域的映射
void dma_unmap_single(struct device *dev, dma_addr_t dma_addr,
                      size_t size, enum dma_data_direction dir);

// 同步单个缓冲区（CPU拥有权 -> 设备拥有权）
void dma_sync_single_for_device(struct device *dev, dma_addr_t dma_addr,
                                size_t size, enum dma_data_direction dir);

// 同步单个缓冲区（设备拥有权 -> CPU拥有权）
void dma_sync_single_for_cpu(struct device *dev, dma_addr_t dma_addr,
                             size_t size, enum dma_data_direction dir);
```

#### 使用示例

```c
// 分配普通内核缓冲区
void *cpu_addr = kmalloc(4096, GFP_KERNEL);
if (!cpu_addr)
    return -ENOMEM;

// 准备数据
memcpy(cpu_addr, data, 4096);

// 映射缓冲区供DMA使用
dma_addr_t dma_addr = dma_map_single(dev, cpu_addr, 4096, DMA_TO_DEVICE);
if (dma_mapping_error(dev, dma_addr)) {
    kfree(cpu_addr);
    return -ENOMEM;
}

// 启动DMA传输
start_dma_transfer(dma_addr, 4096);

// 等待传输完成（通过中断或轮询）
wait_for_dma_completion();

// 取消映射
dma_unmap_single(dev, dma_addr, 4096, DMA_TO_DEVICE);

// 现在CPU可以安全访问缓冲区
kfree(cpu_addr);
```

### 分散 - 聚集 DMA

当需要传输的数据在虚拟内存中连续但物理内存不连续时，使用分散 - 聚集 DMA 可以避免内存拷贝

```c
// 映射分散-聚集列表
int dma_map_sg(struct device *dev, struct scatterlist *sg,
               int nents, enum dma_data_direction dir);

// 取消分散-聚集列表的映射
void dma_unmap_sg(struct device *dev, struct scatterlist *sg,
                  int nents, enum dma_data_direction dir);
```
