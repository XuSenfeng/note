# 编译

下载最基础的文件项目

```bash
git clone https://git.openwrt.org/openwrt/openwrt.git -b openwrt-23.05
cd openwrt
```

安装需要的工具

```bash
sudo apt update
sudo apt install -y \
  build-essential clang flex bison g++ gawk \
  gcc-multilib g++-multilib gettext git libncurses-dev \
  libssl-dev python3-distutils rsync swig unzip zlib1g-dev \
  file wget curl libelf-dev libfuse-dev \
  python3 python3-pip python3-setuptools \
  subversion mercurial bzr cvs \
  zstd libzstd-dev
```

Feeds 是 OpenWRT 的软件包仓库，包含数千个可选软件包。

默认的feeds如下

+   `feeds.conf.default`（默认）：官方提供的标准 feeds 配置，包含核心扩展源
+   `feeds.conf`（自定义）：用户创建后会覆盖默认配置，用于添加自定义源

```bash
(base) jiao@ubuntu22:~/openWRT/openwrt$ cat feeds.conf.default
src-git packages https://git.openwrt.org/feed/packages.git^e59d9ef823bcb581e3939789b4eaeaf900b79759
src-git luci https://git.openwrt.org/project/luci.git^7ce34fe1a53db10bb9dd0223467f5bb71a29a659
src-git routing https://git.openwrt.org/feed/routing.git^4a06b031dc16628c9b8351ac297ecdbc92695dc2
src-git telephony https://git.openwrt.org/feed/telephony.git^98c8a5aa4624312ed758e2e2b6d4039050a1649d
```

>   `packages`：基础路由器特性软件包（如常用工具、网络应用）
>
>   `luci`：OpenWRT 默认 Web 管理界面
>
>   `routing`：路由相关协议软件（如 OSPF、BGP）
>
>   `telephony`：IP 电话相关软件（如 Asterisk）
>
>   `management`：设备管理软件（如 TR069）

更新一下

```bash
cp feeds.conf.default feeds.conf

# 更新 feeds（从远程获取最新包列表）, 存放下载的软件包定义
# ./scripts/feeds：核心管理工具，负责更新、安装、搜索、卸载等操作
./scripts/feeds update -a

# 安装所有 feed 包到编译系统（使其可被 menuconfig 选中）
./scripts/feeds install -a
```

>   | 命令        | 功能                  | 示例                                    |
>   | ----------- | --------------------- | --------------------------------------- |
>   | `list`      | 列出所有可用 feeds    | `./scripts/feeds list`                  |
>   | `search`    | 搜索软件包            | `./scripts/feeds search vim`            |
>   | `uninstall` | 卸载已安装软件包      | `./scripts/feeds uninstall vim`         |
>   | `clean`     | 清除下载的 feeds 数据 | `./scripts/feeds clean`                 |
>   | `info`      | 查看软件包详细信息    | `./scripts/feeds info luci-app-openvpn` |

+   配置目标平台

**进入配置界面**

```bash
make menuconfig
```

这会打开一个基于 ncurses 的交互式配置菜单。

**选择树莓派** 4B 平台

在 menuconfig 中按以下路径选择：

```
Target System  --->
  ( ) Broadcom BCM27xx        ← 选择此项，按空格或回车确认
  
Subtarget  --->
  ( ) BCM2711 boards (64 bit) ← 选择此项（树莓派4B为64位）

Target Profile  --->
  ( ) Raspberry Pi 4B         ← 选择此项
```

> 📌 **注意**：如果你需要 32 位系统，选 `BCM2710 boards (32 bit)`，但 64 位性能更好，推荐优先选择 64 位。

+   目标镜像格式配置

```
Target Images  --->
  [*] ext4  ← 选择 ext4 根文件系统（支持在线扩容）
  [*] squashfs  ← 选择 squashfs（只读压缩，支持恢复出厂）
  (256) Root filesystem partition size (in MB)  ← 调大根分区
```

推荐同时选择 `ext4` 和 `squashfs`，以获得最大兼容性。

+ 内核配置

```
Kernel modules  --->
  Network Devices  --->
    <*> kmod-r8169       ← 如使用 USB RTL8169 网卡
    <*> kmod-usb-net-rtl8152  ← RTL8152 USB 网卡
  USB Support  --->
    <*> kmod-usb3        ← USB 3.0 支持
    <*> kmod-usb-storage ← USB 存储支持
  Filesystems  --->
    <*> kmod-fs-ext4     ← ext4 文件系统
    <*> kmod-fs-vfat     ← FAT32 支持（SD 卡启动分区）
```

全局构建设置

```
Global build settings  --->
  [ ] Compile with full language support  ← 可减小固件体积
```

> 💡 **关于 IPv6**：在 OpenWrt 22.03+ 和 23.05 版本中，IPv6 支持已默认集成，不再需要在 `Global build settings` 中单独开启。如需使用 IPv6 相关功能，直接在 `Network` 菜单中选择对应的软件包即可（如 `odhcp6c`、`ip6tables` 等）。

编译+ 时必选（基础配置）

```
Base system  --->
  <*> dnsmasq          ← DNS/DHCP 服务
  <*> firewall4        ← nftables 防火墙（23.05+）
  <*> opkg             ← 包管理器（必须！）

LuCI  --->
  Collections  --->
    <*> luci           ← Web 管理界面（会自动拉入依赖）

Kernel modules  --->
  Wireless Drivers  --->
    <*> kmod-brcmfmac  ← 博通 WiFi 驱动（树莓派 4B 使用）
```

> 💡 在 menuconfig 中按 `/` 键可搜索任意软件包的位置。

+   检查配置

```bash
# 再次确认 .config 文件已正确生成
grep "CONFIG_TARGET_bcm27xx" .config
grep "CONFIG_TARGET_bcm27xx_bcm2711" .config

# 预期输出：
# CONFIG_TARGET_bcm27xx=y
# CONFIG_TARGET_bcm27xx_bcm2711=y
```

+ 下载软件包源码

```bash
# 预先下载所有依赖源码，避免编译中途因网络中断失败
make download -j$(nproc)

# 如遇下载失败，可多次运行此命令
# 或单线程下载（更稳定）
make download -j1 V=s
```

+   首次完整编译

```bash
# 使用所有可用核心编译（推荐）
make -j$(nproc)

# 或指定核心数（留出 1~2 个核心给系统）
make -j$(($(nproc) - 1))

# 单线程编译（调试时使用，可看到详细输出）
make -j1 V=s
```

编译以后输出的位置是`ls -lh bin/targets/bcm27xx/bcm2711/`

主要输出文件：

```
bin/targets/bcm27xx/bcm2711/
├── openwrt-bcm27xx-bcm2711-rpi-4-ext4-factory.img.gz    ← 全新刷写镜像（ext4）
├── openwrt-bcm27xx-bcm2711-rpi-4-ext4-sysupgrade.img.gz ← 在线升级镜像（ext4）
├── openwrt-bcm27xx-bcm2711-rpi-4-squashfs-factory.img.gz
├── openwrt-bcm27xx-bcm2711-rpi-4-squashfs-sysupgrade.img.gz
├── packages/                                              ← 编译生成的 .ipk 包
└── config.buildinfo                                       ← 本次编译配置记录
```

| 文件类型            | 用途                       |
| ------------------- | -------------------------- |
| `factory.img.gz`    | 首次刷写（写入 SD 卡）     |
| `sysupgrade.img.gz` | 已有 OpenWRT 在线升级      |
| `ext4` 版本         | 支持在线扩展根分区         |
| `squashfs` 版本     | 支持恢复出厂设置，体积更小 |
