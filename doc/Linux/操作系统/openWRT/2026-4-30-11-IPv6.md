# IPv6

## 基础概念

IPv6可以实现为所有的设备分配公网地址, 在使用IPv4的时候, 需要使用NAT进行IP地址的转换, 这个时候公网访问内网的话, 需要做端口转发, UDP打洞等, 而且是路由器有公网IP的时候, 所以需要使用一个服务器做内网穿透

IPv6取消了广播机制，用组播来代替IPv4的广播，可以减少广播报文对全网的性能消耗, 并且本身就具备自动配置地址的能力, 不需要DHCP

### 显示逻辑

一共八段, 每段4个16进制数据, 一共128bit, 使用`:`进行分割, 如果一个段都是0, 可以使用一个0进行表示, 多个段都是0的时候, 相邻的两个可以使用`::`进行表示(一个地址里面只可以有一个), 每组中的前导“0”都可以省略

在使用这个访问网页的时候, 必须使用中括号进行包裹, 用于区分端口号

IPv6的掩码使用“前缀长度”来表示, `FC00:0:130F::9C0:876A:130B/64`, 代表这个地址的前缀长度是64，也就是掩码长度为64位

一般来说: 接口的长度是64位, 前面的分为前缀以及子网, 但是实际使用的时候接口边不是这个规范

+ 前缀（运营商 / 上级给的大网段）

就是运营商分给你家 / 公司的**公网基础前缀**

常见：`/48`、`/56`、`/32`, 相当于 IPv4 运营商给你的主网段

+ 子网（你自己划分的小网段）

在前缀之后、接口 ID 之前，**拿几位出来自己划分子网**

用来分：客厅、卧室、IoT 设备、办公 VLAN 等

+ 接口 ID（后 64 位）

作用：**标识一台设备的网卡**

对标 IPv4：**IP 的最后主机位**

## 特殊地址

+ `2000::/3`: 公网地址
+ `FC00::/7`: 私有地址
+ + `FC00::/8`：官方预留，**不允许私自使用**
    + `FD00::/8`：**实际内网用的唯一本地地址**

> IPv6 ULA：IPv6 地址无限多**根本不缺地址**，设计目的是：**内网隔离、安全隔离、设备内网互访、临时组网**，运营商公网不路由这个地址，不会暴露到互联网

+ `FE80::/10`: 只要开启 IPv6，**所有网卡自动生成**，不用手动配置、不用路由器；**仅同一局域网内生效**
+ `::1/128`: 回环地址
+ `::`: 未指定地址, 类似于`0.0.0.0`
+ `FF00::/8`: 对标 IPv4：**224.0.0.0/4 组播地址**, IPv6 **根本没有广播**

> 服务器**只发 1 份数据**，网络设备自动只转发给**主动加入这个组的设备**，其他人收不到, 想接收数据的设备，**主动申请加入某个组播 IP 群组**
>
> 比如可以使用`ip maddr add 239.1.1.1 dev eth0`进行加入

+ 任播（Anycast）是 IPv6 核心地址类型之一，实现 **“一对一组中最近一个”** 的通信模式 —— 多个设备共享同一地址，数据包会被路由到网络中 “最近” 的那个设备，完美解决负载均衡和服务冗余问题

### 常见的组播

FF02:0:0:0:0:0:0:1，所有节点的组播地址。

FF02:0:0:0:0:0:0:2，所有路由器的组播地址。

FF02:0:0:0:0:1:FFXX:XXXX ，Solicited-Node组播地址。

FF02:0:0:0:0:0:0:5，所有OSPF路由器组播地址。

FF02:0:0:0:0:0:0:6，所有OSPF的DR路由器组播地址。

FF02:0:0:0:0:0:0:D，所有PIM路由器组播地址。

## 报文头部

![img](https://picture-01-1316374204.cos.ap-beijing.myqcloud.com/lenovo-picture/202604302313499.png)

和IPv4头部相比，IPv6头部去除了IHL、identifiers、Flags、Fragment Offset、Header Checksum、 Options、Padding域，只增了流标签域，因此IPv6报文头的处理较IPv4大大简化，提高了处理效率

提出了扩展头部的概念，可以按需对头部字段进行扩展，可以实现所需的功能

### 工作流程

openwrt可以通过拨号获取一个IPv6地址以及一个IPv6的PD, 这是一个前缀委托, 运营商不知道内网的结构, 所以直接给你一个60位的前缀, 路由器分配其他设备

一个网卡默认会给自己生成以及fe80开头的inklocal的内网地址, 通过组播发送一个Router Solicitation请求数据包, 路由器开启LAN 的RA服务以后, 会返回一个RA路由通告

电脑使用这个前缀以及里面的A标志1的时候生成地址, 这个A就是openWRT里面的SLAAC(无状态自动配置)

> 如果使用eui64, 默认是使用MAC地址里面加入一个`FF:Fe`填充, 第七个bit取反, 这个会导致隐私问题
>
> 可以在LAN里面配置
>
> ![image-20260501134002847](https://picture-01-1316374204.cos.ap-beijing.myqcloud.com/lenovo-picture/202605011340917.png)

默认的时候会生成两个IPv6, 其中一个是临时的地址, 用于访问网络, 一定时间以后自动更新, 还有一个对外进行访问

![image-20260501134216922](https://picture-01-1316374204.cos.ap-beijing.myqcloud.com/lenovo-picture/202605011342974.png)

### 生命周期

默认获取的IPv6使用一个生命周期的

+ Tentativw: 拼接以后, DAD重复地址检测
+ Preferred: 正常使用
+ Deprecated: 一定时间以后不可以创建新的连接
+ Invalid: 彻底失效

> 可以使用命令`netsh interface ipv6 show address`

![image-20260501134726656](https://picture-01-1316374204.cos.ap-beijing.myqcloud.com/lenovo-picture/202605011347707.png)

这个前缀不是固定的, 是会变化的, 所以需要定时更新, 路由器定时发送RA通告, 更新周期
