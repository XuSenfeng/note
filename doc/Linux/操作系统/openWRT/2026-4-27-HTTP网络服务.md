# HTTP网络服务

默认可以使用uhttpd进行HTTP的文件的分享

```bash
opkg install uhttpd
```

uHTTPd 的配置通过 OpenWrt 的标准 uci 系统进行。配置文件是 `/etc/config/uhttpd`

```bash
mkdir /www/test
echo "Hello world" >> /www/test/message.txt
```

之后可以在`https://192.168.1.1/test/`查看到对应的文件

## 网页界面

**LuCI**：OpenWrt 路由器的**网页管理后台**（就是你输 192.168.1.1 打开的设置页面）

**uHTTPd**：路由器上运行 LuCI 网页的轻量服务器

**痛点**：

-   默认 LuCI 用 **HTTP 明文**，账号密码容易被窃听；
-   装 HTTPS 加密需要 TLS 库，**小存储路由器没空间安装**；
-   默认 LuCI 对整个局域网开放，容易被暴力破解密码。

### SSh隧道

可以使用SSH隧道加密的协议, 提高安全性

首先在路由器里面配置只监听本地的的连接

```bash
uci -q delete uhttpd.main.listen_http
uci add_list uhttpd.main.listen_http="127.0.0.1:80"
uci add_list uhttpd.main.listen_http="[::1]:80"
uci -q delete uhttpd.main.listen_https
uci add_list uhttpd.main.listen_https="127.0.0.1:443"
uci add_list uhttpd.main.listen_https="[::1]:443"
uci commit uhttpd
/etc/init.d/uhttpd restart
```

路由器的 LuCI 后台，**只允许自己访问自己**，局域网里的电脑 / 手机**直接打不开管理页面**

通过命令, 访问对方内网、开发联调、内网服务本地使用

```bash
ssh -L 本地端口:目标IP:目标端口 服务器账号@服务器
# openwrt.lan = 你自己局域网里 OpenWrt 路由器的内网主机别名
# OpenWrt 系统默认主机名（设备名字）
# .lan局域网专用私有域名后缀，只能在你家 / 本地局域网生效，外网全网打不开
ssh -L127.0.0.1:8000:127.0.0.1:80 root@openwrt.lan
```

之后可以直接在`http://127.0.0.1:8000`打开服务

>   `-R`的含义是反过来的, `ssh -R 服务器端口:本地IP:本地端口 服务器账号@服务器`
>
>   你家里电脑没有公网 IP，外网完全访问不到。用 `-R` 把你本地服务，映射到云服务器端口，别人访问云服务器端口，就会穿透隧道，连上**你家里的电脑**。
