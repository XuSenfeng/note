# 代码架构
## 目录描述
### 顶层目录

|目录/文件|作用|
|---|---|
|`Makefile`|顶层构建入口，整个固件的编译从这里开始|
|`rules.mk`|核心构建规则，定义编译标志、通用变量|
|`Config.in`|顶层配置菜单|
|`feeds.conf.default`|软件源（feeds）配置，定义去哪里下载额外包|
|`BSDmakefile`|BSD make 兼容|
|`COPYING` / `LICENSES`|开源许可证（GPL）|
### 核心构建目录
| 目录               | 作用                                   |
| ---------------- | ------------------------------------ |
| **`target/`**    | **各种 CPU 架构/设备的目标定义** — 这是你最该看的      |
| **`package/`**   | **所有软件包的定义、源码、编译脚本**                 |
| **`include/`**   | **构建系统的核心 .mk 包含文件**（类似函数库）          |
| **`tools/`**     | 主机端编译工具（交叉编译工具链的依赖）                  |
| **`toolchain/`** | 交叉编译工具链本身（gcc, binutils, musl/glibc） |
| **`scripts/`**   | 辅助脚本（镜像打包、刷机工具等）                     |
| **`config/`**    | 内核和构建系统的配置选项                         |
| **`dl/`**        | 下载的源码压缩包缓存                           |
| **`feeds/`**     | 扩展软件源（luci界面、routing协议等）             |
| `build_dir/`     | 编译中间文件                               |
| `staging_dir/`   | 编译好的 staging 文件（头文件、库）里面有可以使用的而根文件系统 |
| `bin/`           | 最终生成的固件镜像                            |
| `tmp/`           | 临时文件                                 |
### 详细子目录
#### target目录

每个子目录对应一种 CPU/平台
- `Makefile` — 平台定义
- `config-<version>` — 内核配置
- `base-files/` — 基础文件系统
- `patches-<version>/` — 内核补丁
- `image/` — 镜像生成配置
- `dts/` — Device Tree 源文件
- `modules.mk` — 内核模块定义

#### package目录

| 子目录               | 内容                                                                           |
| ----------------- | ---------------------------------------------------------------------------- |
| **`base-files/`** | **固件的基础文件系统骨架**（`/etc/config/`, `/sbin/`, 启动脚本等）                             |
| **`boot/`**       | **启动相关**：uboot（各种型号）、grub2、ATF、opensbi                                       |
| **`kernel/`**     | **内核模块**：mac80211（WiFi驱动）、mt76、gpio-button-hotplug 等                         |
| **`network/`**    | **网络协议栈**：config（网络配置）、ipv6、services、utils                                   |
| **`system/`**     | **系统核心服务**：procd（进程管理/init）、ubus（进程间通信）、uci（配置管理）、apk/opkg（包管理）、mtd（flash操作） |
| **`libs/`**       | **基础库**：openssl、mbedtls、libubox、libnl-tiny、libjson-c、uclient 等               |
| `firmware/`       | 无线网卡固件（ath10k、ath11k、cypress 等）                                              |
| `utils/`          | 实用工具：busybox、lua、ucode（OpenWrt的JS-like脚本语言）、jsonfilter                       |
| `devel/`          | 开发调试工具：strace、valgrind、gdb、perf                                              |
| `feeds/`          | 扩展包（指针对各 feeds 源本地编译的包）                                                      |
#### `include/` 关键文件

| 文件                               | 作用                                                 |
| -------------------------------- | -------------------------------------------------- |
| `toplevel.mk`                    | 顶层 Makefile 逻辑，处理 `make menuconfig`、`make world` 等 |
| `package.mk`                     | 所有软件包的通用编译规则                                       |
| `kernel.mk` / `kernel-build.mk`  | 内核编译规则                                             |
| `target.mk`                      | 目标平台编译规则                                           |
| `image.mk` / `image-commands.mk` | 固件镜像生成规则                                           |
| `download.mk`                    | 源码下载规则                                             |
| `feeds.mk`                       | 软件源更新规则                                            |
| `hardening.mk`                   | 安全加固编译选项                                           |
#### build_dir

这个目录是所有已经编译好的文件目录, 用于之后的构建

+ host: 所有主机运行的工具
+ tool_chain: 编译使用的工具链, 以及标准的C库
+ `target`: 编译实际的软件包和Linux内核
#### staging_dir

+ host: 小型 Linux 根目录，它有自己的 `bin/` 、 `lib/` 等目录，主机工具就安装在这里；构建系统的其余部分会将此区域中的目录添加到其 PATH 环境变量中
+ tool_chain: 小型 Linux 根目录，它拥有自己的 `bin/` 、 `lib/` 等目录，其中包含用于构建固件其余部分的交叉 C 编译器。实际上，您可以使用它在 OpenWrt 之外编译简单的 C 程序，并将这些程序加载到固件中
+ target: 实际编译出来的根文件系统











## 原理

这个仓库是一个大的脚本文件，下载各种实际使用的左键，依据实际配置选择使用的组件，具体使用什么是`。config`文件
最后把所有的组件合成一个**完整的可刷机固件** = 内核 + rootfs + 引导加载程序
### 自研
| 项目                                     | 作用          | 说明                                         |
| -------------------------------------- | ----------- | ------------------------------------------ |
| **procd** (`package/system/procd/`)    | 进程管理/init系统 | OpenWrt 自研的 init 守护进程，替代 Linux 标准的 systemd |
| **ubus** (`package/system/ubus/`)      | 进程间通信总线     | 类似 D-Bus 但极轻量（嵌入式专用）                       |
| **uci** (`package/system/uci/`)        | 统一配置接口      | 把 /etc/config/ 下的文本配置统一管理，类似一个嵌入式配置数据库     |
| **ubox** (`package/system/ubox/`)      | 基础工具集       | logd（日志）、kmodloader（内核模块加载）等               |
| **libubox** (`package/libs/libubox/`)  | C 基础库       | 事件循环、链表、JSON 解析 — 整个 OpenWrt 生态的基础库        |
| **rpcd** (`package/system/rpcd/`)      | RPC 后台      | 配合 ubus 做远程过程调用，LuCI 就是通过它管理系统的            |
| **ucode** (`package/utils/ucode/`)     | 脚本语言        | OpenWrt 自己写的类 JavaScript 语言，用于配置模板         |
| **usign** (`package/system/usign/`)    | 签名工具        | 包签名验证，替代 GnuPG（极简实现）                       |
| **apk** → 对 Alpine 的 apk 做了 OpenWrt 定制 |             |                                            |
### 流程

在 OpenWrt 的整套框架里，一个组件（不管是软件还是内核）的生命周期通常被划分为几个标准阶段：`clean`（清理）、`prepare`（准备源码/打补丁）、`compile`（编译代码）、**`install`（安装/打包）**
