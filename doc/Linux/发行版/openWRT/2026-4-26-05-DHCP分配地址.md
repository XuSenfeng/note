# DHCP分配地址

DHCP（自动给设备分配内网 IP）

核心管两个服务：

1. **dnsmasq**：OpenWrt 默认，负责 IPv4 地址分配 + 域名解析
2. **odhcpd**：负责 IPv6 地址、IPv6 网关、IPv6 DNS 推送

## 可用功能

1. 静态租约（固定 IP 绑定）

按**设备 MAC 地址**，给手机 / 电脑 / 智能家居**永久固定内网 IP**

2. MAC 地址过滤

拉黑指定设备 MAC：不给它发 IP，变相断网

全局关闭自动 DHCP：只允许你手动绑定的设备上网（白名单）

3. 系统服务冲突修复

解决路由器自带服务互相抢端口、抢 DHCP 权限的 bug

4. 自定义 DHCP 下发参数

路由器可以**强制给所有设备推送规则**：

- 统一指定全屋设备的网关、DNS、时间服务器
- 设备分组管控
    - 给某几台设备单独分配不一样的 DNS
    - 按设备品牌 / MAC 段批量归类（比如所有安卓、所有智能设备）
- 专门优化 Windows 电脑：关闭老旧 NetBIOS 局域网广播，减少卡顿、弹窗、安全隐患

5. 大型网络 DHCP 配置

适合：公司、出租屋、多设备宿舍

- 自定义超大内网网段、限制最大设备数量
- 防止 IP 地址耗尽、内网拥堵

6. 自定义内网域名 & 解析记录

不用记住 IP，直接用名字访问设备：

- 给设备起内网域名，比如 `mylaptop` 直接访问电脑
- 高级解析：
    - CNAME 别名：比如 ftp.xxx 指向 网站服务器
    - MX 邮件记录：内网邮件服务器正常发信
    - 内网专属域名：所有 `*.home` 都指向你的 NAS

7. PXE 网络开机 / 无盘启动（TFTP）

运维 / 装机用：

- 电脑不用插 U 盘 / 硬盘，网线插路由器就能重装系统、跑无盘系统
- 自动兼容 老式 BIOS、新 UEFI、ARM 各种设备

8. 多套 DHCP/DNS 独立隔离

非常实用：

- 主网、访客 WiFi、IoT 智能家居、VPN 分开两套 DNS+IP
- 访客网不能访问你家里的电脑 / NAS，完全隔离，提升安全

9. 日志、开关单独控制

- 开启 DNS 查询日志：看哪些 APP 在偷跑、访问垃圾域名
- 只关 DHCP 不关 DNS：路由器只做解析，不分配 IP
- 只禁用 IPv6 DNS、保留 IPv6 上网
- 彻底关掉 DNS 功能：路由器只发 IP，不管域名

10. 替换默认服务（进阶）

抛弃自带的 dnsmasq，换成：

**odhcpd + Unbound**

- Unbound 是更安全的递归 DNS
- 支持防污染、DNSSEC 加密、抗劫持
- 适合折腾、需要纯净网络的用户

11. 强制全屋自定义 DNS

强行给手机、电视、游戏机推送你指定的 DNS

- 替换运营商垃圾 DNS
- 可设置公共 DNS、国内纯净 DNS、防污染 DNS
- IPv4 / IPv6 分开单独设置

12. 跨网段 DNS

允许 VPN、异地组网、外网设备，也能使用你路由器的 DNS

13. DNS 缓存 / 转发 / 分流

- 关闭 DNS 缓存：每次都实时查询，适合敏感场景
- **全局转发**：所有上网域名都走指定 DNS

15. 修改上游运营商 DNS

- 改掉宽带运营商强制推送的劫持 DNS
- 手动设置 WAN、IPv6 的上游 DNS
- 关闭运营商 DNS 劫持、防止隐私泄露
- VPN 必备：禁止运营商 DNS 泄漏

### 基础命令

可以使用uci命令进行服务器的配置的设置, 实际操作和你去配置各个服务软件是一样的

| 命令               | 作用                 | 示例                                                         |
| ------------------ | -------------------- | ------------------------------------------------------------ |
| **`uci get`**      | 读取配置值           | `uci get dhcp.lan.start`（获取 DHCP 起始地址）               |


配置某个部分以后, 需要重启

### 基础配置文件

LAN / WAN 不是固定硬件，是「角色」

```bash
# dnsmasq 是 OpenWrt 默认自带的核心程序
# 负责 IPv4 DHCP（给设备发内网 IP）
# 负责 DNS 解析（把网址翻译成 IP）
# 这个块是 dnsmasq 服务的总开关 / 总设置
config dnsmasq
        option domainneeded '1'
        option boguspriv '1'
        option filterwin2k '0'
        option localise_queries '1'
        option rebind_protection '1'
        option rebind_localhost '1'
        option local '/lan/'
        option domain 'lan'
        option expandhosts '1'
        option nonegcache '0'
        option cachesize '1000'
        option authoritative '1'
        option readethers '1'
        option leasefile '/tmp/dhcp.leases'
        option resolvfile '/tmp/resolv.conf.d/resolv.conf.auto'
        option nonwildcard '1'
        option localservice '1'
        option ednspacket_max '1232'
        option filter_aaaa '0'
        option filter_a '0'

# lan = 路由器的【局域网接口】就是你插电脑、连 WiFi、接智能家居的 内网口
# 专门给 家里所有内网设备 分配 IP、网关、DNS
config dhcp 'lan'
        option interface 'lan'
        option start '100' # 分配的 IP 从 192.168.1.100 开始
        option limit '150' # 最多分 150 个 IP
        option leasetime '12h'
        option dhcpv4 'server'
        option dhcpv6 'server'
        option ra 'server'
        option ra_slaac '1'
        list ra_flags 'managed-config'
        list ra_flags 'other-config'

# wan = 路由器的【外网接口】
# WAN 口是获取 IP（从光猫拿上网 IP）
config dhcp 'wan'
        option interface 'wan'
        option ignore '1' # 关闭 WAN 口的 DHCP 服务

# 这是另一个【专用软件服务】 OpenWrt 里 专门管 IPv6 的程序
config odhcpd 'odhcpd'
        option maindhcp '0'
        option leasefile '/tmp/hosts/odhcpd'
        option leasetrigger '/usr/sbin/odhcpd-update'
        option loglevel '4'
```

> ```bash
> 路由器
> ├─ 服务1：dnsmasq  → 管 IPv4 分配 + DNS 解析
> ├─ 服务2：odhcpd   → 管 IPv6 分配
> └─ 接口
>    ├─ lan 内网  → 开启 DHCP，给设备发IP
>    └─ wan 外网  → 关闭 DHCP，只拿IP上网
> ```

### 静态IP

**host** = 主机 → 通俗说就是：**联网的设备**（电脑、手机、打印机、NAS 都叫 host）

每一个 **`config host`**，都代表 **一台指定设备的固定配置**。

```c
config host
        option name 'mylaptop'
        option dns '1'
        option mac '11:22:33:44:55:66'
        option ip '192.168.1.22'
```

### 屏蔽设备

```bash
uci add dhcp host
uci set dhcp.@host[-1].name="mydesktop"
uci set dhcp.@host[-1].mac="00:11:22:33:44:55"
uci set dhcp.@host[-1].ip="ignore"
```

### 关闭自动分配

```c
uci set dhcp.lan.dynamicdhcp="0"
```

只有你手动绑定过的设备（config host） → 能拿到固定 IP、能上网

没绑定的陌生设备 → 连得上 WiFi / 网线，但**拿不到 IP 地址 → 无法上网**

### 强制开启DHCP

```bash
uci set dhcp.lan.force="1"
```

OpenWrt 启动 DHCP 服务前，会**先扫描局域网**：

如果发现**已经有其他设备在发 IP**（比如主路由、另一个路由器开了 DHCP）

→ 为了避免**网络冲突、设备上不去网**

→ OpenWrt 会**自动关闭自己的 DHCP 服务**

你想让**树莓派自己管理所有内网设备的 IP**

+ 但上级主路由也开了 DHCP
+ 不设置 `force=1` → 树莓派的 DHCP 会**自动罢工**
+ 设置后 → 树莓派强行接管 DHCP，稳定给设备发 IP

还可以解决

- 软路由启动时，DHCP 服务**偶尔自动关闭**的 bug
- 网络服务启动顺序冲突（netifd 和 dnsmasq 抢权限）

### ipv6局域网互通

IPv6 不是「同网段就互通」

尤其是 **ULA 内网地址**：

- OpenWrt 默认 `ra_default="0"`

- **只发地址，不发本地路由 / 默认路由**, 设备只有内网 IPv6 地址，但**没有 IPv6 网关**

- 设备拿到了 `fd::xxx` 地址，但是**没有路由规则**

- 结果：

    能看到有 IPv6 地址，但是 ping 不通、访问不了、内网设备连不上

```bash
uci set dhcp.lan.ra_default="1"
uci commit dhcp
service odhcpd restart
```

### 固定的网关/DNS/时间服务器

**3** = 默认网关（设备上网走哪个路由器）

**6** = DNS 服务器（域名解析）

**42** = NTP 时间服务器（设备自动同步时间）

**44** = WINS 服务（Windows 老旧无用功能，直接禁用）

```bash
# 1. 给所有设备推送：网关 = 192.168.1.2, 所有设备上网流量 → 交给另一台设备转发
uci add_list dhcp.lan.dhcp_option="3,192.168.1.2"
# 2. 给所有设备推送：DNS = 172.16.60.64
uci add_list dhcp.lan.dhcp_option="6,172.16.60.64"
# 3. 所有设备推送：时间服务器 = 172.16.60.64
uci add_list dhcp.lan.dhcp_option="42,172.16.60.64"
# 4. 禁用老旧的 WINS 服务（没用，关了更干净）
uci add_list dhcp.lan.dhcp_option="44"
```

### 自定义域名

#### 单设备

给你的内网一个设备起一个「专属内网名字」，全屋所有设备都能用这个名字访问它

```bash
# 1. 添加一条【内网域名记录】
# domain 是固定类型关键词，不能改
uci add dhcp domain
# 2. 域名：mylaptop
uci set dhcp.@domain[-1].name="mylaptop"
# 3. 绑定 IPv4 地址
uci set dhcp.@domain[-1].ip="192.168.1.23"
# 4. 再添加一条记录（给同一个域名绑定IPv6）
uci add dhcp domain
uci set dhcp.@domain[-1].name="mylaptop"
# 5. 绑定 IPv6 地址
uci set dhcp.@domain[-1].ip="fdce::23"

# 保存配置
uci commit dhcp
# 重启DNS服务生效
service dnsmasq restart
```

| 配置              | 作用       | 核心功能                           |
| ----------------- | ---------- | ---------------------------------- |
| **config host**   | 静态租约   | **给设备固定 IP**（不让 IP 乱变）  |
| **config domain** | 主机名解析 | **给 IP 起名字**（用名字访问设备） |

#### 批量

```bash
# 核心规则：所有 home 域名 + 所有子域名，强制解析到 10.10.10.1
uci add_list dhcp.@dnsmasq[0].address="/home/10.10.10.1"
```

address="/域名/IP", 绑定**主域名 + 所有子域名**（如 *.home）

### 网络装机/无盘启动

让电脑 / 设备**不用 U 盘、不用硬盘**，通电联网就自动从局域网另一台服务器下载引导文件、启动系统

```bash
# 新建一条【网络启动规则】
# boot 是固定类型（专门管PXE/TFTP启动）
# linux 是你随便起的规则名，可以改成 pxe/tftp 都行
uci set dhcp.linux="boot"
# 指定：设备开机要下载的引导文件
# pxelinux.0 是 Linux 通用网络引导程序
uci set dhcp.linux.filename="/tftpboot/pxelinux.0"
# 指定 TFTP 服务器地址：所有启动文件都从 192.168.1.2 下载
uci set dhcp.linux.serveraddress="192.168.1.2"
# 给这台启动服务器起个别名（随便写，无实际影响）
uci set dhcp.linux.servername="fileserver"
# 高级参数：告诉无盘设备，系统根目录挂载在服务器的 /data/netboot/root
# 实现完整无盘运行系统
uci add_list dhcp.linux.dhcp_option="option:root-path,192.168.1.2:/data/netboot/root"
# 保存+重启生效
uci commit dhcp
service dnsmasq restart
```

关键：**联网不是靠 Windows / Linux 系统，是靠「网卡本身」**

所有主流有线网卡、部分无线网卡，出厂自带 **PXE 固件**：

1. 电脑开机 → BIOS 优先调用网卡里的 PXE 小程序
2. 这个小程序**独立、极小、固化在网卡里**
3. 它会自动发 DHCP 请求（连你的 OpenWrt）
4. 先拿到 IP、TFTP 服务器地址，完成联网

完整的流程

1. 网卡 PXE 发起 DHCP 请求
2. 你的 OpenWrt（dnsmasq）返回特殊引导参数

- `serveraddress=192.168.1.2` 👉 去找这台服务器
- `filename=pxelinux.0`        👉 先下载这个引导程序

3. 网卡通过轻量 TFTP 协议，下载引导文件
4. 引导文件再下载 Linux 内核、临时文件系统
5. 最后通过网络协议（NFS）挂载远端路径：

```
192.168.1.2:/data/netboot/root
```

### 创建两个网络

**同时运行 2 个完全独立的 DNS/DHCP 服务**，一个给**主局域网（lan）**，一个给**访客网络（guest）**，两个网络的 IP 分配、DNS 解析、设备列表**彻底隔离、互不干扰**

+ 访客能看到主网设备 → 不安全
+ 共用一套 DNS/IP 池 → 无法单独管控

> 创建 2 个 DHCP 配置，分别绑定 2 个独立的 dnsmasq 服务进程

```bash
# Remove default instances
# 删除OpenWrt自带的单个dnsmasq、dhcp配置
while uci -q delete dhcp.@dnsmasq[0]; do :; done
while uci -q delete dhcp.@dhcp[0]; do :; done
 
# Use network interface names for DHCP/DNS instance names
# 这两个名字在network里面配置实际使用的硬件的网卡
# 创建两个分组, 里面绑定的是不同的网卡
INST="lan guest"
for INST in ${INST}
do
# 新建一个独立的 dnsmasq 服务进程, 两个服务完全分开、互不干扰
uci set dhcp.${INST}_dns="dnsmasq"
# 只有完整域名才去外网查询，纯内网名字（如 nas、openwrt）只在内网解析
uci set dhcp.${INST}_dns.domainneeded="1"
# 拦截内网私有 IP 的恶意反向解析
uci set dhcp.${INST}_dns.boguspriv="1"
uci set dhcp.${INST}_dns.filterwin2k="0"
uci set dhcp.${INST}_dns.localise_queries="1"
uci set dhcp.${INST}_dns.rebind_protection="1"
uci set dhcp.${INST}_dns.rebind_localhost="1"
uci set dhcp.${INST}_dns.local="/${INST}/"
uci set dhcp.${INST}_dns.domain="${INST}"
uci set dhcp.${INST}_dns.expandhosts="1"
uci set dhcp.${INST}_dns.nonegcache="0"
# 告诉所有设备「我是这个网段唯一的官方 DHCP/DNS」
uci set dhcp.${INST}_dns.authoritative="1"
uci set dhcp.${INST}_dns.readethers="1"
# 独立 IP 租约文件
uci set dhcp.${INST}_dns.leasefile="/tmp/dhcp.leases.${INST}"
# 使用系统自动生成的外网 DNS
uci set dhcp.${INST}_dns.resolvfile="/tmp/resolv.conf.d/resolv.conf.auto"
uci set dhcp.${INST}_dns.nonwildcard="1"
# 只监听对应的网卡
uci add_list dhcp.${INST}_dns.interface="${INST}"
# 排除回环口，防止两个服务冲突
uci add_list dhcp.${INST}_dns.notinterface="loopback"

#  创建一个实际的 dhcp 配置
uci set dhcp.${INST}="dhcp"
uci set dhcp.${INST}.instance="${INST}_dns"
uci set dhcp.${INST}.interface="${INST}"
uci set dhcp.${INST}.start="100"
uci set dhcp.${INST}.limit="150"
uci set dhcp.${INST}.leasetime="12h"
done
uci -q delete dhcp.@dnsmasq[0].notinterface
uci commit dhcp
service dnsmasq restart
```

+ `lan_dns` → 只管主网
+ `guest_dns` → 只管访客网

> ```bash
> # 1. 定义网桥：把物理网卡捆在一起
> config device 'lan_br_dev'
>         option name 'br-lan'      # 网桥名字
>         option type 'bridge'
>         list ports 'eth0'        # 物理网口 LAN
>         list ports 'wlan0'       # 物理WiFi 2.4G
>         list ports 'wlan1'       # 物理WiFi 5G
> 
> # 2. 定义逻辑接口 lan（关键！）
> config interface 'lan'
>         option device 'br-lan'    # lan 绑定网桥 br-lan
>         option proto 'static'
>         option ipaddr '192.168.1.1'
>         option netmask '255.255.255.0'
>         
> # 新建访客网桥 + 绑定访客WiFi
> config device 'guest_br'
>         option name 'br-guest'
>         option type 'bridge'
>         list ports 'wlan0-1'  # 访客WiFi物理网卡
> 
> # 新建逻辑接口 guest
> config interface 'guest'
>         option device 'br-guest'
>         option proto 'static'
>         option ipaddr '192.168.2.1'
> ```

### 局域网外DNS解析

允许「非本地局域网」的设备（最常见：VPN 远程设备）使用你路由器的 DNS 解析服务

dnsmasq 默认**只认家里局域网**（lan、guest 网段）：

- 家里手机 / 电脑 → 可以用路由器 DNS ✅
- **远程连 VPN 的设备** → 不属于本地局域网 → 被拒绝 DNS 请求 

这是安全保护，防止外部网络恶意访问你的 DNS。

````bash
uci set dhcp.@dnsmasq[0].localservice="0"
uci commit dhcp
service dnsmasq restart
````

### 自定义DNS配置

默认的处理流程

1. 光猫 / 宽带运营商 → 自动给你路由器 **推送运营商 DNS**
2. 路由器 WAN 口收下这个 DNS → 存到系统解析文件
3. `dnsmasq` 默认**自动读取这个运营商 DNS**，当做上游
4. 手机 / 电脑 连 WiFi → DHCP 拿到「路由器 IP 当做 DNS」
5. 设备 → 路由器 dnsmasq → **运营商 DNS**

+ 优先级

1. 【最低】运营商自动 DNS（peerdns=1 默认）
2. 【中低】WAN 手动自定义 DNS（`peerdns=0 + network.wan.dns`）
3. 【中高】dnsmasq 全局转发 `server="8.8.8.8"`
4. 【最高】选择性 DNS 转发（指定域名单独走 DNS）

案例 1：纯默认

- WAN：用运营商 DNS

- dnsmasq：没写 server、noresolv=0

    👉 结果：全网走 运营商 DNS

案例 2：只改 WAN 上游 DNS（你刚看的文档）

- `peerdns=0`

- 手动设置 `wan.dns=8.8.8.8`

- dnsmasq 无自定义 server

    👉 结果：全网走 你设置的谷歌 DNS

案例 3：不改 WAN，只改 dnsmasq 转发

- WAN 还是运营商 DNS

- dnsmasq 加 `server=114.114.114.114`

    👉 结果：无视 WAN，全网强制走 114

#### 设备直连

```bash
# Configure dnsmasq 这部分是IPv4的配置
uci -q delete dhcp.lan.dhcp_option
# 给设备直接推送DNS	设备自己连 DNS，绕过路由器
uci add_list dhcp.lan.dhcp_option="6,8.8.8.8,8.8.4.4"
uci commit dhcp
service dnsmasq restart
 
# Configure odhcpd 这里是Ipv6的
uci -q delete dhcp.lan.dns
uci add_list dhcp.lan.dns="2001:4860:4860::8888"
uci add_list dhcp.lan.dns="2001:4860:4860::8844"
uci commit dhcp
service odhcpd restart
```

#### 统一转发

路由器**统一转发**DNS设备只信任路由器，路由器代查, 路由器**不自己查** → 转发给 `8.8.8.8`, 直接覆盖 WAN 的 DNS

```bash
# 1. 清空路由器原来的所有上游DNS服务器（防止冲突）
uci -q delete dhcp.@dnsmasq[0].server

# 2. 添加第一个上游DNS：谷歌 8.8.8.8
uci add_list dhcp.@dnsmasq[0].server="8.8.8.8"
# 3. 添加备用上游DNS：谷歌 8.8.4.4
uci add_list dhcp.@dnsmasq[0].server="8.8.4.4"

# 保存+重启生效
uci commit dhcp
service dnsmasq restart
```

### 只用指定的DNS

不读取系统自动获取的**运营商 DNS**, 路由器**只使用**你手动用 `server=` 配置的上游 DNS

```bash
service dnsmasq stop            # 先停止服务，防止修改配置时冲突
uci set dhcp.@dnsmasq[0].noresolv="1"  # 【关键】忽略WAN的DNS文件（resolv.conf）
uci commit dhcp                 # 保存配置
service dnsmasq start           # 重启生效
```

分割 DNS 后，**强制路由器本机也使用自己的 dnsmasq 服务**，避免路由器自己走外网 DNS。

```bash
uci set dhcp.@dnsmasq[0].localuse="1"  # 【关键】路由器本机强制使用dnsmasq
```

给「指定域名 / 域名后缀」单独分配 DNS 服务器，精准分流，优先级高于全局转发

```bash
uci add_list dhcp.@dnsmasq[0].server="/example.com/192.168.2.1"
uci commit dhcp
service dnsmasq restart
```

#### 黑白名单

```bash
# 屏蔽 example.com 和 example.net 这两个域名
uci add_list dhcp.@dnsmasq[0].server="/example.com/"
uci add_list dhcp.@dnsmasq[0].server="/example.net/"
```

路由器收到这个域名的解析请求，**直接返回空 / 解析失败**

```bash
# 放行指定域名
uci add_list dhcp.@dnsmasq[0].server="/example.com/#"
# 全局放行所有域名（顶级白名单）
uci add_list dhcp.@dnsmasq[0].server="/#/"
```

### 摆脱运营商

更改原始的上游的DNS配置

```bash
# -------- IPv4 上游DNS --------
uci -q delete network.wan.dns          # 清空WAN原来残留的自定义DNS
uci add_list network.wan.dns="8.8.8.8"  # 手动指定IPv4上游DNS
uci add_list network.wan.dns="8.8.4.4"

# -------- IPv6 上游DNS --------
uci -q delete network.wan6.dns
uci add_list network.wan6.dns="2001:4860:4860::8888"
uci add_list network.wan6.dns="2001:4860:4860::8844"

# 关闭运营商自动分配DNS
uci set network.wan.peerdns="0"
uci set network.wan6.peerdns="0"

uci commit network
service network restart
```

系统原始、底层的 DNS 地址池, dnsmasq **默认会读取这里** 作为转发目标

加上`noresolv="1"   # dnsmasq 彻底不读取 WAN/系统任何DNS`
