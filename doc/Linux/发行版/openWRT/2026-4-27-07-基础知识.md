# 基础知识

## Shell

在启动的时候, Linux会启动守护进程, 目前使用的是`procd`会启动`/etc/rc.d`里面的各种程序, 包括一个shell

默认安装了Dropbear支持SSH, 静态链接二进制文件仅约**110KB**（x86 平台，uClibc，最小选项），内存占用仅**1-5MB**，远低于 OpenSSH 的 10-50MB

实际使用的shell是busybox里面的`ash`, Almquist Shell, 它是一款**超轻量的命令行终端解释器**。

在启动的时候会使用脚本`/etc/profile`进行环境变量的设置以及别名的设置

## 系统结构

**上层服务依赖下层硬件**，每一层单独可配置

1.   LuCI 网页管理（你操作的界面）, **流量规则、安全隔离**, 核心：划分不同「安全区域 Zone」
2.   防火墙 Zone 区域（安全隔离）, 把**逻辑接口**划分到不同安全域, `lan` 接口 → 绑定LAN区域, `wan` 接口 → 绑定 WAN 区域
3.   逻辑接口：`LAN / WAN / WAN6`（配 IP、拨号、DHCP）, `lan`：局域网逻辑接口（负责内网 IP、DHCP）, `wan`：IPv4 上网接口（拨号 / DHCP 获取运营商 IP）, `wan6`：IPv6 上网接口, 一个 `lan` 逻辑接口，可以同时绑定**多个完全不同的硬件**
4.   底层设备：网桥 / 物理网卡 `br-lan / eth0`, 实际的硬件层, 接口要绑定底层硬件设备
5.   交换机 VLAN + 无线射频 radio
6.   机器内部端口、无线硬件标识
7.   路由器外壳上厂商印的物理标签

>   **`lan`、`wan`、`wan6` 全是「逻辑抽象接口」** 不是物理网卡、不是固定网口、不是硬件。
>
>   真正的**物理硬件**只有：`eth0` 以太网卡、无线`radio0/1`、交换机芯片、VLAN 端口、USB 网卡等。
>
>   防火墙里的 `lan区域/wan区域`，又是**另一层独立抽象**, 只管流量放行 / 拦截，和物理网口彻底解绑

所有网络**统一由 UCI 管理**，配置全在 `/etc/config/network`

网络分两层：

-   下层 `device`：管硬件、网桥、VLAN、物理网卡
-   上层 `interface`：管 IP、拨号、DHCP、路由（lan/wan 都在这）

物理网口靠 **交换机 VLAN** 分组，再绑网桥，再绑逻辑接口

### 四层流水

#### 最底层硬件

最底层 → 纯物理网口

就是路由器外壳上实打实的硬件, **本身没有归属、没有 IP、不分内外网**：

-   物理 WAN 口（插光猫）
-   物理 LAN1 / LAN2 / LAN3 / LAN4（插电脑、交换机）
-   2.4G/5G 无线硬件（radio）

#### 交换机芯片

交换机芯片 + VLAN → 给物理口「分组分家」

>   交换机芯片专门干**网线之间数据转发**（比如电脑连电视传文件）, 这颗芯片原生支持 VLAN 划分

靠 **VLAN** 做强制分组、隔离：

+   把 4 个 LAN 物理口 → 划入 **VLAN 1** → 你的内网（手机、电脑、电视）
+   把 1 个 WAN 物理口 → 划入 **VLAN 2** → 外网（光猫、运营商）
+   留一个「CPU 专用口」，负责和路由器主控通信

分组后，系统会生成**虚拟子网卡**：

-   `eth0.1` = VLAN1 整组内网口
-   `eth0.2` = VLAN2 单独外网口

路由器**主控 CPU 本身，只有 1 张真实物理网卡**： `eth0`, `eth0` 这一根线，**只连到 交换机芯片**

>   CPU **从来直接看不到**：WAN 口、LAN1、LAN2、LAN3、LAN4 这些真实网口
>
>   -   CPU 只管发 / 收 `eth0.1`、`eth0.2` 这种**虚拟流量**
>
>   -   交换机芯片独自负责
>
>       识别 VLAN 标签 → 把流量分发到对应的真实物理网口

#### 网桥

把**多个独立的网络设备**，强行融合成**同一个局域网**，实现完全互通。

经过 VLAN 划分后，你路由器内网有**3 个互相孤立的设备**：

1.  `eth0.1`  ：所有有线 LAN 口（电脑插网线）
2.  2.4G WiFi：手机连低频 wifi
3.  5G WiFi  ：手机连高频 wifi

三个网络**互相隔离、不能投屏、不能传文件、不能访问打印机**，还要分别设置 IP、开 3 套 DHCP，完全没法正常家用。

+   有了网桥 `br-lan` 之后: 

有线、2.4G、5G **全部在同一个网段** `192.168.1.x`, 共用一套 DHCP、一套内网规则

+   逻辑接口 `lan`：绑定网桥，配置内网 IP / 防火墙

#### 逻辑接口

lan /wan 抽象逻辑接口

在上层做 IP、DHCP、拨号、防火墙绑定：

1.  逻辑接口 `lan`  → 绑定网桥 `br-lan` 配内网 IP：`192.168.1.1`、开 DHCP 给手机电脑发 IP
2.  逻辑接口 `wan`  → 绑定 VLAN 网卡 `eth0.2` 或者配置的无线网络, 配上网方式：DHCP / PPPoE 拨号、连外网

### 控制

| 命令                      | 到底在管什么？                                   |
| ------------------------- | ------------------------------------------------ |
| `service network reload`  | 软重启**整个网络系统**（不断网，加载新配置）     |
| `service network restart` | 硬重启**整个网络系统**（断网重连）               |
| `ifup wan` / `ifdown wan` | 只开关**逻辑接口 wan**（外网入口）               |
| `ifup lan`                | 只开关**逻辑接口 lan**（内网）                   |
| `wifi up`                 | 修复 WiFi 和网桥`br-lan`的绑定                   |
| `ifstatus wan`            | 查看**逻辑接口 wan**的状态（IP、网关、绑定硬件） |

### 默认配置

```bash
# 本地回环（系统自用，不用管）
config interface 'loopback'
	option device 'lo'
	option proto 'static'
	option ipaddr '127.0.0.1'
	option netmask '255.0.0.0'
 
# 全局配置（IPv6 内网前缀）, 给家里所有设备分配内网 IPv6 地址用的随机前缀
config globals 'globals'
	option ula_prefix 'fd27:70fa:5c1d::/48'
 
# 内网网桥（最关键的合并工具）, br-lan（内网标准网桥）
# eth0.1（VLAN1 虚拟网卡 = 所有物理 LAN 口）
config device 'lan_br'
	option name 'br-lan'
	option type 'bridge'
	list ports 'eth0.1' # 实际连接到eth0.1

# LAN 局域网接口,把 4 个物理 LAN 口合并成一个局域网
config interface 'lan'
	option device 'br-lan'
	option proto 'static'
	option netmask '255.255.255.0'
	option ip6assign '60'
	option ipaddr '192.168.1.1'

# WAN 广域网接口, 物理网卡 eth0 + VLAN 2
config interface 'wan'
	option device 'eth0.2' # 实际连接到eth0.2
	option proto 'dhcp'

# IPv6上网的口
config interface 'wan6'
	option device 'eth0.2' # 实际连接到eth0.2
	option proto 'dhcpv6'

# 内置硬件交换机, 开启 VLAN 虚拟局域网
config switch
	option name 'switch0'
	option reset '1'
	option enable_vlan '1'

# VLAN 1 划分, 路由器的 4 个物理 LAN 口 和 CPU 内部端口
config switch_vlan
	option device 'switch0'
	option vlan '1'
	option ports '1 2 3 4 5t'

# VLAN 2 划分, 路由器的 1 个物理 WAN 口 和 CPU 内部端口
config switch_vlan
	option device 'switch0'
	option vlan '2'
	option ports '0 5t'
```

> `eth0` 是路由器**CPU 自带的唯一一块物理以太网卡**, 路由器核心芯片**唯一的网线总通道**。
>
> 1. 路由器 CPU 只有 **1 个物理网口：`eth0`**
>
> 2. 机身后面的 5 个网口（4 个 LAN + 1 个 WAN），不是直接接 CPU，
>
>     
>
>     而是接在独立交换芯片 switch0上
>
> 3. 交换芯片最后**只拉一根线**连到 CPU 的 `eth0`
>
> 所有 WAN、LAN 的流量，**全部挤在 `eth0` 这一条物理通道**进出 CPU。
>
> 只有一根物理网口 `eth0`，想要分开内网、外网，就用 **VLAN 标签隔离**：
>
> - `eth0.1`  **绑定 VLAN1** → 当做 LAN 口（局域网）
> - `eth0.2`  **绑定 VLAN2** → 当做 WAN 口（外网）
>
> 这俩叫 **VLAN 子接口**，逻辑上拆分，物理上还是走同一个 `eth0`。

可以使用命令`ubus list network.interface.*`查看接口的列表

要查看关于特定接口(UCI名称而不是物理接口)的所有信息: `ifstatus lan`

### 配置内容

+ `globals` 节包含了影响一般网络配置的接口独立选项。
+ `interface`类型的节声明逻辑网络作为IP地址配置的容器

一个最小的接口声明由以下几行组成

```bash
network.wan=interface
network.wan.ifname='eth0.2'
network.wan.proto='dhcp'
```

这个interface的名字是可以自定义的, 但是核心服务（防火墙、DHCP、WiFi、网页后台），全部默认绑定这三个固定名字
