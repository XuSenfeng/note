# 中断系统

Linux里面很多的功能通过中断实现, 硬件, 系统调用, 自定义中断, 信号中断, 系统异常和错误

## 中断系统

### 分类

+ 硬件中断: 电脑的硬件中断控制芯片发出的以及ARM中断控制器发出的中断
+ 软件中断: 异常, CPU自行保留的中断, 系统调用异常

### 代码结构

中断前的保存以及中断后的恢复	 中断的执行函数

kernel/ams.s文件				kernel/trap.c				硬件中断处理

kernel/system_call.s文件    		fork.c, signal.c, exit.c, sys.c	软件系统调用的处理

## 中断工作流程

+ CPU工作模式的变化
+ 寄存器的拷贝压栈
+ 设置中断向量表
+ 保存当前的位置
+ 跳转服务函数执行
+ 跳转返回正常工作函数继续运行

## 代码

所有的中断记录在idt里面, 使用`set_trap_gate`和`set_system_gate`等宏定义进行设置, 主要是在trap_init函数(kernel\traps.c)里面配置, 系统调用使用的0x80中断, 所有系统调用记录在`_sys_call_table`结构体(include\linux\sys.h)中, 由函数`_system_call`处理

在处理系统调用的时候是不处理信号的, 但是返回的时候会进行判断, 有未处理的信号的时候, 首先去到这部分信号里面

信号默认动作

+ SIGINT (2) 终止进程
+ SIGKILL (9) 强制杀死（不能改）
+ SIGSTOP暂停进程
+ SIGCHLD (17)忽略（父进程可自己处理）
+ SIGSEGV崩溃 + core dump
