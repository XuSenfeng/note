# 内存管理

Linux的物理内存使用

内核内存 => 高速缓冲区 => (虚拟盘) => 主内存

> 640K=>1M: BIOS以及显存

+ 逻辑地址: 每一个进程代码访问的地址, 是一个段里面的偏移量
+ 线性地址: 总线地址, CPU使用的分段, 线性地址 = 段基地址 + 偏移地址， 段基 = 0，和逻辑地址数值上相等
+ 物理地址:物理内存的实际的地址 

Linux使用4K为单位进行内存的管理

**GDT, LDT**: 全局描述符和局部描述符

**GDT**里面记录内核代码段, 内核数据段, 系统段, 各个任务的ldt和tss

每个任务的**ldt**里面记录当前任务的代码段和数据段

在 linux0.11里面, 每个任务可以使用的内存是64M, 依次向后排布, 醉的的任务个数是64, 所以虚拟地址的大小是4G

**页目录表**: 二级页表对应也目录项, 记录虚拟地址和物理地址的映射, 每一个页表对应一个页表项, 表的地址记录在CR3寄存器里面, 放在地址0

## 缺页处理

在读取页表的时候, 某一个位置发现页表没有分配 ,这个时候会触发一个异常处理函数, 此时才会分配内存page_fault, 处理代码在mm\page.s

### Cope on write

在任务的拷贝的时候, 只是处理原始进程的虚拟内存, A和B使用的是同一个物理地址 `copy_page_tables`

这里 `&= ~2` 清除了 **R/W 位（写权限位）**，使得父子进程的页表项都指向**同一个物理页面，但都是只读的**。

1. **初始共享，但只读**
    父子进程都能读，但不能写。
2. **触发写保护异常**
    任何进程尝试写这个页面 → CPU 触发缺页异常（Page Fault）。
3. **异常处理中才复制**
    异常处理函数 `do_wp_page()` 被调用，见 memory.c:283-301。
    它会 `get_free_page()` 分配新页面，`copy_page()` 复制数据，然后把页表项更新为新页面地址并置 R/W=1。

- 数据拷贝**延迟到需要写的时刻**
- 如果某个进程从不修改某页面，就永远不复制，节省内存, 两个都复制出来可写的页面的时候, 之前的界面引用次数归零释放, 引用次数是在`mem_map`里面
- 两个进程修改时各自有自己的副本，互不影响

### 内存分配

所有的内存的管理使用的数组`mem_map`, 记录每一个内存页的引用次数, 在初始化的时候, 清空一下可以使用的位置, 设置为0, 表示是空闲的, 之后使用引用次数判断某一个位置是不是可以使用

再之后的磁盘里面, 会检测页表里面的数据使用的时间, 长时间没有使用的页表会把数据放在磁盘里面, 实际使用的时候再次出发异常, 从内存里面加载数据

在进行写实拷贝的页表复制的时候, 需要处理一下只有缓冲区之后的内存复制的时候原来的页面设置为只读的(只配置用户的区域)`put_page`: 把一个已分配的物理页 `page` 映射到指定线性地址 `address`。这个需要是一个独占的地址

+   `write_verify`: 页面验证, 判断是不是可写的, 不是可写的页面的时候复制一个新的页面出来
+   `get_empty_page`: 获取一个空闲的物理页, 映射到对应的虚拟地址
+   `try_to_share`: 和另一个进程分享一个地址, 计算实际的地址偏移(加上不同进程代码段的起始位置)检查是不是
+   `share_page`: 对于try_to_share的封装, 在当前进程发生缺页时，尝试“借用”其他进程中同一可执行文件的现成物理页，实现页面共享，避免重复从磁盘读入。如果共享成功返回 1；失败返回 0，调用方再走正常的分配页+读盘流程。
+   
+   `mem_init`: 初始化内存的分配的表, 在主函数里面使用

### 映射刷新

Linux经常访问一个表的时候, 保这个表记录在高速缓冲里面, 在页表变化以后, 需要刷新一下页表(重新加载)

```c
#define invalidate() \
__asm__("movl %%eax,%%cr3"::"a" (0))
```

## 异常

页面的异常有两种, 一种是页面的权限异常, 一种是没有分配, 判断以后使用两个C语言函数进行处理

`do_no_page`: 处理一个任务是不是缺页, 检查这个任务是否是从文件加载的, 不是的话直接分配一个新的页, 是的尝试从别的任务查看是否有相同可执行文件加载得来的, 进行复制, 全部失败以后在从

`do_wp_page`: 实际就是对`un_wp_page`函数的调用, 解除写保护, 解除的时候查看之前的页是不是共享的, 是的话在处理的时候新分配一个页, 设置这个新的页是可以使用的,处理写时复制的错误中断就是使用这个函数
