# 任务调度

## 等待队列

有两种等待队列, swait（simple waitqueue）和 wait（regular waitqueue）主要区别如下：

1. 实时性与确定性

- swait 设计用于实时场景，保证中断和锁持有时间有严格上限，行为更可预测。
- wait 适用于通用场景，功能更丰富，但可能有较长的锁持有时间。

2. 功能精简

- swait 只支持 exclusive（互斥）等待，唤醒时每次只唤醒一个等待者，不能混合 INTERRUPTIBLE/UNINTERRUPTIBLE 睡眠，也不支持自定义唤醒回调。
- wait 支持 exclusive 和非 exclusive 模式，可以混合不同类型的等待，支持自定义唤醒回调。

3. 数据结构更轻量

- swait 结构体更小巧，适合对性能和实时性要求极高的场合。
- wait 结构体更通用，功能更全。

1. 使用限制

- swait 只建议在有严格实时需求的地方使用，普通场景推荐用 wait。

4. 锁和内存屏障

- swait 允许无锁检测队列是否有等待者，但需要开发者手动保证内存屏障（smp_mb）正确，否则容易丢失唤醒。
- wait 也有类似的无锁检测，但更常规地配合锁使用。

### swait

```c
struct swait_queue_head {
	raw_spinlock_t		lock;
	struct list_head	task_list;
};

struct swait_queue {
	struct task_struct	*task;
	struct list_head	task_list;
};
```

#### 等待

```c
#define ___swait_event(wq, condition, state, ret, cmd)			
({									
	__label__ __out;						
	struct swait_queue __wait;	// 创建等待队列			
	long __ret = ret;						
					
	INIT_LIST_HEAD(&__wait.task_list);
	for (;;) {
		// 当前任务进等待队列 设置状态
		long __int = prepare_to_swait_event(&wq, &__wait, state);
									
		if (condition)		// 条件达成跳出
			break;			
		// 区分当前等待是否需要响应信号打断，决定是否在收到信号时提前退出等待
		if (___wait_is_interruptible(state) && __int) {
			__ret = __int;	
			goto __out;		
		}	
		// 一般处理实际的休眠, 使用schedule()以及schedule_timeout(__ret)
		cmd;
	}
	finish_swait(&wq, &__wait);
__out:	__ret;
})

// 检查一下任务是否有信号, 设置状态, 挂载到队列里面去
long prepare_to_swait_event(struct swait_queue_head *q, struct swait_queue *wait, int state)
{
	unsigned long flags;
	long ret = 0;

	raw_spin_lock_irqsave(&q->lock, flags); // 加锁保护
    // 检查当前的任务是否有待处理的信号
	if (signal_pending_state(state, current)) {
		/*
		 * See prepare_to_wait_event(). TL;DR, subsequent swake_up_one()
		 * must not see us.
		 */
		list_del_init(&wait->task_list);
		ret = -ERESTARTSYS;
	} else {
		__prepare_to_swait(q, wait); // 进行挂载
		set_current_state(state); // 设置当前任务状态
	}
    // 解锁
	raw_spin_unlock_irqrestore(&q->lock, flags);

	return ret;
}

void __prepare_to_swait(struct swait_queue_head *q, struct swait_queue *wait)
{
	wait->task = current;// 当前任务挂载
	if (list_empty(&wait->task_list))
		list_add_tail(&wait->task_list, &q->task_list);
}

// 退出的时候设置任务的状态以及从链表里面移除
void finish_swait(struct swait_queue_head *q, struct swait_queue *wait)
{
	unsigned long flags;

	__set_current_state(TASK_RUNNING);

	if (!list_empty_careful(&wait->task_list)) {
		raw_spin_lock_irqsave(&q->lock, flags);
		list_del_init(&wait->task_list);
		raw_spin_unlock_irqrestore(&q->lock, flags);
	}
}
```

#### 唤醒

设置任务的标志同时加入调度

```c
void swake_up_one(struct swait_queue_head *q)
{
	unsigned long flags;

	raw_spin_lock_irqsave(&q->lock, flags);
	swake_up_locked(q);
	raw_spin_unlock_irqrestore(&q->lock, flags);
}

void swake_up_locked(struct swait_queue_head *q)
{
	struct swait_queue *curr;

	if (list_empty(&q->task_list))
		return;
	// 唤醒覅一个任务
	curr = list_first_entry(&q->task_list, typeof(*curr), task_list);
	wake_up_process(curr->task);
	list_del_init(&curr->task_list);
}

int wake_up_process(struct task_struct *p)
{
	return try_to_wake_up(p, TASK_NORMAL, 0);
}
```
