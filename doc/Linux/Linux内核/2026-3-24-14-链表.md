# 链表

## 数据结构

### 基础链表

```c
struct list_head {
	struct list_head *next, *prev;
};
```

只是一个基础的双向链表

```c
#define LIST_HEAD_INIT(name) { &(name), &(name) }

#define LIST_HEAD(name) \
	struct list_head name = LIST_HEAD_INIT(name)
```

初始化的时候设置为两个头部都是指向自己

> 在 __list_add 函数中，prev->next 的写操作使用了 WRITE_ONCE（保证可见性和顺序性），而 prev 本身的赋值（如 new->prev = prev）没有用 WRITE_ONCE。这是因为：
>
> - prev 指针的写入只会被当前 CPU 直接使用，不会被其他 CPU 并发读取。它属于“新节点”自身的私有成员，赋值后只有本地代码会用到。
> - 而 prev->next 是链表中“前一个节点”的 next 指针，可能被其他 CPU 并发遍历链表时读取，所以必须用 WRITE_ONCE 保证写入对其他 CPU 立即可见，防止乱序。
> - next->prev 也是类似道理，通常链表遍历是通过 next 指针进行的，prev 主要用于本地回溯，不涉及并发可见性。

### 哈希链表

- struct hlist_head：链表头，只包含一个指向第一个节点的指针 first。
- struct hlist_node：链表节点，包含 next（下一个节点指针）和 **pprev（指向前一个节点的 next 指针的指针）。

适合哈希表的桶（bucket）场景，每个桶用 hlist_head 作为表头，桶内元素用 hlist_node 串联。hlist 其实就是“为哈希表优化的链表”，并不是说它本身就是哈希表

```c
struct hlist_head {
	struct hlist_node *first;
};

struct hlist_node {
	struct hlist_node *next, **pprev;
};
```
