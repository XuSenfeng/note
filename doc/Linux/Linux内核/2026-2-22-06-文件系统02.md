+ 初始化文件表数组
+ 初始化超级块数组
+ 读取根文件设备的超级块

## 文件

fs\namei.c

文件的权限, 文件系统的操作chmod chown mknod, 内存区域的管理检索

### 权限

+ `permission`: 获取当前的请求的用户对于一个inode的权限

### 查询

+ `match`: 文件名和一个文件夹里面的项的匹配
+ `find_entry`: 在指定的目录中查找一个与名字匹配的目录项, 返回对应的dir_entry结构, 和目录文件缓冲区
+ `get_dir`根据一个路径的名称获取一个目录, 获取对应的inode节点, 是一个处理字符串, 依次查询inode的函数, 不是目录返回NULL, 返回最后一个名字之后是`/`的目录
+ `dir_namei`: 在 `get_dir()` 基础上，**额外返回“最后一段名字”和长度**，用于创建/查找该名字, 获取的是顶层目录的最后一个/之前的目录的inode
+ `namei`: 获取指定的路径的i节点
+ `open_namei`: 打开指定的文件的inode, 使用文件的打开标志以及属性

### 文件夹

```c
struct dir_entry {
	unsigned short inode;          // i 节点
	char name[NAME_LEN];           // 文件名
};
```

> 目录文件的数据块就是一串 `struct dir_entry` 记录，用来把“名字 → i 节点号”映射起来

i 节点本身**不直接包含文件夹内容**，而是通过**数据块指针** `i_zone[]` 指向存储目录项的数据块。
