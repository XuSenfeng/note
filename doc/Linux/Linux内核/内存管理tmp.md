在进行写实拷贝的页表复制的时候, 需要处理一下只有缓冲区之后的内存复制的时候原来的页面设置为只读的(只配置用户的区域)

```BASH
物理地址范围              用途                    mem_map管理
0x000000 - 0x100000    内核代码/页表              ✗ 不管理, 不设置只读
         │ (1MB)       
         │
0x100000 ─┼───────────── ← LOW_MEM ──────────────
         │             缓冲区 (3MB)             ✓ mem_map[0-767]=USED 不分配
         │             
0x400000 ─┼───────────── ← main_memory_start ───
         │             
         │             主内存区 (12MB)          ✓ mem_map[768-3839]=0
         │             (用户进程使用)
        ─┼───────────── 实际的内存大小 ───
         │																		✓ mem_map=USED
0x1000000─┴───────────── ← HIGH_MEMORY ─────────
```

页面在进行管理的时候是按照4M为单位进行管理的, 对应的是一个一级页表, 这个一级页表有1024项, 里面的数据指向4G大小的位置

```bash
物理内存布局：
┌──────────────────────────────┐
│ 0x0000 - 0x0FFF (4KB)        │ ← _pg_dir（页目录，一级页表）
├──────────────────────────────┤
│ 0x1000 - 0x1FFF (4KB)        │ ← pg0（页表0，二级页表）
├──────────────────────────────┤
│ 0x2000 - 0x2FFF (4KB)        │ ← pg1（页表1，二级页表）
├──────────────────────────────┤
│ 0x3000 - 0x3FFF (4KB)        │ ← pg2（页表2，二级页表）
├──────────────────────────────┤
│ 0x4000 - 0x4FFF (4KB)        │ ← pg3（页表3，二级页表）
├──────────────────────────────┤
│ 0x5000+                      │ ← _tmp_floppy_area 等其他数据
└──────────────────────────────┘
```

+   `put_page`: 把一个已分配的物理页 `page` 映射到指定线性地址 `address`。这个需要是一个独占的地址
+   `un_wp_page`解除写保护, 解除的时候查看之前的页是不是共享的, 是的话在处理的时候新分配一个页, 设置这个新的页是可以使用的,处理写时复制的错误中断就是使用这个函数
+   `write_verify`: 页面验证, 判断是不是可写的, 不是可写的页面的时候复制一个新的页面出来
+   `get_empty_page`: 获取一个空闲的物理页, 映射到对应的虚拟地址
+   `try_to_share`: 和另一个进程分享一个地址, 计算实际的地址偏移(加上不同进程代码段的起始位置)检查是不是
+   `share_page`: 对于try_to_share的封装, 在当前进程发生缺页时，尝试“借用”其他进程中同一可执行文件的现成物理页，实现页面共享，避免重复从磁盘读入。如果共享成功返回 1；失败返回 0，调用方再走正常的分配页+读盘流程。
+   `do_no_page`: 处理一个任务是不是缺页, 检查这个任务是否是从文件加载的, 不是的话直接分配一个新的页, 是的尝试从别的任务查看是否有相同可执行文件加载得来的, 进行复制, 全部失败以后在从
