## 数据类型

```c
struct pipe_inode_info {
	struct mutex mutex;
	wait_queue_head_t rd_wait, wr_wait;
  // 生产者生产数据的指针 pipe_buffer, 这两个指针在 ring buffer（环形缓冲区）

	unsigned int ring_size;

	unsigned int writers;
	unsigned int files;

	unsigned int w_counter;

	struct fasync_struct *fasync_readers;
	struct fasync_struct *fasync_writers;

	struct watch_queue *watch_queue;
#endif
```

实际的数据存储

```c
struct pipe_buffer {
	struct page *page; // 管道缓冲区的数据所在的页
	unsigned int offset, len; // 数据在页内的偏移和长度
  // 与这个缓冲区相关的操作函数集合, 包括 confirm, release, try_steal, get 等函数
	const struct pipe_buf_operations *ops; 
  // 管道缓冲区的标志, 包括 PIPE_BUF_FLAG_LRU, PIPE_BUF_FLAG_ATOMIC, 
  // PIPE_BUF_FLAG_GIFT, PIPE_BUF_FLAG_PACKET, PIPE_BUF_FLAG_CAN_MERGE, 
  // PIPE_BUF_FLAG_WHOLE 等
	unsigned int flags; 
	unsigned long private;
};
```

## 实现

### 创建

pipe() 和 pipe2() 都是 Linux 用户空间用于创建匿名管道的系统调用，但它们有如下区别：

1.  pipe()

-   原型：int pipe(int pipefd[2]);
-   只能创建最基本的管道，不能直接设置额外的标志。
-   创建后，文件描述符的属性（如 O_NONBLOCK、O_CLOEXEC）需要通过 fcntl() 额外设置。

2.   pipe2()

-   原型：int pipe2(int pipefd[2], int flags);
-   是 pipe() 的增强版，允许在创建管道时直接指定标志（如 O_NONBLOCK、O_CLOEXEC、O_DIRECT 等）。
-   更高效、更安全，避免了竞态条件（race condition），因为标志在创建时就已设置。
