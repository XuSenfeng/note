# UDEV

https://www.freedesktop.org/software/systemd/man/latest/udev.html#

udev 是一个**规则驱动的设备事件管理器**，完全运行在用户空间。它的核心职责是：

-   监听内核发送的设备热插拔事件（uevent）
-   根据预定义的规则处理设备事件
-   动态管理`/dev`目录下的设备节点和符号链接
-   设置设备权限、所有者和组
-   运行自定义脚本或程序响应设备变化

>   是一个根文件系统里面的可执行文件, 内核通过不同的机制触发它们来处理设备事件。

```bash
内核检测到设备变化 → 发送uevent事件（通过netlink套接字）
↓
systemd-udevd主进程接收事件 → 放入事件队列
↓
工作进程处理事件 → 匹配udev规则
↓
执行规则定义的操作（创建设备节点、设置权限、运行脚本等）
↓
更新/dev目录和系统设备状态
```

## 核心组件

-   **systemd-udevd**：守护进程，负责监听和处理设备事件
-   **udevadm**：命令行工具，用于管理和调试 udev
-   **libudev**：库文件，提供应用程序访问设备信息和事件的 API
-   **规则文件**：位于`/etc/udev/rules.d/`和`/usr/lib/udev/rules.d/`

## 常用命令

```bash
# 查看设备的基本信息
udevadm info --name=/dev/sda

# 查看设备的所有属性（用于编写规则）
udevadm info --attribute-walk --name=/dev/ttyUSB0

# 查看设备的sysfs路径
udevadm info --query=path --name=/dev/sda

# --- 监控 ---
# 实时监控所有设备事件
sudo udevadm monitor

# 只监控内核事件
sudo udevadm monitor --kernel

# 只监控udev处理后的事件
sudo udevadm monitor --udev

# 只监控USB子系统的事件
sudo udevadm monitor --subsystem-match=usb

# -- 规则管理 ---
# 重新加载所有udev规则
sudo udevadm control --reload-rules

# 触发所有设备重新应用规则（无需重启）
sudo udevadm trigger

# 触发特定子系统的设备重新应用规则
sudo udevadm trigger --subsystem-match=usb

# 等待所有pending的udev事件处理完成
sudo udevadm settle

# --- 规则测试 ---
# 测试设备是否匹配规则
sudo udevadm test $(udevadm info --query=path --name=/dev/sda)

# 测试特定动作的规则
sudo udevadm test --action=add /sys/class/net/eth0
```

## 规则文件

+   **`/usr/lib/udev/rules.d/`**：系统提供的默认规则，**不要修改**
+   **`/etc/udev/rules.d/`**：用户自定义规则，优先级高于系统规则
+   规则文件名必须以`.rules`结尾，数字前缀决定执行顺序（数字越小，优先级越高）

### 实现规则

udev 规则是单行文本，由**匹配键**（用`==`）和**操作键**（用`=`或`+=`）组成：

```
匹配键1=="值1", 匹配键2=="值2", ..., 操作键1="值1", 操作键2+="值2"
```

#### 匹配键

| 匹配键          | 描述                | 示例                                               |
| --------------- | ------------------- | -------------------------------------------------- |
| `ACTION`        | 设备事件类型        | `ACTION=="add"`、`ACTION=="remove"`                |
| `SUBSYSTEM`     | 设备所属子系统      | `SUBSYSTEM=="tty"`、`SUBSYSTEM=="block"`           |
| `KERNEL`        | 内核设备名          | `KERNEL=="ttyUSB*"`、`KERNEL=="sd*"`               |
| `ATTR{属性名}`  | 设备的 sysfs 属性   | `ATTR{idVendor}=="0403"`、`ATTR{serial}=="123456"` |
| `ATTRS{属性名}` | 父设备的 sysfs 属性 | `ATTRS{idProduct}=="6001"`                         |
| `ENV{变量名}`   | 环境变量            | `ENV{ID_FS_LABEL}=="MY_USB"`                       |

`ACTION=="add"`：设备插入

`ACTION=="remove"`：设备拔出

`ACTION=="change"`：设备状态变化（如磁盘分区变化、网络接口状态变化）

`ACTION=="bind"`：驱动绑定到设备

`ACTION=="unbind"`：驱动从设备解绑

#### 操作键

| 操作键    | 描述           | 示例                                 |
| --------- | -------------- | ------------------------------------ |
| `SYMLINK` | 创建符号链接   | `SYMLINK+="my_usb_serial"`           |
| `OWNER`   | 设置设备所有者 | `OWNER="john"`                       |
| `GROUP`   | 设置设备组     | `GROUP="dialout"`                    |
| `MODE`    | 设置设备权限   | `MODE="0660"`                        |
| `RUN`     | 运行命令或脚本 | `RUN+="/usr/local/bin/backup.sh %k"` |
| `TAG`     | 添加标签       | `TAG+="systemd"`                     |

#### 特殊变量

+   `%k`：内核设备名（如`ttyUSB0`）
+   `%n`：设备编号（如对于`sda1`，`%n`是`1`）
+   `%p`：设备的 sysfs 路径
+   `%N`：设备节点路径（如`/dev/sda1`）

#### 示例

自动处理U盘的挂载

```c
# /etc/udev/rules.d/99-usb-mount.rules
ACTION=="add", SUBSYSTEM=="block", KERNEL=="sd*[0-9]", SUBSYSTEMS=="usb", RUN+="/usr/bin/mount /dev/%k /media/usb"
ACTION=="remove", SUBSYSTEM=="block", KERNEL=="sd*[0-9]", RUN+="/usr/bin/umount /media/usb"
```

#### 获取设备属性

可以使用命令查看一个设备可以使用的属性

```bash
udevinfo -a -p /sys/block/sda
```

## mdev

是一个轻量级的udev, 可以在busybox里面配置使用, Linux System Utilities里面有mdev的配置

在初始化文件`/etc/init.d/rcS`文件里面可以看到

```bash
/bin/mount -t sysfs sysfs /sys
/bin/mount -t tmpfs mdev /dev
# 设置内核热插拔处理程序, 使用/bin/mdev
echo /bin/mdev > /proc/sys/kernel/hotplug
mdev -s # 扫描所有的/sys目录, 里面包含设备号的时候在/dev下面创建对应的节点
```

mdev 通过环境变量中的ACTION 和DEVPATH，来确定此次热插拔事件的动作以及影响了/sys 中的那个目录, 接着会看看这个目录中是否有“dev”的属性文件, 如果有就利用这些信息为这个设备在/dev 下创建设备节点文件

### 功能对比

| 维度             | mdev (BusyBox)                                  | udev (systemd-udevd)                    |
| ---------------- | ----------------------------------------------- | --------------------------------------- |
| **事件监听机制** | 基于`uevent_helper`内核回调                     | 基于`netlink`套接字                     |
| **运行方式**     | 无守护进程，内核每次事件**fork 一个新进程**处理 | 常驻守护进程，主进程 + 工作线程异步处理 |
| **设计目标**     | 最小体积、最快启动、满足基本需求                | 功能完整、高度灵活、可扩展              |
| **典型体积**     | ~50KB (集成在 BusyBox 中)                       | ~500KB+ (独立二进制 + 依赖)             |
| **内存占用**     | 运行时 < 1MB                                    | 运行时 10-50MB                          |

单一配置文件：`/etc/mdev.conf`

语法极简：`设备正则 用户:组 权限 [操作]`

匹配逻辑：**匹配到第一条规则即停止**

支持的操作：

-   设置权限、所有者和组
-   创建符号链接
-   设备插入 / 拔出时执行简单脚本

匹配能力：只能匹配内核设备名，**不能匹配设备属性**（如厂商 ID、序列号）
