# 内存映射

32 位系统的内核空间占用 1G，位于最高处，剩下的 3G 是用户空间。大小用`TASK_SIZE`表示，即0X100000000

## 用户空间

(1) 代码段，包括代码和常量等。

(2) 数据段，包括全局变量等。

(3) 未初始化数据段，例如int赋值0等。

(4) 堆，包括动态分配的内存。

(5) 内存映射区域，把文件区间映射到虚拟地址空间。

(6) 栈，包括局部变量和函数调用。底部存放环境变量和参数字符串。

```bash
+----------------------+ 0xFFFFFFFF (4GB)
| 固定映射区 (Fixmap)  | 用于临时固定映射（kmap_atomic、特殊功能项、调试等）
+----------------------+   FIXADDR_START（接近 4GB）

+----------------------+
| 持久内核映射区       | PKMap：持久内核映射（kmap），用于临时映射高端物理内存页
| (PKMap / kmap)       |   典型：PKMAP_BASE ≈ 0xFE000000
+----------------------+   VMALLOC_END（vmalloc 底部边界）

+----------------------+
| vmalloc / ioremap 区 | 虚拟连续、物理不连续的内核映射区（内核模块、大块缓冲等）
+----------------------+   VMALLOC_START（略高于 lowmem 顶点）

+----------------------+
| 低内存线性映射区     | lowmem / physmap：物理内存 0 ~ max_low_pfn 的线性映射
| (DMA / NORMAL 物理区) |   起始：PAGE_OFFSET = 0xC0000000
+----------------------+   0xC0000000（3GB）

+----------------------+
| 用户空间 (User Space)|
+----------------------+ 0x00000000
```

- **lowmem/physmap**：线性直接映射，内核镜像本身, 网络栈、文件系统缓冲元数据、各种内核数据结构，所有高效小对象分配（kmalloc、slab）都在这里。
- **vmalloc/ioremap**：大块“虚拟连续、物理不连续”的内核映射（模块、大缓冲）。
- **PKMap/kmap**：用于临时/持久地映射高端内存页（highmem 页）的“窗口区”。32 位系统里，>896MB 的物理内存就是 highmem，内核不能一直保持直接映射。当内核需要访问这些 highmem 页时，就用 kmap 在 PKMap 区里“开个窗口”，用完再关
- **Fixmap**：编译期就确定的“常量地址”，映射一些特殊外设/内核结构，访问速度快。

## 虚拟地址

![image-20260417140708104](https://picture-01-1316374204.cos.ap-beijing.myqcloud.com/mac-picture/image-20260417140708104.png)

**TLB：**Translation Lookaside Buffer，MMU 内部的**高速映射缓存**，缓存最近使用的虚拟地址 (VA)→物理地址 (PA) 映射，单周期完成地址转换，是性能关键

**TTW：**Translation Table walk，**页表遍历**，TLB 未命中时硬件自动遍历多级页表获取映射，并将结果写入 TLB，是地址转换的 “慢路径”

## memmap映射

**memmap** 是 Linux 中与物理内存映射相关的概念，包含两重含义：

- **内核参数**：作为启动参数，用于在系统启动时**保留或标记特定物理内存区域**，避免这些区域被内核或用户空间程序占用。
- **内存映射机制**：广义上指内核将物理内存（如设备内存、保留内存）映射到虚拟地址空间的过程，以便内核或用户空间程序直接访问。

### 启动阶段

在系统启动阶段，内核会解析 `memmap` 参数，将指定的物理内存区域标记为“保留”（Reserved），排除在常规内存管理之外。语法如下

```bash
memmap=<size>$<start>  # 保留从 <start> 地址开始、大小为 <size> 的物理内存
```

保留的物理内存不会被内核的伙伴系统（buddy system）分配，需通过以下方式访问：

- **内核空间**：通过 `ioremap()` 函数将物理地址映射到内核虚拟地址，供驱动程序直接读写。
- **用户空间**：通过 `/dev/mem` 设备文件，结合 `mmap()` 系统调用映射物理内存（需 root 权限，且受内核安全限制）。

### 相关信息

- **`/proc/iomem`**：虚拟文件，展示系统物理内存的分配情况（包括保留区域、设备内存等）。
- **`/proc/cmdline`**：查看内核启动参数，验证 `memmap` 是否生效。
- **`ioremap()`**：内核函数，将物理地址映射到内核虚拟地址(定义于`<linux/io.h>`)。

## 代码实现

### 基础数据

内核使用内存描述符`mm_struct`描述进程的用户地址空间,`mm_struct` 包含装入的可执行映像信息以及进程的**第一级页目录指针**`pgd`

![img](https://picture-01-1316374204.cos.ap-beijing.myqcloud.com/lenovo-picture/202604061540733.jpeg)

`mm_struct`中`mmap`字段存放虚拟内存区域链表，`rb_root_mm_rb`字段存放虚拟内存区域红黑树

```c
struct mm_struct
{
    struct vm_area_struct *mmap;            /* VMA 链表 */
    struct rb_root mm_rb;			        /* VMA 红黑树 */
    struct vm_area_struct *mmap_cache;      /* last find_vma result */
    ...
    unsigned long start_code, end_code, start_data, end_data;
    unsigned long start_brk, brk, start_stack;
    ...
};
```

Linux将虚拟内存组织成一些**段**的集合。`vm_area_struct`代表进程的一个虚拟地址区间，即**段**

```c
struct vm_area_struct
{
    struct mm_struct *vm_mm;        /* The address space we belong to. */
    // 虚拟地址的起始以及结束的位置
    unsigned long vm_start;         /* Our start address within vm_mm. */
    unsigned long vm_end;           /* The first byte after our end address
                                           within vm_mm. */
    ....
    /* linked list of VM areas per task, sorted by address */
    struct vm_area_struct *vm_next;
    ....
    /* describe the permissable operation */
    unsigned long vm_flags;
    /* operations on this area */
    const struct vm_operations_struct *vm_ops;
    
    struct file * vm_file; /* File we map to （can be NULL）. */
} ;
```

里面的`vm_operations_struct`描述了在这个区间的操作

+ `open`: 开启
+ `close`: 关闭
+ `nopage`: 没有页的时候的操作

### 实现原理

![img](https://pic3.zhimg.com/v2-3cb981902c1e4984d906128b3951c30a_1440w.jpg)

mmap函数就是要创建一个新的vm_area_struct结构，并将其与文件的物理磁盘地址相连。一个vm_area_struct映射到一组连续的页表项。页表项又指向物理内存page，这样就把一个文件和物理内存页相映射。

如果没有匹配到，就报段错误，访问了一个没有分配的虚拟地址；如果匹配到了vm_area_struct，根据虚拟地址和页表的映射关系，找到对应的页表项PTE，如果PTE没有分配，就报一个缺页异常，去加载相应的文件数据到物理内存，如果PTE分配，就去相应的物理页的偏移位置读取数据

所以最终要初始化的是`vm_operations_struct`

```c
struct vm_operations_struct {
	void (*open)(struct vm_area_struct * area);
	void (*close)(struct vm_area_struct * area);
    // 当VMA被分割时调用, 比较少使用
	int (*split)(struct vm_area_struct * area, unsigned long addr);
    // 当VMA被重新映射时调用
	int (*mremap)(struct vm_area_struct * area);
    // 处理页面错误（page fault），当用户访问未映射的虚拟地址时调用
    // 这是mmap设备驱动的核心函数。您需要在这里分配物理页面、
    // 映射设备内存（如IO内存或DMA缓冲区），并返回页面信息
	vm_fault_t (*fault)(struct vm_fault *vmf);
	vm_fault_t (*huge_fault)(struct vm_fault *vmf,
			enum page_entry_size pe_size);
    // 批量映射多个页面，避免逐个调用 fault。
	vm_fault_t (*map_pages)(struct vm_fault *vmf,
			pgoff_t start_pgoff, pgoff_t end_pgoff);
	unsigned long (*pagesize)(struct vm_area_struct * area);

	/* notification that a previously read-only page is about to become
	 * writable, if an error is returned it will cause a SIGBUS */
    // 当只读页面即将变为可写时调用
	vm_fault_t (*page_mkwrite)(struct vm_fault *vmf);

	/* same as page_mkwrite when using VM_PFNMAP|VM_MIXEDMAP */
	vm_fault_t (*pfn_mkwrite)(struct vm_fault *vmf);

	/* called by access_process_vm when get_user_pages() fails, typically
	 * for use by special VMAs that can switch between memory and hardware
	 */
	int (*access)(struct vm_area_struct *vma, unsigned long addr,
		      void *buf, int len, int write);

	/* Called by the /proc/PID/maps code to ask the vma whether it
	 * has a special name.  Returning non-NULL will also cause this
	 * vma to be dumped unconditionally. */
	const char *(*name)(struct vm_area_struct *vma);

#ifdef CONFIG_NUMA
	/*
	 * set_policy() op must add a reference to any non-NULL @new mempolicy
	 * to hold the policy upon return.  Caller should pass NULL @new to
	 * remove a policy and fall back to surrounding context--i.e. do not
	 * install a MPOL_DEFAULT policy, nor the task or system default
	 * mempolicy.
	 */
	int (*set_policy)(struct vm_area_struct *vma, struct mempolicy *new);

	/*
	 * get_policy() op must add reference [mpol_get()] to any policy at
	 * (vma,addr) marked as MPOL_SHARED.  The shared policy infrastructure
	 * in mm/mempolicy.c will do this automatically.
	 * get_policy() must NOT add a ref if the policy at (vma,addr) is not
	 * marked as MPOL_SHARED. vma policies are protected by the mmap_lock.
	 * If no [shared/vma] mempolicy exists at the addr, get_policy() op
	 * must return NULL--i.e., do not "fallback" to task or system default
	 * policy.
	 */
	struct mempolicy *(*get_policy)(struct vm_area_struct *vma,
					unsigned long addr);
#endif
	/*
	 * Called by vm_normal_page() for special PTEs to find the
	 * page for @addr.  This is useful if the default behavior
	 * (using pte_page()) would not find the correct page.
	 */
	struct page *(*find_special_page)(struct vm_area_struct *vma,
					  unsigned long addr);

#ifdef CONFIG_SPECULATIVE_PAGE_FAULT
	bool (*allow_speculation)(void);
#endif
};
```

虚拟地址空间已分配，但**物理页面尚未映射**。这意味着映射区域是“虚拟的”，访问它会立即触发页面错误（page fault），除非您在 `mmap` 中预先映射了一些页面

+ 一个简单的初始化示例

```c
// 结构体包含错误信息，如虚拟地址、VMA指针等
static vm_fault_t my_fault(struct vm_fault *vmf) {
    struct page *page = alloc_page(GFP_KERNEL);  // 或从设备获取
    if (!page) return VM_FAULT_OOM;
    vmf_insert_page(vmf->vma, vmf->address, page);
    return VM_FAULT_LOCKED;  // 成功
}

static const struct vm_operations_struct my_vm_ops = {
    .open = my_open,
    .close = my_close,
    .fault = my_fault,  // 核心：处理页面映射
    // 其他函数根据需要设置
};

static int my_mmap(struct file *file, struct vm_area_struct *vma) {
    // 设置VMA属性，如 vm_flags |= VM_IO;
    vma->vm_ops = &my_vm_ops;
    my_open(vma);  // 可选：立即调用open
    return 0;
}
```

## 静态映射

在将Linux 移植到目标电路板的过程中，通常会建立外设I/O 内存物理地址到虚拟地址的静态映射，这个映射通过在电路板对应的 `map_desc`

静态内存映射是在**内核启动早期 (MMU 开启前或初始化阶段)** 建立的**固定虚拟地址→物理地址映射**，主要用于：

-   映射处理器内部寄存器和片上外设
-   映射物理内存区域 (如 RAM、ROM)
-   为内核提供固定的硬件访问地址

这种映射在系统运行期间**通常不会改变**，与动态映射 (ioremap) 形成互补。

在 ARM 平台 Linux 移植中，板级支持包 (BSP) 会定义一个`struct map_desc`数组，描述该板卡所有需要静态映射的硬件资源

```c
struct map_desc {
	unsigned long virtual; // 虚拟地址
	unsigned long pfn; // 物理地址转PFN
	unsigned long length; // 映射的长度
	unsigned int type; // 内存的类型
};
```

### 实际的映射

```c
// 批量映射
void __init iotable_init(struct map_desc *io_desc, int nr);
// 单个映射
int create_mapping(struct map_desc *md);
```

#### 使用示例

```c
static struct map_desc myboard_io_desc[] __initdata = { ... };

// 2. 板级映射初始化函数
void __init myboard_map_io(void)
{
    // 调用iotable_init创建所有静态映射
    iotable_init(myboard_io_desc, ARRAY_SIZE(myboard_io_desc));
}

// 3. 注册映射初始化函数
MACHINE_START(MYBOARD, "My Development Board")
    .map_io     = myboard_map_io,
    // 其他板级信息...
MACHINE_END
```

在使用设备树以后, 信息记录在设备树里面, 保留场景：内核启动**最早期阶段**（MMU 初始化、早期串口调试、SCU 等关键硬件访问），这些必须在设备树解析前完成的静态映射
