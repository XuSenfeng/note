# Bresenham 直线算法

在**离散栅格 / 像素点阵**上，用整数运算**逼近一条理想直线**，是计算机光栅绘图、机器人栅格地图轨迹采样的经典算法，DWA 路径碰撞检测、A * 栅格遍历里经常用它求两点连线经过哪些栅格

![image.png](https://picture-01-1316374204.cos.ap-beijing.myqcloud.com/lenovo-xiangong/20260604112033875.png?imageSlim)

假设我们要绘制如上图中所示直线$f(x, y)$，它的起点在$(x_k​,y_k​)$，上图中的每一个圆圈是一个led灯

我们的起点和终点都是整数，并且直线的斜率 $0<m<1$， 毫无疑问，我们首先点亮$(x_k, y_k)$这个led灯

那么下一个led灯，我们应该点亮哪一个呢？观察上图可以很容易发现，我们应该点亮$(x_k+1, y_k)$和$(x_k+1, y_k+1)$中的一个，那究竟选择哪个呢？一个很朴素的想法是，谁在y方向上距离直线最近，那就点亮谁。这个想法也正是布雷森汉姆直线算法的思想

对于上图，直线与$(x_k+1, y_k)$和$(x_k+1, y_k+1)$连线的交点，如果在线段一半的上面，那么就应该选择点亮$(x_k+1, y_k+1)$，否则点亮$(x_k+1, y_k)$

那么，我们只需要将中点坐标$(x_k+1, y_k+\frac{1}{2})$坐标带入直线函数f(x,y)中:
+ 如果$f(x_k+1, y_k+\frac{1}{2}) > 0$，则说明直线的交点在$(x_k+1, y_k+\frac{1}{2})$的下面，此时需要点亮$(x_k+1, y_k)$
+ 如果$f(x_k+1, y_k+\frac{1}{2}) < 0$，则说明直线的交点在$(x_k+1, y_k+\frac{1}{2})$的上面，此时需要点亮$(x_k+1, y_k+1)$

## 误差分析

![image.png](https://picture-01-1316374204.cos.ap-beijing.myqcloud.com/lenovo-xiangong/20260604112844927.png?imageSlim)
是我们想要绘制的线，它的斜率是 $0<m<1$，现在考虑在x点，直线对应纵轴的值 $y+\epsilon$。
![image.png](https://picture-01-1316374204.cos.ap-beijing.myqcloud.com/lenovo-xiangong/20260604113342195.png?imageSlim)

所以(x,y)点距离直线更近，所以把该点点亮。一旦选择了(x,y)点，那么此时到直线就产生了误差：$\epsilon$

那么当我们绘制下一个点时，其横坐标是x+1，它对应的纵坐标是 $y+m+\epsilon$， 此时我们需要点亮(x+1, y+1)对应的灯， 此时误差就变成了

![image.png](https://picture-01-1316374204.cos.ap-beijing.myqcloud.com/lenovo-xiangong/20260604113516904.png?imageSlim)

伪代码

```python
 def line(x0, x1, y0, y1):
     deltax = x1 - x0
     deltay = y1 - y0
     error = 0
     deltaerr = deltay / deltax
     int y = y0
     for x from x0 to x1
         plot(x,y)
         error = error + deltaerr
         if abs (error) ≥ 0.5 then
             y = y + 1
             error = error - 1.0
```

## 其他斜率

上面介绍的布雷森汉姆直线算法，是在第一象限，并且斜率$0<m<1$时的情况。如果超过了这个限制，那么上面的过程就不能直接应用了，必须要做一些改进

![image.png](https://picture-01-1316374204.cos.ap-beijing.myqcloud.com/lenovo-xiangong/20260604113831596.png?imageSlim)

```python
from matplotlib import pyplot as plt
from matplotlib.pyplot import MultipleLocator


def line(x0, y0, x1, y1):
    x = []
    y = []
    steep = abs(y1 - y0) > abs(x1 - x0)
    if steep:  # 如果为真，说明斜率绝对值大于1，则主要以y方向递增
        x0, y0 = y0, x0
        x1, y1 = y1, x1

    if x0 > x1:  # 如果为真，说明起点大于终点，此时则交换方向
        x0, x1 = x1, x0
        y0, y1 = y1, y0

    delta_x = x1 - x0
    delta_y = abs(y1 - y0)
    error = 0
    delta_error = delta_y / delta_x
    yk = y0

    if y0 < y1:
        y_step = 1
    else:
        y_step = -1

    for xk in range(x0, x1):
        if steep:
            x.append(yk)
            y.append(xk)
        else:
            x.append(xk)
            y.append(yk)

        error = error + delta_error
        if error >= 0.5:
            yk = yk + y_step
            error = error - 1.0
    return x, y


if __name__ == '__main__':
    x, y = line(30, 30, 4, -20)

    plt.xlim((-30, 30))
    plt.ylim((-30, 30))
    plt.grid()
    plt.plot(x, y)

    y_major_locator = MultipleLocator(1)
    x_major_locator = MultipleLocator(1)
    ax = plt.gca()
    ax.xaxis.set_major_locator(x_major_locator)
    ax.yaxis.set_major_locator(y_major_locator)
    plt.show()
```
