排查 EoE 和 CoE 的数据混在一起

![image.png](https://picture-01-1316374204.cos.ap-beijing.myqcloud.com/lenovo-xiangong/20260624181401369.png?imageSlim)

```
EoE(Boot Request from 00:11:22:33:44:0a)[Malformed Packet]   ← EoE 的流量 EoE 句柄正在给虚拟网卡 eoe0s1 要 IP
Mbx(CoE SDO Req : 'Initiate Download' (1) Idx=0x1600 Sub=4)   ← CoE 的流量 主站发的 SDO 写（下载）
```

主站发一条 CoE SDO 写后，会**等从站回 CoE 应答**。等不到 → 超时 → 重发。这里连发 4 次都没成功，原因就是每次的**应答都被 EoE 占着的邮箱顶掉了**（每帧都挂着 `EoE(Boot Request)[Malformed]`，说明邮箱里塞的是 EoE BOOTP，不是主站要的 CoE 应答）。

- Wireshark 在这帧的**邮箱里解出了 EoE 的 BOOTP 内容**，于是用 BOOTP 的地址（0.0.0.0→广播）填进 Source/Dest 列，协议标成 `EoE-BOOTP`。
- 但 Info 末尾又有 `Mbx(CoE SDO Req ... 0x1600 Sub=4)`——说明**这帧里还夹着 CoE 的 SDO 写**。
- `[Malformed Packet]` = 按 EoE 解时解不通 → 邮箱里协议串了。
- **一句话：这类行 = 邮箱被 EoE 内容占着，同时主站又想写 CoE。**
