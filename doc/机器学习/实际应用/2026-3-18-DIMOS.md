# DIMOS

AI智能控制机器人的系统

## 目录

+   dimos
+   +   Agents（智能体相关）负责 LLM 智能体、技能容器、推理、MCP 客户端等。通过 @skill 装饰器暴露物理技能，供 LLM 调用。
    +   Core（核心框架）
    +   Control（控制模块）
    +   Hardware（硬件抽象）
    +   Manipulation（操作/机械臂）
    +   Mapping（建图/地图）
    +   Memory（记忆/存储）
    +   Models（模型/AI相关）存放 AI 推理相关模型，如视觉、语言、嵌入等。被 perception、agents 等模块调用。
    +   Msgs（消息类型/通信）
    +   Navigation（导航/路径规划）
    +   Perception（感知/识别）负责视觉、激光、语音等多模态感知。通过流式接口将感知结果传递给 agent、navigation、manipulation 等模块。
    +   Protocol（协议/通信层）封装 LCM、ROS2、DDS 等底层通信协议。为模块间数据流提供传输保障。
    +   Robot（机器人本体/蓝图）定义具体机器人（如 Go2、G1、xArm 等）的蓝图和配置。组合各功能模块形成可运行的机器人系统。
    +   RxPyBackpressure（流控/反压）
    +   Simulation（仿真）
    +   Skills（技能/动作）
    +   Spec（接口/协议规范）
    +   Stream（数据流/通信）
    +   Teleop（远程操作/遥控）
    +   Types（类型定义）
    +   Utils（工具/通用函数）
    +   Visualization（可视化）
    +   Web（Web接口/服务）
    +   Assets（资源/配置）
    +   Exceptions（异常处理）
    +   E2ETests（端到端测试）

## 概念

### Module

模块是机器人上的自治子系统，通过标准化消息与其他子系统通信。

一些模块示例：

-   摄像头（输出图像）
-   导航（输入地图和目标，输出路径）
-   检测（输入图像和视觉模型如 YOLO，输出检测流）

```bash
┌┴─────────────┐
│ CameraModule │
└┬─────────────┘
 ├─ color_image: Image
 ├─ camera_info: CameraInfo
 │
 ├─ RPC start()
 ├─ RPC stop()
 │
 ├─ Skill take_a_picture
```

输出两个流, 两个生命周期RPC, 一个skill

### 蓝图

蓝图（`_BlueprintAtom`）是初始化 `Module` 的说明书。

通常你不会只运行一个模块，所以多个蓝图会一起被 `Blueprint` 管理。

### 技能

```python
from dimos.core.core import rpc
from dimos.core.module import Module
from dimos.agents.annotation import skill
from dimos.core.global_config import GlobalConfig

class SomeSkill(Module):

    @skill
    def some_skill(self) -> str:
        """供 LLM 使用的技能描述。"""
        return "result"
```

有两种skill对应`dimos/skills`以及`dimos/agent/skills`两个文件夹

+   `dimos/skills`是比较底层的skills, 里面的是一下比较底层的控制, 比如抓取机械臂操作等具体动作或感知能力
+   `dimos/agent/skills`这里的 skill 更偏向于“高层智能、任务编排、对话交互、抽象能力”的实现。它们通常作为 agent 的“工具”暴露给大模型
