# 贝尔曼最优公式

是贝尔曼公式的一个特殊情况, 在计算的时候, 某一个位置可以多个不同的action Value, 可以选择Action Value最大的动作

不同$\pi$的可以采集到不同的state value, 如何某一个$\pi$获得的v是最大的, 对应的策略是最优的

> 需要考虑的这个策略是否存在, 是不是唯一的, 是确定的还是非确定的以及如何得到

<img src="https://picture-01-1316374204.cos.ap-beijing.myqcloud.com/lenovo-picture/202602061717706.png" alt="image-20260206171723669" style="zoom:150%;" />

现在这个里面的$\pi$以及两个v是需要求解的, 一般的贝尔曼公式是给定的, 矩阵形式的公式为
$$
v = max_{\pi}(r_{\pi} + \gamma P_{\pi}v)
$$
之后就是如何获取这个公式的解, 这个公式可以进一步演化为

<img src="https://picture-01-1316374204.cos.ap-beijing.myqcloud.com/lenovo-picture/202602061730636.png" alt="image-20260206173044592" style="zoom:150%;" />

可以获取到, 在$q(s, a)$最大的时候使得$\pi(a|s)$的值为1即可获取到最大值$max_{a\in A} q(s, a)$, 可以进一步简化这个公式

![image-20260206174050897](https://picture-01-1316374204.cos.ap-beijing.myqcloud.com/lenovo-picture/202602061740951.png)

从而获取到$v = f(v)$, 进一步计算这个公式需要使用收缩映射定理

### 收缩映射定理

在一个函数里面如果有一个点$f(x^{*}) = x^{*}$, 这个点是一个固定点

同时如果一个函数满足$\|f(x_{1}) - f(x_{2}\| ) \leqslant \gamma \|x_{1} - x_{2} \|$即可获取到几个性质

+ 这个函数里一定有固定点
+ 这个点是唯一存在的
+ 可以通过迭代的方式获取到$x_{k+1} = f(x_{k})$

### 应用

贝尔曼最优公式是一个contraction mapping, 所以可以使用之前的定理求解出来$v = f(v)$的收敛值
$$
v_{k+1} = f(v_{k}) = max_{\pi \in \Pi}(r_{\pi} + \gamma P_{\pi}v_{k})
$$
使用这个公式迭代即可获取到$\pi ^{*}$, 带回可以获取到$v^{*} = r_{\pi^{*}} + \gamma P_{\pi ^{*}}v^{*}$, 对应使用这个策略的特殊的贝尔曼公式, 在这个位置计算出来的$v^{*}$是最大的, 为最优的策略

## 示例

首先，给每个状态**随机分配一个初始动作**（也可以手动设定简单策略）。

以你图中的 5×5 网格为例，我们可以设定一个初始策略 π0：

- 所有普通状态的初始动作：**向右**（箭头指向右）
- 目标状态（蓝色格子）和禁止状态（橙色格子）：无动作（终止状态）

解**贝尔曼期望方程**，计算当前策略 π 下每个状态的期望累积折扣奖励, 对于有限 MDP（如网格世界），我们用**迭代法**计算，直到 $V^{\pi}$ 收敛

#### 具体操作：

1. **初始化状态值**：所有状态的初始值 $V_{0}(s)=0$
2. 迭代更新：对每个状态 s，用当前策略 π 计算新值：$V_{k+1}(s)=R(s,π(s),s′)+γVk(s′)$
    - 例如，初始策略 π0 下，状态 (1,1) 的动作是 “向右”，移动到 (1,2)，奖励 R=0，因此：$V_{k+1}(1,1)=0+0.9×V_{k}(1,2)$
    - 目标状态 (4,3) 是吸收态，进入后获得奖励 R=1 且不再转移，因此：$V_{π}(4,3)=1+0.9×V_{π}(4,3)⟹V_{π}(4,3)=\frac{1}{1−0.9}=10$
3. **收敛判定**：当所有状态值的变化小于阈值（如$ ∣V_{k+1}(s)−V_{k}(s)∣<10^{−4}）$时，停止迭代，得到 Vπ。

<img src="https://picture-01-1316374204.cos.ap-beijing.myqcloud.com/lenovo-picture/202602061835855.png" alt="2a8a87df-6683-4588-8d69-9eaebbd7c0f1" style="zoom:150%;" />

在这个图片里面, 由于$\gamma$比较大, 所以这个策略比较远视, 采取穿过墙的方式到达目的地, 到达以后并不会停止, 而是循环获取激励, 右侧是根据这个最优策略, 计算出来的state value

![image-20260206183734808](https://picture-01-1316374204.cos.ap-beijing.myqcloud.com/lenovo-picture/202602061837866.png)

使用比较小的$\gamma$的时候, 可以获取到一个比较近视的策略, 注重下一步是不是会被罚

使用的所有激励值变为$ar + b$的时候不会影响路径的规划
