# OTA

Flipper 不能直接在运行固件时改写自身 Flash，所以设计了**RAM 启动更新模式**: 

1.   把专用**更新器镜像**加载到 RAM 运行
2.   RAM 里的更新器拥有**整个 Flash 完整写入权限**
3.   能安全更新**主固件 + 蓝牙协处理器（Core2）**，全程校验，失败可重试

## 实际流程

1.   备份内部存储 /int

+   把系统配置、用户数据打包成 tar 存 SD 卡，防止重分区丢数据

2.   执行设备更新（核心）

-   检查并更新蓝牙无线栈（Radio stack）
-   校验修复底层 Option Bytes
-   校验 CRC 后刷入 .dfu 固件文件

3.   恢复数据 + 更新资源

重启进新固件，还原 /int 数据，释放资源包到 SD 卡

## 使用

1. 完整包（固件 + 蓝牙栈 + 资源，推荐）

```bash
./fbt COMPACT=1 DEBUG=0 updater_package
```

2. 最小包（仅固件，精简用）

```bash
./fbt COMPACT=1 DEBUG=0 updater_minpackage
```

3. 自定义蓝牙栈（换完整版 BLE）

```bash
./fbt updater_package COMPACT=1 DEBUG=0 COPRO_OB_DATA=scripts/ob_custradio.data COPRO_STACK_BIN=stm32wb5x_BLE_Stack_full_fw.bin COPRO_STACK_TYPE=ble_full
```

生成的包在 `dist/` 目录，可用 qFlipper 或 SD 卡刷机
