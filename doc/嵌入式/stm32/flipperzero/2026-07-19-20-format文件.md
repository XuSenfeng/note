# format文件

Flipper 里有一个文本配置文件读写助手，不是通用二进制序列化库。它专门处理一种很简单的“键: 值”格式，支持注释、空白字符、版本头，以及几类常见数据类型。
下面是一个简单的示例

> 库所在的路径`lib/flipper_format`
```txt
Version: 1
# Example file for flipper_format
Name: Hello Flipper
Value: 42
Enabled: true false
Hex: DE AD BE EF
```

## 使用

一个典型写文件流程就是：先写 Filetype 和 Version，再写各字段。一个典型读文件流程就是：先读头，校验类型和版本，再按 key 读值。

```c
Storage* storage = furi_record_open(RECORD_STORAGE);  
FlipperFormat* ff = flipper_format_file_alloc(storage);

if(flipper_format_file_open_always(ff, path)) {  
	flipper_format_write_header_cstr(ff, "My File", 1);
	
	flipper_format_write_string_cstr(ff, "Name", "abc");
	
	uint32_t value = 42;
	flipper_format_write_uint32(ff, "Value", &value, 1);
}

flipper_format_free(ff);  
furi_record_close(RECORD_STORAGE);
```
> Storage 是“存储服务”的句柄，负责文件系统、目录、文件打开/关闭、创建删除、路径管理这些底层操作。
> FlipperFormat 是建立在 Storage 之上的“文本配置文件读写助手”，专门读写 Flipper 这种 `Key: Value` 格式文件。它不直接管磁盘，而是借助 Storage 提供的文件能力来打开文件、顺序读取、写入、更新和删除字段。
```c
Storage* storage = furi_record_open(RECORD_STORAGE);  
FlipperFormat* ff = flipper_format_file_alloc(storage);  
FuriString* tmp = furi_string_alloc();  
uint32_t version = 0;

if(flipper_format_file_open_existing(ff, path)) {  
	if(flipper_format_read_header(ff, tmp, &version)) {  
		flipper_format_read_string(ff, "Name", tmp);  
		flipper_format_read_uint32(ff, "Value", &value, 1);  
	}  
}

furi_string_free(tmp);  
flipper_format_free(ff);  
furi_record_close(RECORD_STORAGE);
```

- flipper_format_file_alloc 只是分配对象，不等于打开文件；还要再调用 open_existing、open_always、open_new 之类的函数。
- 读字符串时用 FuriString 接收；写字符串既可以用 FuriString，也可以直接用 C 字符串版本。
- 数组类型都要传长度，函数会按元素个数读写。
