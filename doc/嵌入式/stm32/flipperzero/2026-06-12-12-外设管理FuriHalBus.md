# 外设管理

系统如何统一管理 STM32 上各类外设的"使能 / 失能 / 复位",以及哪些外设由系统接管、哪些必须由用户代码自己负责

FuriHalBus 的本质是一个**外设生命周期登记簿**:

1. 通过强约束的 enable/disable/reset 三件套,杜绝"重复打开"、"忘记关闭"这类静默 bug —— 一旦违反直接崩溃,迫使开发者发现问题。
2. 通过文档里的三张表,明确每个外设的归属(系统独占 / 系统按需 / 用户负责),避免用户代码与系统功能争抢同一硬件资源。
3. 用户代码写新驱动时的决策路径:**先查表 → 有 API 就用 API → 没 API 才用 `furi_hal_bus_*` 自己管**。

## 设计理念

系统启动时,**大多数外设默认处于复位状态、不提供时钟**。这样做有两个目的:

- 降低功耗;
- 保证每次访问前外设都处于一致的初始状态。

只有少数关键外设由系统在启动时直接打开,其他外设必须由用户代码显式启用

## 核心API

| API                      | 前置条件                 | 作用                                 |
| ------------------------ | -------------------- | ---------------------------------- |
| `furi_hal_bus_enable()`  | 外设当前**必须是 disabled** | 启用外设(连续 enable 两次会崩溃), 参数是一个enum类型 |
| `furi_hal_bus_disable()` | 外设当前**必须是 enabled**  | 失能外设                               |
| `furi_hal_bus_reset()`   | 外设当前**必须是 enabled**  | 把寄存器全部复位回初值,但不失能                   |
## 外设分类

### 默认使能

这类是系统自身运行的基础设施， 如果有对应的 API,必须走 API 访问,不要直接操作寄存器

- `DMA1/2`、`DMAMUX` —— 在 furi_hal_dma.c 中启用
- `GPIOA~H` —— 在 furi_hal_resources.c 中启用
- `PKA` / `AES2` / `HSEM` / `IPCC` —— 蓝牙协处理器相关,在 furi_hal_bt.c 中启用
- `FLASH` —— 由硬件直接使能

### 系统按需要开启

有现成的高层封装,**优先使用 API**,API 内部会自己处理 enable/disable

|外设|API|
|---|---|
|RNG|`furi_hal_random.h`|
|SPI1/SPI2|`furi_hal_spi.h`|
|I2C1/I2C3|`furi_hal_i2c.h`|
|USART1/LPUART1|`furi_hal_serial.h`|
|USB|`furi_hal_usb.h`|

### 用户自己负责

默认不开,使用前必须 `furi_hal_bus_enable()`,用完 `furi_hal_bus_disable()`。包括 `CRC`、`TSC`、`ADC`、`QUADSPI`、各类 `TIM`、`LPTIM`、`SAI1`、`LCD` 等

| Peripheral     | System     | Purpose                                |                       |
| -------------- | ---------- | -------------------------------------- | --------------------- |
| CRC            |            |                                        |                       |
| TSC            |            |                                        |                       |
| ADC            |            |                                        |                       |
| QUADSPI        |            |                                        |                       |
| TIM1           | yes        | subghz, lfrfid, nfc, infrared, etc...  |                       |
| TIM2           | yes        | subghz, infrared, etc...               |                       |
| TIM16          | yes        | speaker                                |                       |
| TIM17          | yes        | cc1101_ext                             |                       |
| LPTIM1         | yes        | tickless idle timer                    |                       |
| LPTIM2         | yes        | pwm                                    |                       |
| SAI1           |            |                                        |                       |
| LCD            |            |                                        |                       |

"System"列标 `yes` 的那些 —— 例如 `TIM1`(SubGhz / LFRFID / NFC / Infrared)、`TIM2`(SubGhz / Infrared)、`TIM16`(扬声器)、`LPTIM1`(tickless idle 定时器)、`LPTIM2`(PWM)—— 系统在使用对应功能时**会临时接管**这个外设。也就是说同一个外设在不同时间可能被系统或用户代码占用,需要避免冲突

### DMA

DMA1/2 比较特殊:每个 DMA 控制器有多个独立通道,系统已经占用了一部分

- **DMA1** 通道 1/2 用于数字信号;通道 4 用于脉冲读取(pulse reader);通道 6/7 用于 USART_Rx / LPUART_Rx。
- **DMA2** 通道 1/2 给 infrared、lfrfid、subghz;通道 3/4/5 给 cc1101_ext;通道 6/7 给 SPI。
- 仅 **DMA1 通道 3 和 5** 是空的,留给用户使用。
