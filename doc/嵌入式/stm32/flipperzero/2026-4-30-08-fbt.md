# fbt

fbt是用于flipperzero构建的工具, 是基于https://scons.org/创建的

## 环境变量

这个文件实际是一个脚本文件, 默认的环境变量不会影响全局的变量, 但是可以使用命令创建一个包含环境变量的shell

>   -   Windows 下，直接运行 [fbtenv.cmd](vscode-file://vscode-app/Applications/Visual Studio Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html)。
>   -   Linux 和 MacOS 下，在新 shell 里运行 `source scripts/toolchain/fbtenv.sh`。
>   -   你也可以在 shell 里输入 `. `./fbt -s env` `（注意开头的点）。

如果你的系统不支持预编译工具链，或你想用自定义依赖版本，可以设置 `FBT_NOENV=1`。此时 `fbt` 会跳过工具链和环境配置，要求所有工具都在系统 PATH 中可用。

如果设置了 `FBT_TOOLCHAIN_PATH` 变量，`fbt` 会将工具链解压到该目录。默认会下载到仓库根目录下的 `toolchain` 子目录。

如需为 `fbt` 及工具链管理脚本启用额外调试输出，可以设置 `FBT_VERBOSE=1`

fbt` 启动时总会执行 `git submodule update --init`，除非你在环境变量中设置 `FBT_NO_SYNC=1

>   -   Windows 下，在运行 `fbt` 的 shell 里设置 `set "FBT_NO_SYNC=1"`
>   -   *nix 下，使用 `$ FBT_NO_SYNC=1 ./fbt ...`

## 构建

`fbt` 会在 `build` 目录下为 updater 和固件分别创建子目录，名称取决于优化设置（`COMPACT` 和 `DEBUG` 选项）。为方便 IDE 集成，最新构建变体的目录总是链接为 `build/latest`。此外，`compile_commands.json` 也会生成在该文件夹（用于 IDE 代码补全）。

只有*固件构建目标*会更新 `build/latest` 软链接和编译数据库。运行其他任务（如固件烧录或为不同配置/硬件目标构建更新包）不会更新 `build/latest`

## scons使用

需要创建文件`SConstruct`用于编译

SCons 支持的编译类型有：

+   Program： 编译成可执行程序（在 Windows 平台上即是 exe 文件），这是最常用的一种编译类型。
+   Object： 只编译成目标文件。使用这种类型，编译结束后，只会产生目标文件。在 POSIX 系统中，目标文件以 .o 结尾，在 Windows 平台上以 .OBJ 结尾。
+   Library： 编译成库文件。SCons 默认编译的库是指静态链接库。
+   StaticLibrary： 显示的编译成静态链接库，与上面的 Library 效果一样。
+   SharedLibrary： 在 POSIX 系统上编译动态链接库，在 Windows 平台上编译 DLL。

### 初级使用

```bash
Program("hello.c")
```

这个C文件就是使用的代码文件, 在这里，没有指定生成的可执行程序的名字，SCons 会把源代码 文件名字的后缀去掉，用来作为可执行文件的名字

### 各种参数

+ 可执行文件的名字

```python
Program('myhello', 'helloscons.c')
```

+   清理

不需要像 Makefile 那样指定清理的动作，就可以执行清理任务。在 SCons 中，执行清理任务由参数 -c 指定

```bash
scons -c
```

+   简化输出

```bash
scons -Q
```

+   复杂编译

如果你的项目由多个源文件组成，而且你想指定一些编译的宏定义，以及显式的指定使用某些库

```python
Program('helloscons2', ['helloscons2.c', 'file1.c', 'file2.c'], 
       LIBS = 'm', 
       LIBPATH = ['/usr/lib', '/usr/local/lib'], 
       CCFLAGS = '-DHELLOSCONS')
```

如果你的源程序代码文件很多，有十几个甚至上百个，那不要一个个的将他们都列出来，你可以使用 glob(’*.c’) 来代替源代码列表

>   +   `LIBS`： 显示的指明要在链接过程中使用的库，如果有多个库，应该把它们放在一个列表里面。这个例子里，我们使用一个称为 m 的库。
>   +   `LIBPATH`： 链接库的搜索路径，多个搜索路径放在一个列表中。这个例子里，库的搜索路径是 /usr/lib 和 /usr/local/lib。
>   +   `CCFLAGS`： 编译选项，可以指定需要的任意编译选项，如果有多个选项，应该放在一个列表中。这个例子里，编译选项是通过 -D 这个 gcc 的选项定义了一个宏 HELLOSCONS。
>   +   `CPPPATH`：指定头文件的路径

+   指定环境变量

```bash
env = Environment(CC = 'g++')
env.Program("client", "client.c", LIBS = 'm', CPPPATH = '../include', CCFLAGS = '-std=c++11')
```
