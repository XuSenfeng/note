# 启动流程

启动的汇编文件在`lib/stm32wb_cmsis/Source/Templates/gcc/startup_stm32wb55xx_cm4.s`
这个文件里面有中断向量表, 在里面配置了复位函数`Reset_Handler`

实际的复位以后运行的代码在`targets/f7/src/stm32wb55_startup.c`文件里面

```c
/** Start your journey here */

FURI_NAKED void Reset_Handler(void) {
// Funny thing: SP and MSP are set to _stack_end if we came here after MCU reset
// Now, what if we came from boot loader? Lets set SP to _stack_end again.
// By the way Furi stage loader doing it too, but we don't know who called us.
asm volatile("ldr r0, =_stack_end");
asm volatile("mov sp, r0"); // 配置栈指针

// ST chip initialization routine
SystemInit();

// Copy data section from flash
// 真正把 .data 初值从 Flash 搬到 RAM 的
memcpy((void*)&_sdata, &_sidata, &_edata - &_sdata);
  
// Wipe BSS 清空bss段
memset((void*)&_sbss, 0x00, &_ebss - &_sbss);
  
// Core2 related quirks: wipe MB_MEM2 section
memset((void*)&_sMB_MEM2, 0x00, &_eMB_MEM2 - &_sMB_MEM2);
  
// libc init array, 使用c++的时候, 调用全局变量构造函数之类的初始化配置
__libc_init_array();
  
// Our main
main();
  
// You should never exit from main, but we'll catch you if you do
furi_crash("WhyExit?");
}
```

对应的链接脚本

把程序中「有初始值的全局变量 / 静态变量」+「需要在 RAM 运行的函数」，编译后存放在 Flash（只读存储），程序运行时加载到 RAM（读写内存），同时做内存对齐、标记段的起止地址

```bash

# _sidata 指向 Flash 里那份初始数据, 这是一个内置函数, 获取flash里面的data的位置
_sidata = LOADADDR(.data);

# >RAM1 .data 运行时地址在 RAM1, AT> FLASH：但它的初始内容存放在 Flash
.data : {
    _sdata = .; # 在 RAM 中的起止地址
    *(.data)
    *(.data*)
    _edata = .;
} >RAM1 AT> FLASH 
```
