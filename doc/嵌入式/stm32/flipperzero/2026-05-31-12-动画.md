# 动画

## 添加动画

在asset文件夹里面保存各种动画的配置文件
动画分为三种

- blocking - 用于阻塞系统通知的关键动画。打包为 `assets_dolphin_blocking.[h,c]`。
- internal - 用于海豚空闲动画的内部动画。转换为 `assets_dolphin_internal.[h,c]`。
- external - 用于海豚空闲动画的外部动画。打包到资源文件夹并放在 SD 卡上。

### 配置文件

#### manifest.txt

包含随机动画选择的动画枚举, 海豚的起点

```bash
# 固定的文件头
Filetype: Flipper Animation Manifest
Version: 1

# Animation 1
# 动画名称。必须与动画目录名完全一致
Name: L0_NoDb_128x51
# 此动画对应的 Dolphin `butthurt` 值范围
Min butthurt: 0
Max butthurt: 0
# 此动画对应的 Dolphin 等级范围。如果为 `0`，则该动画不会参与随机待机动画选择，
# 只能通过精确名称选中
Min level: 0
Max level: 0
# 该动画在随机选择时被选中的概率权重
Weight: 0

# Animation 2
Name: L0_SdBad_128x51
Min butthurt: 0
Max butthurt: 0
Min level: 0
Max level: 0
Weight: 0

```

#### meta.txt

按顺序播放的 Flipper Format 文件

```bash
# 固定头部
Filetype: Flipper Animation
Version: 1

# 动画宽度，单位为像素
Width: 128
Height: 51
# 播放一次的动画的位图帧数量
Passive frames: 6
# 循环的动画帧的数量
Active frames: 0
# 位图帧的播放顺序，其中前 N 帧是播放一次，后续 M 帧是循环状态。
# 顺序中的每个数字 X 都对应一个位图帧，其文件名为 `frame_X.bm`。
# 该文件必须存在。任意 X 都可以重复，以便在动画中引用同一帧
Frames order: 0 1 2 3 2 1

# N 个循环帧在完整主动周期中的重复次数。例如，如果主动周期帧是 `6` 和 `7`，
# 并且 `Active cycles` 为 `3`，那么完整循环周期会播放 `6 7 6 7 6 7`。
Active cycles: 0
# 每秒播放的帧数
Frame rate: 2
# 单个动画总播放时长
Duration: 0
# 进入循环之前的暂停时间
Active cooldown: 0

# 显示文字的序号
Bubble slots: 0
```

可以使用气泡显示一些提示信息, 在进入循环里面的时候随机选择一个slot播放里面的信息, 一个slot里面可以有多个不同的内容

```bash
Bubble slots: 2

# 同一个 Slot 的多条记录会串成一条序列，按文件出现顺序依次播放

Slot: 0
X: 22
Y: 40
Text: I miss you
AlignH: Right
AlignV: Bottom
StartFrame: 9
EndFrame: 11


# slot 0 是一条两段式对白序列：先显示 I miss you，然后切到 my friend...
Slot: 0
X: 17
Y: 40
Text: my friend...
AlignH: Right
AlignV: Bottom
StartFrame: 12
EndFrame: 15

Slot: 1

Y: 29
Text: Why are you\nalways away?
AlignH: Right

```
