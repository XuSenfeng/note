# 编程

uFBT 部署的应用会存到 Flipper Zero 的**SD 卡**中，且 Flipper 完全支持**动态加载**（通过 FAP 格式 + App Loader 实现，无需刷写固件即可运行新应用）
## 工具

使用软件 [Vscode](https://code.visualstudio.com/)
笔记软件 [obsidian](https://obsidian.md/), 随记 [花笺](https://github.com/Achilng/floral-notepaper)

### 环境搭建

-   `applications` - Applications and services used in firmware
-   `applications_users` - Place for your additional applications and services
-   `assets` - Assets used by applications and services
-   `documentation` - Documentation generation system configs and input files
-   `furi` - Furi Core: OS-level primitives and helpers
-   `lib` - Our and 3rd party libraries, drivers, tools and etc...
-   `site_scons` - Build system configuration and modules
-   `scripts` - Supplementary scripts and various python libraries
-   `targets` - Firmware targets: platform specific code

使用的代码

```c
#include "furi_hal.h"

int gpio_control_main(void* p) {
    UNUSED(p);

    furi_hal_gpio_init_simple(&gpio_ext_pa7, GpioModeOutputPushPull);
    do {
        furi_hal_gpio_write(&gpio_ext_pa7, true);
        furi_delay_ms(500);
        furi_hal_gpio_write(&gpio_ext_pa7, false);
        furi_delay_ms(500);
    } while(furi_hal_gpio_read(&gpio_button_ok));
    furi_hal_gpio_init_simple(&gpio_ext_pa7, GpioModeAnalog);

    return 0;
}
```

### 编译烧录

烧录全部的代码

```bash
./fbt flash # 调试器SWD
# 烧录固件最小更新包
./fbt flash_usb
# 烧录完整的固件，包含主固件、资源、无线电堆栈等所有组件
./fbt flash_usb_full

# 生成clangd使用的文件, 文件在.vscode里面
ufbt cdb

./fbt firmware_all 或者 fw_dist # 编译完整固件 + 生成刷机包
./fbt faps 或者 fap_dist # 编译所有的第三方 FAP 应用
./fbt fap_{APPID} 或者 build APPSRC={APPID} # 编译一个应用
./fbt fap_deploy # 安装所有的fap到开发板
```

只有一个软件的烧录

```bash
ufbt launch
```

当执行 `ufbt launch` 时，uFBT 会通过 USB 将 `.fap` 上传到 Flipper 的**SD 卡指定目录**：

- 默认开发板上面的路径：

    ```
    /ext/apps/<CATEGORY>/<APPID>.fap
    ```

    -   `<CATEGORY>` 由应用 `application.fam` 中的 `fap_category` 字段定义（如 `Tools`、`GPIO`、`Games`）
    -   示例：`/ext/apps/Tools/my_first_app.fap`

    

- 存储介质：**必须是 SD 卡**（Flipper 内部 Flash 仅用于固件和系统配置，不存储第三方应用）

- 可以使用快捷键打开执行默认的各种task(但是实际使用的时候需要添加你使用的ufbt路径在task.json文件里面)

> 在vscode里面可以使用命令`Ctrl + Shift+ B`打开任务列表, 里面有烧录的任务

## 所有的task

- **Build App** 是为了开发单个小程序。
- **Build Firmware** 是为了编译系统内核。
- **Build update bundle**（更新包）则是为了**打包整个生态环境**，方便在不同设备间分发和安装。

### 编译固件

+ **`[Release]`** Build Firmware: 编译整个 Flipper Zero 系统的固件, 开启了代码压缩 (`COMPACT=1`) 并关闭了调试符号 (`DEBUG=0`)，生成的固件体积最小
+ **`[Debug] Build Firmware`**: 以调试模式编译固件。包含调试符号，方便开发者进行代码调试。

### 硬件烧录

+ **`[Release/Debug] Flash (SWD)/(JLink)`**: 使用标准的 SWD/JLink 接口
+ **`(USB, w/o resources)`**: 通过 USB 数据线更新固件，但不更新 SD 卡上的资源文件（图标、数据库等）
+ **`[Release/Debug] Flash (USB, with resources)`**: 通过 USB 更新固件，并同步更新 SD 卡上的所有资源文件
+ **`[Debug:unit_tests] Flash (USB)`**: 专门编译并烧录包含单元测试的固件

### 更新包制作

- **`Build update bundle`**: 制作一个可以在 SD 卡上运行的更新包（通常是 `.tgz` 格式）。
- **`Build updater`**: 专门构建更新器（Updater）程序本身。

### 应用开发

- **`Build FAPs`**: 编译系统中所有的外部应用程序。
- **`Build App`**: **（非常有用）** 仅编译你当前正在编辑的应用文件夹。它使用 `${relativeFileDirname}` 自动识别你当前打开的文件所在的目录。
- **`Launch App on Flipper`**: **（开发首选）** 编译当前应用并直接通过 USB 发送到 Flipper Zero 硬件上立即运行。
- **`Launch App on Flipper with Serial Console`**: 运行应用的同时打开串口终端，方便查看代码中的 `FURI_LOG_D` 等调试输出。
- **`Build and upload all FAPs...`**: 编译所有应用并批量上传到 Flipper 的 SD 卡中。

### 辅助工具

- **`Create PVS-Studio report`**: 使用 PVS-Studio 工具生成静态代码分析报告，用于排查潜在的代码漏洞。
- **`Serial Console`**: 打开 Flipper Zero 的命令行终端（CLI）。你可以通过它与设备交互，就像在 Windows 的 CMD 里一样，输入指令控制 Flipper，或者查看系统运行日志。

## 配置文件

默认配置变量写在配置文件 `fbt_options.py` 里。**命令行输入的参数优先级比配置文件更高**, 也可以新建一个文件叫 `fbt_options_local.py`，加载默认配置时会自动读取它；这样你就能永久修改默认配置，**还不用改动官方的原始配置文件**。

运行 `./fbt -h` 可以查看所有可用的配置选项

官方的**总配置文件**，里面存了编译固件的所有默认参数（比如编译模式、输出路径、默认包含的功能）

### 参数

+   FIRMWARE_APP_SET

应用预设套餐用 `FIRMWARE_APPS` 配置，它是一个**映射表**：`配置名称: 应用列表`。想指定用哪个套餐编译，就把 `FIRMWARE_APP_SET` 设置为对应的名称。

例子：编译带单元测试的固件，运行 `./fbt FIRMWARE_APP_SET=unit_tests`。

扫描的目录在`APPDIRS`

```python
FIRMWARE_APPS = {
    "default": [
        # Svc
        "basic_services",
        # Apps
        "main_apps",
        "system_apps",
        # Settings
        "settings_apps",
    ],
    "unit_tests": [
        "basic_services",
        "updater_app",
        "radio_device_cc1101_ext",
        "unit_tests",
        "js_app",
        "infrared",
        "archive",
    ],
}
```

+   `VERBOSE`：是否打印完整编译命令
+   `FORCE`：强制执行编译 / 刷机
+   `DEBUG`：开启调试版固件
+   `TARGET_HW`：**Flipper 硬件版本**, 自动识别
+   `DIST_SUFFIX / UPDATE_VERSION_STRING`：固件版本后缀
+   `LOADER_AUTOSTART`: 开机启动的程序
+   所有 `COPRO_` 开头：蓝牙协处理器底层配置
+   `SVD_FILE / OPENOCD_OPTS / BLACKMAGIC`：专业调试器配置
+   `FLIP_PORT / LANG_SERVER` 等：工具链 / 端口配置, 自动处理

## 数据存储

Flipper Zero 采用**分层存储架构**，将系统关键数据与用户大容量数据分离，通过统一的**Storage 服务层**管理所有文件操作，提供类 Unix 的路径访问方式。核心设计原则：**内部 FLASH 存关键配置（快但小），SD 卡存用户数据与应用（大但访问慢）**

### 文件架构

Flipper Zero 通过**虚拟文件系统**抽象存储介质，提供两个核心挂载点

+   **`/int/`**：内部 FLASH 存储（LittleFS 文件系统），空间约**1-2MB**，访问速度快，断电数据不丢失
+   **`/ext/`**：外部 microSD 卡存储（FAT32/exFAT 文件系统），容量可达**64GB**，适合大容量数据

所有文件操作通过统一的**Storage API**完成，应用程序无需关心底层存储介质

### 内部FLASH

内部 FLASH 采用**LittleFS 文件系统**，位于 STM32WB55 微控制器的内置闪存中，是固件更新后**唯一保留数据**的区域

| 分区                              | 作用               | 内容                                     |
| --------------------------------- | ------------------ | ---------------------------------------- |
| **引导加载程序（Bootloader）**    | 启动设备、校验固件 | 初始化代码、固件校验逻辑                 |
| **主固件（Firmware）**            | 核心系统功能       | 操作系统、内置应用、驱动程序             |
| **蓝牙协处理器栈（Copro Stack）** | 蓝牙通信           | ST 官方 BLE 协议栈（ble_light/ble_full） |
| **LittleFS 存储区**               | 用户配置与系统数据 | 所有`/int/`下的文件（动态分配剩余空间）  |

在最后一个分区里面可以找到以下的文件

| 目录 / 文件         | 用途                   | 重要性                                        |
| ------------------- | ---------------------- | --------------------------------------------- |
| **/int/config/**    | 系统配置文件           | 网络设置、蓝牙配对信息、UI 偏好、模块校准数据 |
| **/int/pairing/**   | 蓝牙配对密钥           | 已配对设备的加密密钥，确保连接安全            |
| **/int/logs/**      | 系统日志               | 错误报告、调试信息，用于故障排查              |
| **/int/apps_data/** | 内置应用数据           | 计算器历史、设置状态、小工具配置              |
| **/int/keys/**      | 密钥备份（无 SD 卡时） | 仅在无 SD 卡时使用，不推荐                    |

OTA 更新时会先备份`/int/`数据到 SD 卡，更新后自动恢复

### 外部SD卡

```bash
/ext/
├── apps/              # FAP应用包（Flipper Application Package）
├── assets/            # 资源文件（图标、字体、动画、红外数据库）
├── badusb/            # BadUSB脚本（ Rubber Ducky 格式）
├── dolphin/           # 海豚宠物数据与皮肤
├── infrared/          # 红外遥控码库（.ir文件）
├── ibutton/           # iButton密钥（.ibtn文件）
├── lfrfid/            # RFID/LF卡数据（.rfid文件）
├── nfc/               # NFC卡数据（.nfc文件）
├── subghz/            # Sub-GHz遥控数据（.sub文件）
├── u2f/               # U2F安全密钥数据
├── music_player/      # 音乐文件（.wav格式）
├── Manifest           # 资源清单文件（系统自动生成）
└── favorites.txt      # 收藏夹配置文件
```

+   **固件更新**：支持通过 SD 卡本地升级固件（将更新包放在`/ext/update/`目录）
+   **资源扩展**：可通过添加自定义资源包（如红外码库、主题）扩展设备功能
+   **动态应用加载**：FAP 文件从 SD 卡加载到 RAM 执行，不占用 FLASH 空间

1. Furi 核心 (/furi/)

基于 FreeRTOS 的操作系统抽象层，提供：                                                                         
  - 线程管理 - thread.h/c
  - 同步原语 - mutex.h, semaphore.h
  - 消息队列 - message_queue.h
  - 事件循环 - event_loop.h
  - 发布订阅 - pubsub.h
  - 服务注册 - record.h
  - 内存管理 - memmgr.h
  - 日志系统 - log.h
    2. 硬件抽象层 (/targets/furi_hal_include/, /targets/f7/furi_hal/)

  统一的硬件接口：                     
  - furi_hal_nfc.h - NFC 读写
  - furi_hal_subghz.h - Sub-GHz 无线电
  - furi_hal_infrared.h - 红外收发
  - furi_hal_rfid.h - 低频 RFID
  - furi_hal_bt.h - 蓝牙低功耗
  - furi_hal_usb.h - USB 协议栈
  - furi_hal_gpio.h - GPIO 控制

### 应用框架

应用类型定义在 application.fam 清单文件中：
  - SERVICE - 启动时运行的服务                                                                                   
  - APP - 主菜单应用                                                                                             
  - SYSTEM - 系统应用 (可通过CLI启动)  
  - SETTINGS - 设置菜单
  - EXTERNAL - 外部 .fap 应用 (从SD卡加载)                                                                                                              

+   主要应用 (/applications/main/):

  - nfc/ - NFC/RFID 读写 
  - subghz/ - Sub-GHz 无线电
  - infrared/ - 红外遥控
  - bad_usb/ - USB HID 攻击
  - gpio/ - GPIO 控制和 UART 桥接 
  - lfrfid/ - 低频 RFID
  - ibutton/ - iButton 密钥                                                                                                             

+   系统服务 (/applications/services/):                                                                            

  - gui/ - 图形界面服务                                                                                          
  - storage/ - 文件系统服务                                                                                      
  - cli/ - 命令行接口                                                                                            
  - loader/ - 应用加载器                                                                                         
  - bt/ - 蓝牙服务                                                                                               
  - power/ - 电源管理
