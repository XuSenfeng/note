Linux CAN

[[2025-11-19-72-CAN]]

CAN 帧是 CAN 协议中数据传输的基本单元，它包括以下几种类型：

  • 数据帧：用于传输数据。
  • 遥控帧：请求发送数据帧。
  • 错误帧：标识错误的帧。
  • 过载帧：用于延长数据帧或遥控帧的传输。

Socket CAN

SocketCAN 是 Linux 内核中实现 CAN 协议的框架，它将 CAN 设备作为网络接口来处理，使得用户可以使用标准的网络协议栈与 CAN 设备进行通信
它为用户空间应用程序提供了一个 socket 接口，并且建立在 Linux 网络层之上，从而可以使用所有提供的排队功能
可以同时打开多个 socket，在不同的或相同的协议模块上，它们可以监听/发送不同或相同的 CAN ID 的帧。多个 socket 在同一个接口上监听具有相同 CAN ID 的帧，都会收到相同的匹配 CAN 帧
希望使用特定传输协议（例如 ISO-TP）进行通信的应用程序只需在打开 socket 时选择该协议，然后就可以读取和写入应用程序数据字节流，而无需处理 CAN-ID、帧等

  • candump：用于监视 CAN 设备上的数据传输。
  • cansend：用于向 CAN 设备发送数据帧。
  • canreceive：用于接收 CAN 设备上的数据帧。

与通常已知的 TCP/IP 和以太网网络不同，CAN 总线是一个仅广播的媒介，没有像以太网这样的 MAC 层寻址。CAN 标识符 (can_id) 用于 CAN 总线上的仲裁。因此，CAN-ID 必须在总线上唯一选择。在设计 CAN-ECU 网络时，CAN-ID 被映射为由特定的 ECU 发送。因此，CAN-ID 可以最好地被视为一种源地址

can-utils

一组用于 Linux 下 SocketCAN 的实用程序集合，它包括了许多用于测试和调试 CAN 设备的工具

  • canconfig：用于配置 CAN 接口的参数。
  • cantest：用于测试 CAN 接口的发送和接收功能。
  • canfdtest：类似于 cantest，但支持 CAN FD（Flexible Data Rate)

candump

candump 是一个用于监视和记录 CAN 总线上所有通信的实用工具。它能够实时显示所有接收到的 CAN 帧，并且可以将数据记录到文件中，以便后续分析

cansend

cansend 工具允许用户手动发送 CAN 帧到指定的 CAN 接口, 用户可以指定 CAN 帧的 ID、数据长度和要发送的数据

canreceive

用于接收并显示从 CAN 接口收到的数据帧。它通常用于测试或调试目的

配置网络

设置一个接口参数

sudo ip link set can0 up type can bitrate 500000

can0 是接口名称，bitrate 500000 设置了 CAN 接口的波特率为 500kbps

网络设置

创建一个虚拟的CAN卡用于测试

sudo ip link add vcan0 type vcan
sudo ip link set vcan0 up

如果与远程 CAN 网络通信，需要设置远程地址

sudo ip link set can0 address <remote-can-address>

在多接口的环境下面需要配置路由规则

sudo ip route add <destination> dev <interface>

配置参数

sudo canconfig can0 bitrate 1000000
# 配置CANFD的参数
sudo ip link set can0 up type can fd on bitrate 500000 fd bitrate 2000000

数据收发

cansend can0 123#11FF2233

API

有两种 CAN 协议可供选择，即 raw socket 协议和广播管理器 (BCM)

s = socket(PF_CAN, SOCK_RAW, CAN_RAW);
s = socket(PF_CAN, SOCK_DGRAM, CAN_BCM);

成功创建 socket 后，您通常会使用 bind(2) 系统调用将 socket 绑定到 CAN 接口
绑定 (CAN_RAW) 或连接 (CAN_BCM) socket 后，您可以像往常一样从/向 socket read(2) 和 write(2)，或者在 socket 上使用 send(2)、sendto(2)、sendmsg(2) 和 recv* 对等操作

一帧数据

Classical CAN 帧结构（也称为 CAN 2.0B）、CAN FD 帧结构

struct can_frame {
        canid_t can_id;  /* 32 bit CAN_ID + EFF/RTR/ERR flags */
        union {
                /* CAN frame payload length in byte (0 .. CAN_MAX_DLEN)
                 * was previously named can_dlc so we need to carry that
                 * name for legacy support
                 */
                __u8 len; // 有效数据的长度
                __u8 can_dlc; /* deprecated */
        };
        __u8    __pad;   /* padding */
        __u8    __res0;  /* reserved / padding */
        // 当 len 元素为 8 时，len8_dlc 元素可以包含值 9 .. 15
        // （所有大于或等于 8 的 DLC 值的实际有效负载长度）
        __u8    len8_dlc; /* optional DLC for 8 byte payload length (9 .. 15) */
        __u8    data[8] __attribute__((aligned(8)));
};

使用示例

int s;
struct sockaddr_can addr;
struct ifreq ifr;

s = socket(PF_CAN, SOCK_RAW, CAN_RAW);

# 进行实际的绑定工作
strcpy(ifr.ifr_name, "can0" );
ioctl(s, SIOCGIFINDEX, &ifr);

addr.can_family = AF_CAN;
// 可以设置为0表示绑定任意的CAN接口
addr.can_ifindex = ifr.ifr_ifindex;

bind(s, (struct sockaddr *)&addr, sizeof(addr));

struct can_frame frame;

# 读取数据
nbytes = read(s, &frame, sizeof(struct can_frame));

if (nbytes < 0) {
        perror("can raw socket read");
        return 1;
}

/* paranoid check ... */
if (nbytes < sizeof(struct can_frame)) {
        fprintf(stderr, "read: incomplete CAN frame\n");
        return 1;
}

# 写入数据
nbytes = write(s, &frame, sizeof(struct can_frame));

要在绑定到“任何”CAN 接口的 socket 上写入 CAN 帧，必须确定传出接口

strcpy(ifr.ifr_name, "can0");
ioctl(s, SIOCGIFINDEX, &ifr);
addr.can_ifindex = ifr.ifr_ifindex;
addr.can_family  = AF_CAN;

nbytes = sendto(s, &frame, sizeof(struct can_frame),
                0, (struct sockaddr*)&addr, sizeof(addr));

CAN FD API

一帧数据

struct canfd_frame {
        canid_t can_id;  /* 32 bit CAN_ID + EFF/RTR/ERR flags */
        __u8    len;     /* frame payload length in byte (0 .. 64) */
        __u8    flags;   /* additional flags for CAN FD */
        __u8    __res0;  /* reserved / padding */
        __u8    __res1;  /* reserved / padding */
        __u8    data[64] __attribute__((aligned(8)));
};

当 struct can_frame 的内容被复制到 struct canfd_frame 中时，所有结构元素都可以按原样使用 - 只有 data[] 变得扩展了

过滤器

使用 CAN_RAW socket 接收 CAN 帧可以通过定义 0 .. n 个带有 CAN_RAW_FILTER socket 选项的过滤器来控制

struct can_filter {
        canid_t can_id;
        canid_t can_mask;
};

可以获取到<received_can_id> & mask == can_id & mask的所有数据

struct can_filter rfilter[2];

rfilter[0].can_id   = 0x123;
rfilter[0].can_mask = CAN_SFF_MASK;  // 标准帧全掩码（0x7FF）
rfilter[1].can_id   = 0x200;
rfilter[1].can_mask = 0x700; // 掩码：只匹配高3位，低8位无视

setsockopt(s, SOL_CAN_RAW, CAN_RAW_FILTER, &rfilter, sizeof(rfilter));

可以使用以下的命令禁用所有的接受数据

setsockopt(s, SOL_CAN_RAW, CAN_RAW_FILTER, NULL, 0);

调试

watch -n1 sudo ip -d -s link show can1

实时、每秒自动刷新监控 can1 网卡的【详细硬件状态 + 收发统计 + CAN 错误计数器】，不用你反复手动敲命令，故障变化一眼就能看到。

cangen can1

自动生成随机的 CAN 报文，并持续发送到指定的 CAN 接口（can0/can1）

candump any -tA -xe

实时监听系统里所有 CAN 接口（can0 + can1 同时看），打印带绝对时间戳的报文，并且强制显示 CAN 总线错误帧、异常状态
