LInux库控制

2026-06-25
编辑本页

实际使用的库是 libethercat

IgH EtherCAT为运行于Linux系统的开源EtherCAT主站程序，IgH EtherCAT主站通过构建Linux字符设备，应用程序通过对字符设备的访问实现与EtherCAT主站模块的通信。

IgH EtherCAT开发包配套EtherCAT工具，该工具提供各种可在Linux用户层运行的命令，可直接实现对从站的访问和设置，如设置从站地址、显示总线配置、显示PDO数据、读写SDO参数等。

https://docs.etherlab.org/ethercat/1.5/doxygen/index.html 文档
https://gitlab.com/etherlab.org/ethercat/-/tree/stable-1.6 源码

内核

EtherCAT IgH需要保证高实时性，Preempt-RTRT是一种针对实时性能进行了优化的Linux内核

  1. 实时性能： Preempt-RT提供了更可靠和更精确的实时性能。它采用了一些实时调度策略和机制，使得任务能够按照严格的时间要求执行，从而适用于需要高度可预测性和低延迟的应用场景，如工业自动化、机器人控制等。
  2. 硬实时能力： Preempt-RT具有硬实时能力，即能够确保任务在规定的时间内完成，而不会受到其他任务或中断的干扰。这对于需要严格的时间限制的应用非常重要，如航空航天、医疗设备等领域。
  3. 任务调度： Preempt-RT使用了更加高效和优化的任务调度算法，如基于优先级的实时调度算法，以确保高优先级任务能够及时响应并完成，而低优先级任务不会影响到实时任务的执行。
  4. 中断处理： Preempt-RT针对中断处理进行了优化，使得中断的响应时间更短，能够更快地响应外部事件。
  5. 内核定时器： Preempt-RT提供了更精确和可配置的内核定时器，使得可以实现微秒级的定时精度，适用于对时间要求极高的应用场景。
  6. 实时扩展： Preempt-RT提供了一些实时扩展机制，使得用户能够方便地对内核进行定制和扩展，以满足特定应用的需求。

SM（Sync Manager，同步管理器）：EtherCAT 从站的硬件数据通道，共 4 个标准通道，分工明确：

  • SM0/SM1：邮箱通道，传输非周期数据（比如配置参数、SDO 读写、故障告警），不参与周期实时控制。
  • SM2/SM3：过程数据通道，传输周期实时数据，也就是 PDO 数据，是运动控制的核心。
  • RxPDO：站在从站视角，是「从站接收的数据」，也就是主站→从站的控制指令（比如目标位置、控制字）。
  • TxPDO：站在从站视角，是「从站发送的数据」，也就是从站→主站的状态反馈（比如实际位置、状态字）。

对象字典索引：比如 0x6040:00，是 CoE 协议的标准寻址方式，冒号前是主索引，冒号后是子索引。0x6000~0x6FFF 是 CiA 402 伺服标准定义的对象，0x2000~0x2FFF 是厂商自定义对象。

API控制流程

  1. 使用命令启动服务, 开启ethernet设备
  2. 初始化申请一个master
  3. 创建domain的实际数据发送使用的内存空间
  4. 创建从站的数据句柄
  5. 注册 PDO 条目
  6. 激活主站
  7. 获取实际的映射指针
  8. -循环-
  9. 接受数据
  10. 处理数据
  11. 写入控制字
  12. 输出排入队列
  13. 发送
  14. -结束循环-

API函数

示例

#include <ecrt.h>
#include <unistd.h>

/* ───── 1. 全局句柄 ───── */
static ec_master_t  *master    = NULL;   // 主站
static ec_domain_t  *domain    = NULL;   // 过程数据域
static uint8_t      *domain_pd = NULL;   // 域的进程映像指针

/* 从站身份 (按你的设备改) */
#define ALIAS       0
#define POSITION    0                 // 总线位置
#define VENDOR_ID   0x00000168
#define PRODUCT_ID  0x00000011

/* ───── 2. 要交换的 PDO 条目, 偏移会被自动填进这两个变量 ───── */
static unsigned int off_ctrl;         // 0x6040 控制字 (输出)
static unsigned int off_status;       // 0x6041 状态字 (输入)

/* ───── 3. 注册表: "把这些对象登记到 domain, 偏移写回 offset 变量" ───── */
const static ec_pdo_entry_reg_t domain_regs[] = {
    {ALIAS, POSITION, VENDOR_ID, PRODUCT_ID, 0x6040, 0x00, &off_ctrl},
    {ALIAS, POSITION, VENDOR_ID, PRODUCT_ID, 0x6041, 0x00, &off_status},
    {}                                // 结束标记(全0)
};

int main(void)
{
    /* ① 请求主站 (索引0) */
    master = ecrt_request_master(0);
    if (!master) return -1;

    /* ② 创建过程数据域 */
    domain = ecrt_master_create_domain(master);
    if (!domain) return -1;

    /* ③ 获取从站配置句柄 */
    ec_slave_config_t *sc =
        ecrt_master_slave_config(master, ALIAS, POSITION, VENDOR_ID, PRODUCT_ID);
    if (!sc) return -1;

    /* ④ 注册 PDO 条目 (算偏移)， 实际的偏移记录在给的指针里面 */
    if (ecrt_domain_reg_pdo_entry_list(domain, domain_regs)) return -1;

    /* ⑤ 激活主站 (之后不能再改配置) */
    if (ecrt_master_activate(master)) return -1;

    /* ⑥ 拿到进程映像指针 */
    domain_pd = ecrt_domain_data(domain);
    if (!domain_pd) return -1;

    /* ⑦ 实时循环 */
    while (1) {
        ecrt_master_receive(master);   // 收: 把网线上回来的数据搬进映像
        ecrt_domain_process(domain);   // 处理输入

        /* —— 在这里读写过程数据 —— */
        uint16_t status = EC_READ_U16(domain_pd + off_status);  // 读状态字
        EC_WRITE_U16(domain_pd + off_ctrl, 0x000F);             // 写控制字
        (void)status;

        ecrt_domain_queue(domain);     // 把输出排进队列
        ecrt_master_send(master);      // 发: 把映像发到网线

        usleep(1000);                  // 1ms 周期
    }
    return 0;
}
