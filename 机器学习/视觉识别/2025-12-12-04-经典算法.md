# 经典算法

## two-stage和one-stage

+   one-stage: 输入到输出只要一个网络, Yolo系列

速度比较快, 适合实时任务, 但是效果比较差

+   tow-stage: 在输出的时候首先输出一系列结果作为初选, 之后在进一步进行预测, Fast-rcnn Mask-rcnn系列

速度比较慢, 达不到实时

## 指标

### map

一般来看精度以及召回率, 精度是边框是不是准确的位置, 召回率是看是不是都找到了

Precision = TP / (TP + FP)

Recall = TP / (TP + FN)

一般来说这两个值是负相关的, 把这两个值通过不同的置信度阈值进行分别计算画图, map是围成的图像的面积

<img src="https://picture-01-1316374204.cos.ap-beijing.myqcloud.com/mac-picture/image-20251212190539505.png" alt="image-20251212190539505" style="zoom:50%;" />

使用的是红框的面积

>   TP: True positives选出来正确获取到的数量
>
>   FP: false positives 选出来不正确
>
>   FN: false negatives 正类型判断为负类型
>
>   TN: true negatives 负类型判断为负类型
>
>   实际在获取以上的数值的时候, 需要考虑IoU以及判断出来的置信度, 不同的置信度阈值对应的数值是不同的



### IoU

两个框的交集和并集的比值 ,可以算两个框的吻合程度
